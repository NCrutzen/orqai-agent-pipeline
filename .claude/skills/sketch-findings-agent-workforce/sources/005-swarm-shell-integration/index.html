Variant C — Unified swarm dashboard at /swarm/[swarmId]

The existing /swarm/[swarmId] becomes the home for everything. A pill-button section nav switches between ⚙ Operations (today's briefing + delegation graph + jobs Kanban + timeline + terminal — engineer-focused) and the new 📥 Bulk Review + 📋 Email Kanban sections (Phase 76 — operator-focused). The URL stays /swarm/[swarmId]?section=kanban or with a hash. /automations/[swarm]/* routes either redirect to this shell or are removed entirely.

Single URL per swarm — easiest to reason about, easiest to share. Engineer + operator views in one place. The "swarm" mental model becomes the central anchor. Future surfaces (cost dashboards, agent runs viewer, etc.) drop in as new sections.
Significant re-architecture: Bulk Review's existing routes (with cursor pagination, ?selected=, ?topic= filters, etc.) all migrate to the section-nav pattern. Existing bookmarks break. Engineer + operator audiences see overlapping nav even when one of the modes is irrelevant to them. Different visual densities (engineer dashboard is dense + multi-section; operator surfaces are typically focused + single-task) clash on one shell. Highest risk for Phase 76's timebox.
