## Integration Breaks

### Break 1: Holdout Dataset ID Schema Mismatch

**From:** `prompt-editor.md` Phase 1.2
**To:** `dataset-prep.json` (written by `dataset-preparer.md` Phase 8)

prompt-editor reads `holdout_dataset_id` via `per_agent_datasets[]` array path. But `dataset-prep.json` stores per-agent data under `agents.{agent_key}` as a flat map. The `per_agent_datasets[]` shape exists only in `test-results.json`.

**Impact:** Holdout re-test path is unreachable. `holdout_dataset_id` will be null, triggering prompt-editor's STOP guard. The iterate loop will still run (spec editing + deployer work) but won't verify improvements via holdout re-test.

**Fix:** Update `prompt-editor.md` Phase 1.2 to read holdout IDs from `dataset-prep.json` using `agents.{agent_key}.holdout_dataset_id`, or from `test-results.json`'s `dataset.per_agent_datasets[]` (matching the old iterator.md pattern).

**Affected:** ITPIPE-05, EXPR-05, LOOP-01

### Break 2: mcp_available Not Forwarded by iterate.md

**From:** `iterate.md` Step 5.4
**To:** `prompt-editor.md` (which delegates to `deployer.md`)

iterate.md detects MCP availability in Step 2 but doesn't forward it to prompt-editor. When MCP is unavailable, deployer will attempt MCP calls first and fail before falling back to REST.

**Impact:** Performance degradation in MCP-unavailable environments (unnecessary MCP attempt + fallback delay). Not a hard break -- deployer's REST fallback will eventually succeed.

**Fix:** Add `mcp_available` to iterate.md Step 5.4 forwarded context list.

**Affected:** LOOP-01, ITPIPE-05
