## Gap: Embedding Model Recommendation Missing from Researcher Output

**Status:** PARTIAL FAILURE vs ROADMAP success criterion 2

**Root cause:** ROADMAP.md success criterion 2 states the researcher should include "embedding model recommendation." However, plan 02 explicitly decided to defer embedding model selection to Phase 4.5, and implemented a constraint prohibiting any embedding model mention. The implementation is coherent and deliberate — the gap is between the ROADMAP criterion wording and the plan's design decision.

**Evidence of deliberate deferral:**
- `researcher.md` line 117: "Embedding model selection is deferred to Phase 4.5. Do NOT recommend specific embedding models"
- `researcher.md` line 204: "Note: Embedding model selection deferred to Phase 4.5."
- `orchestration-generator.md` line 586: "No embedding model recommendations in KB Design section" (pre-output checklist item)
- Plan 02 summary: "Metadata fields constrained to source and timestamp only — no additional fields per user decision"

**Recommended resolution:** Update ROADMAP.md success criterion 2 to remove "embedding model recommendation" and replace with "chunking strategy and metadata fields" to accurately reflect the implemented scope. This is a ROADMAP documentation update, not a code fix.
