**Insert after line 164 (after backup cleanup `rm -rf "${INSTALL_DIR}.backup"`) and before the success banner:**

1. **Tier Selection** (INST-01):
   - Display tier comparison table (per research code example):
   ```
   Select your capability tier:

     ┌──────────┬─────────────────────────────────────────┐
     │ Tier     │ What You Get                            │
     ├──────────┼─────────────────────────────────────────┤
     │ core     │ /orq-agent (spec generation only)       │
     │ deploy   │ + /orq-agent:deploy (push to Orq.ai)    │
     │ test     │ + /orq-agent:test (automated testing)   │
     │ full     │ + /orq-agent:iterate (prompt iteration)  │
     └──────────┴─────────────────────────────────────────┘

     Each tier includes all capabilities of lower tiers.
   ```
   - Read user input, validate (core|deploy|test|full), default to core on invalid input
   - If this is a re-install (config.json exists), show current tier and offer to keep or change

2. **API Key Validation** (INST-02):
   - Only prompt if tier is NOT core (deploy/test/full need API access)
   - If ORQ_API_KEY already exists in environment, offer to keep existing or enter new
   - Read API key with `-sp` (silent, no echo)
   - Validate via: `curl -sf -o /dev/null -w "%{http_code}" -H "Authorization: Bearer $ORQ_API_KEY" "https://api.orq.ai/v2/models"`
   - If HTTP 200: success message
   - If not 200: error message with "Check your key and try again", exit 1
   - Write to shell profile idempotently:
     - Detect profile: `$HOME/.zshrc` if exists, else `$HOME/.bashrc`
     - If `export ORQ_API_KEY=` already exists: update in-place with `sed -i.bak`
     - If not: append with comment `# Orq.ai API Key (added by orq-agent installer)`
     - Clean up `.bak` file
     - Export in current session: `export ORQ_API_KEY`
   - NEVER write the API key to config.json or any other file

3. **MCP Server Registration** (INST-03):
   - Only if tier is NOT core
   - Run: `claude mcp add --transport http --scope user orqai-mcp "https://mcp.orq.ai" --env ORQ_API_KEY="$ORQ_API_KEY" 2>/dev/null`
   - Note: MCP server URL may need to be configurable. Use a variable `ORQAI_MCP_URL` defaulting to `https://mcp.orq.ai` -- user can override. If registration fails, warn but don't abort (MCP fallback per INST-05 design).
   - Track success/failure in `MCP_REGISTERED` variable

4. **Config File Creation**:
   - Create `.orq-agent/` directory inside `$INSTALL_DIR`
   - Write `config.json` with: tier, model_profile (default "quality"), model_overrides (empty object), installed_at (ISO timestamp), orqai_mcp_registered (boolean)
   - On re-install: merge with existing config (preserve model_overrides if user changed them)

5. **Update Success Banner**:
   - Show installed tier in success message
   - If deploy+ tier: mention API key is stored and MCP is registered
   - Add `/orq-agent:deploy`, `/orq-agent:test`, `/orq-agent:iterate` to quick-start based on tier
   - Add note: "Restart your terminal or run `source ~/.zshrc` to activate API key"

Use `sed -i.bak` for portability (works on macOS and Linux) per research pitfall #4.
