<task type="auto">
  <name>Task 2: Create install.sh with prerequisite checks, version comparison, and rollback</name>
  <files>install.sh</files>
  <action>
Create `install.sh` at the repository root. This is the curl one-liner entry point per user's locked decision.

The script MUST implement these behaviors in order:

1. **Shebang and safety:** `#!/usr/bin/env bash` with `set -euo pipefail`

2. **Color definitions:** RED, GREEN, YELLOW, NC (no color) for output formatting

3. **Header banner:** Display "Installing Orq Agent Designer..." with styled output

4. **Prerequisite checks (MUST have clear error messages per user decision):**
   - Check `node` exists: if missing, print RED error with install link (https://nodejs.org/)
   - Check `claude` exists: if missing, print RED error with install command (`npm install -g @anthropic-ai/claude-code`)
   - Exit 1 on any missing prerequisite

5. **Install directory:** Set `INSTALL_DIR="$HOME/.claude/skills/orq-agent"` -- research recommends skills directory over commands directory to avoid plugin namespace double-naming. Per research: "Install to ~/.claude/skills/orq-agent/ for maximum simplicity. This creates /orq-agent as a personal skill without plugin namespacing overhead."

6. **Version comparison (per locked decision: check before downloading, skip if up to date):**
   - Fetch remote VERSION from GitHub raw URL: `https://raw.githubusercontent.com/OWNER/REPO/main/VERSION` (use placeholder URL; will be updated when repo is created)
   - Compare with local `$INSTALL_DIR/VERSION` if it exists
   - If versions match, print GREEN "Already up to date (vX.Y.Z)" and exit 0

7. **Backup for rollback (per locked decision: auto-rollback on failure):**
   - If `$INSTALL_DIR` exists, copy to `$INSTALL_DIR.backup`

8. **Download and install:**
   - Create temp directory with `mktemp -d`
   - Set trap to clean up temp dir and backup on exit
   - Download GitHub archive as tarball: `curl -sL "https://github.com/OWNER/REPO/archive/main.tar.gz" | tar xz -C "$TEMP_DIR"`
   - Create install directory: `mkdir -p "$INSTALL_DIR"`
   - Copy the `orq-agent/` contents (not the full repo) into `$INSTALL_DIR/`: agents/, commands/, references/, templates/, SKILL.md
   - Also copy VERSION, CHANGELOG.md, .claude-plugin/ to `$INSTALL_DIR/` for version tracking

9. **Verification (triggers rollback on failure):**
   - Check `$INSTALL_DIR/SKILL.md` exists
   - Check `$INSTALL_DIR/commands/orq-agent.md` exists
   - If either missing: print RED "Installation failed -- restoring backup", restore from backup if exists, exit 1

10. **Clean up backup on success:** Remove `$INSTALL_DIR.backup`

11. **Quick-start guide (per locked decision: 2-3 example commands):**
    ```
    Orq Agent Designer v1.0.0 installed successfully!

    Quick start:
      /orq-agent "Build a customer support triage system"
      /orq-agent:help
      /orq-agent:update
    ```

Make the script executable: the file should have a note at the top (comment) about `chmod +x` but the actual permission will be set in the git commit.

**IMPORTANT:** Use placeholder GitHub URLs (OWNER/REPO) -- these will be replaced when the actual repository is created. Add a comment near the URLs: `# TODO: Replace OWNER/REPO with actual GitHub repository path`
