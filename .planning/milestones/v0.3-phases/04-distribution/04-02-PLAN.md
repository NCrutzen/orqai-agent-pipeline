  <action>
**1. Update `orq-agent/commands/orq-agent.md`** -- Add argument parsing BEFORE Step 1 (Capture Input):

Insert a new section between the frontmatter and Step 1 called "## Step 0: Parse Arguments". This section:

a. Parses `$ARGUMENTS` for flags:
   - `--gsd`: Boolean flag. When present, sets GSD_MODE=true. Does NOT change output directory (per research recommendation: output to `./Agents/` regardless). The flag signals the invocation is part of a GSD workflow for metadata and logging purposes.
   - `--output <path>`: String flag. Overrides the default output directory. If not provided, defaults to `./Agents/`. Example: `--output ./my-agents` outputs to `./my-agents/[swarm-name]/`.

b. After extracting flags, the remaining text in `$ARGUMENTS` becomes the use case description passed to Step 1.

c. Flag parsing rules:
   - Flags can appear anywhere in the arguments string
   - `--output` consumes the next token as the path value
   - Everything that is not a flag or flag value is the use case description
   - If `--gsd` flag present but no `--output`, output directory remains `./Agents/`

d. Add to Step 1 (Capture Input): After storing the input, also store the parsed OUTPUT_DIR and GSD_MODE values for use in later steps.

e. Update the output directory setup step (Step 5 in current orchestrator) to use OUTPUT_DIR instead of hardcoded `./Agents/`.

**IMPORTANT:** Do NOT rewrite the entire orchestrator file. Only ADD the Step 0 section and MODIFY the specific lines in Step 1 and Step 5 that reference the output directory. Preserve all existing orchestrator logic.
