  <name>Task 1: Upgrade orchestration-generator.md with delegation frameworks and tool overlap detection</name>
  <files>orq-agent/agents/orchestration-generator.md</files>
  <action>
Transform the orchestration-generator.md so it produces ORCHESTRATION.md documents that include explicit delegation frameworks and performs cross-swarm tool overlap detection.

**Changes to orchestration generation guidance:**

1. **Delegation framework generation:** The orchestration generator must produce a `<delegation_framework>` section to be embedded in the orchestrator agent's instructions (via spec generator coordination). This section includes:
   - Named list of all sub-agents with roles and when to call each
   - Task decomposition heuristic: assess complexity first, handle simple tasks directly, delegate multi-faceted requests
   - Per-delegation specification requirements: objective, output format, task boundaries (not vague "research X")
   - Synthesis directive: synthesize coherent responses from sub-agent results, never concatenate raw outputs
   - See RESEARCH.md Pattern 4 and Example 2 for the exact framework structure

2. **Effort scaling guidelines:** Add effort scaling section to the generated ORCHESTRATION.md:
   - Simple queries (single-domain, factual): orchestrator handles directly with 3-10 tool calls
   - Moderate queries (comparison, multi-source): delegate to 2-4 agents with clear domain division
   - Complex queries (open-ended, creative): delegate to multiple agents with distinct responsibilities
   - Wrap in `<effort_scaling>` tags within the delegation framework

3. **Collaborative scope awareness (division of labor):** The generated ORCHESTRATION.md must specify that:
   - The orchestrator has the full agent map (all agents, their roles, their boundaries)
   - Each worker agent knows its own scope AND its neighboring agents' responsibilities
   - This information must be explicitly included in both the orchestrator instructions AND each worker agent's instructions (flagged for spec generator to implement)
   - Per CONTEXT.md: "Anthropic found collaborative framework prompts outperform individual-behavior-only prompts"

4. **Tool overlap detection (cross-validation):** Add a new cross-validation section to the orchestration generator's workflow. After reading all generated agent specs:
   - Compare tool lists across all agents in the swarm
   - Flag any tools that appear in multiple agents with identical or overlapping functionality
   - Recommend resolution: either remove from one agent, or clarify distinct purposes in tool descriptions
   - This addresses CONTEXT.md decision: "Tool overlap detection at two stages: during individual spec generation AND cross-validation across the full swarm"
   - Per RESEARCH.md Open Question 2: implement this in the orchestration generator which already reads all specs

5. **Self-contained tool description enforcement:** Add guidance that all tool descriptions in the ORCHESTRATION.md must be self-contained:
   - No cross-references between tools ("use this after calling Y")
   - Minimal viable parameter set
   - Clear single-purpose description
   - Per CONTEXT.md: "If humans can't pick the right tool, agents won't either"

**What NOT to change:**
- Mermaid diagram generation
- Error handling table structure
- HITL identification criteria
- Single-agent swarm simplified output pattern

<task type="auto">
  <name>Task 2: Upgrade researcher.md with context management and heuristic prompt recommendations</name>
  <files>orq-agent/agents/researcher.md</files>
  <action>
Transform the researcher.md so its research briefs include context management recommendations, Memory Store identification, and heuristic-first prompt strategy guidance.

**Changes to the research brief format:**

1. **Context management recommendations per agent:** Add a new mandatory section to the research brief output: "Context Management Strategy." For each agent researched, the researcher must recommend:
   - Whether the agent needs just-in-time retrieval (tool-heavy agents: yes) or can work with upfront context (simple classifiers: no)
   - Whether the agent benefits from summarization directives (agents making multiple tool calls: yes)
   - Expected context budget intensity (low/medium/high based on task complexity and tool count)

2. **Memory Store candidate identification:** Add explicit guidance for identifying Memory Store candidates:
   - Multi-turn conversation agents (customer support, coaching, advisory)
   - Long-running task agents (research, data processing, workflow management)
   - Agents that need to track user preferences, decisions, or progress across interactions
   - When identified, the research brief should include specific Memory Store description text recommendation (what the memory store should be configured to store)
   - See RESEARCH.md Example 3 and Open Question 3

3. **Heuristic prompt strategy guidance:** Transform the "Prompt Strategy" section of the research brief from suggesting rigid rules to suggesting heuristic approaches:
   - Instead of: "The agent should follow these steps: 1) identify intent, 2) categorize, 3) retrieve..."
   - Produce: "The agent should approach tasks the way a skilled [role] would: understand the actual need first, match to available capabilities, act decisively when clear, ask focused questions when ambiguous"
   - Include guidance on which rules should remain explicit (security, data boundaries) vs. which should become examples (tone, edge cases, style)
   - Per CONTEXT.md: "Target heuristic-first altitude -- encode how skilled humans approach the task with decision heuristics, not rigid rules"

4. **Tool description quality guidance:** Add recommendations for tool description quality per agent:
   - Each tool description should be self-contained and minimal
   - Flag potential tool overlap across agents in the same swarm
   - Recommend token-efficient return value structures (structured JSON, not verbose narratives)
   - Per CONTEXT.md: "Tool return values designed for token efficiency always"

**What NOT to change:**
- One-researcher-for-entire-swarm pattern
- 5 mandatory research brief sections (this adds to, not replaces them)
- Web search protocol
- Confidence scoring (HIGH/MEDIUM/LOW)
