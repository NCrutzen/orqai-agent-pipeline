**Step 1: Capture Input** -- Keep EXACTLY as-is. No changes.

**Step 2: Discussion** -- NEW. Replace the current Step 2 (Classify Input Depth) and Step 3 (Confirm with User) entirely with the following subsections:

- **2.1: Analyze Use Case** -- Read the user's input and determine: (1) the domain (customer support, content generation, data processing, workflow automation, etc.), and (2) 3-4 domain-specific gray areas. Gray areas are implementation decisions the user cares about -- things that could go multiple ways and would change the result. They must be specific to the use case domain (NOT generic like "UI" or "Error handling"), and must NOT ask about technical implementation details (models, tools, prompts -- those are Claude's job). Include domain examples:
  - FAQ bot: "Knowledge sources", "Tone & personality", "Escalation behavior", "Scope boundaries"
  - Customer support triage: "Routing criteria", "Priority detection", "Handoff protocol", "Response style"
  - Content generation pipeline: "Output format & style", "Quality guardrails", "Source handling", "Personalization depth"
  - Data processing workflow: "Input validation rules", "Error recovery strategy", "Output structure", "Notification triggers"

- **2.2: Present Gray Areas** -- Display using ORQ banner style (`ORQ > DISCUSSION`). Show domain analysis and gray areas as multi-select with format matching the research Example 1 (checkboxes, 1-2 example questions per area). Include instructions: select comma-separated numbers, "all", or "skip" to proceed directly. If user types "skip": proceed to Step 2.5 with no discussion decisions.

- **2.3: Discuss Selected Areas** -- For each selected area: (1) Announce the area, (2) Ask 4 questions one at a time, each with 2-3 concrete options plus a "You decide" option where reasonable, (3) After 4 questions ask "More questions about [area], or move to next?" -- "More" gives up to 4 more then forces move, "Next" proceeds. Follow GSD discuss-phase cadence exactly.

- **2.4: Compile Discussion Summary** -- After all areas discussed, compile structured summary using ORQ banner (`ORQ > DISCUSSION SUMMARY`). Format:
  ```
  ### Use Case
  [Original user input -- verbatim]

  ### Decisions Made
  - **[Area 1]:** [Decision summary]

  ### Additional Context
  - [Key details surfaced]

  ### Open Areas (Claude's Discretion)
  - [Areas not discussed or "you decide"]
  ```
  Keep summary to 100-300 words. Include original input verbatim.

- **2.5: Internal Classification (Not User-Facing)** -- Silently evaluate the enriched input (original + discussion decisions) against the same 5-dimension researcher skip criteria from the old Step 2: (1) Model selection with rationale, (2) Tool requirements, (3) Guardrail requirements, (4) Prompt strategy, (5) Context needs. Store researcher decision (RUN or SKIP) for use in Step 5. Do NOT display to user.
