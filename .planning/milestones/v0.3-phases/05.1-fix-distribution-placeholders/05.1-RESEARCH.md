# Phase 05.1: Fix Distribution Placeholders - Research

**Researched:** 2026-03-01
**Domain:** String replacement in markdown command files
**Confidence:** HIGH

## Summary

This phase is a straightforward find-and-replace operation. During Phase 04 (Distribution), command files were created with `OWNER/REPO` placeholder literals in GitHub URLs, with TODO comments noting they should be replaced when the actual repository was created. The `install.sh` script already contains the correct values (`NCrutzen/orqai-agent-pipeline`), but these were never propagated to the command files. The v0.3 milestone audit identified this as a high-severity gap that breaks the entire update flow (Flow D-Update).

The scope is slightly larger than the phase description states. The phase description lists 5 files (update.md, deploy.md, test.md, iterate.md, help.md), but the actual grep reveals **7 files** containing `OWNER/REPO`: update.md (7 occurrences), deploy.md (2), test.md (2), iterate.md (2), set-profile.md (1), SKILL.md (1), and help.md (0 -- help.md actually has no OWNER/REPO occurrences). The planner must account for all 7 files.

**Primary recommendation:** Replace every literal `OWNER/REPO` with `NCrutzen/orqai-agent-pipeline` across all affected files, remove stale TODO comments, and verify resulting URLs resolve to valid GitHub endpoints.

## Common Pitfalls

### Pitfall 1: Incomplete File Coverage
**What goes wrong:** Replacing in the 5 files listed in the phase description but missing `set-profile.md` and `SKILL.md`.
**Why it happens:** Phase description says "5 command files" but the actual grep shows 6 files with occurrences (plus SKILL.md).
**How to avoid:** Use the complete inventory from this research, not the phase description list.
**Warning signs:** Post-replacement grep for `OWNER/REPO` returns any results.

### Pitfall 2: Partial URL Replacement
**What goes wrong:** Replacing the placeholder in curl commands but leaving TODO comments that reference `OWNER/REPO`.
**Why it happens:** TODO comments are in HTML comment blocks, easy to overlook.
**How to avoid:** Remove all three TODO comments in update.md after replacing the URLs they refer to.
**Warning signs:** Grep still finds `OWNER/REPO` in HTML comments.

### Pitfall 3: Breaking Markdown Formatting
**What goes wrong:** Accidentally altering indentation or code block formatting during edits.
**Why it happens:** The URLs appear inside fenced code blocks and inline code within display messages.
**How to avoid:** Use exact string replacement (old string -> new string) rather than regex or line-level edits.
**Warning signs:** Command file display output looks malformed.
