**Banner (verbatim):** `ORQ ► COMPARE FRAMEWORKS`

**Tier note:** deploy+ (runs live invocations + creates/submits experiments to orq.ai).

**Section 1 — Constraints (NEVER/ALWAYS):**
- NEVER run the full experiment before each agent passes the smoke-invocation precheck (XFRM-03).
- NEVER allow different datasets, different evaluator sets, or different models across jobs unless the user explicitly sets `--isolate-model` for the model dimension (XFRM-02).
- ALWAYS emit exactly one evaluatorq job block per agent across all 5 frameworks: orq.ai, LangGraph, CrewAI, OpenAI Agents SDK, Vercel AI SDK (XFRM-01).
- ALWAYS support `--lang python|ts` so users can pick the host language of the generated script (XFRM-01).
- ALWAYS share a single `experiment_id` across all 5 jobs so results surface side-by-side in the orq.ai Experiment UI (XFRM-03).

**"Why these constraints:"** paragraph — fairness is the scientific guarantee; model isolation requires explicit intent; smoke precheck prevents silent bad-runs; side-by-side UI depends on shared experiment_id.

**Section 2 — When to use / When NOT to use:**
- Use: user owns the same conceptual agent implemented across multiple frameworks and wants apples-to-apples quality/cost comparison.
- Don't use: user has only one framework (use `/orq-agent:test` instead) or needs automated framework detection (Phase 37 `/orq-agent:observability`).

**Section 3 — Companion Skills (directional handoffs):**
- ← `/orq-agent:prompt-optimization` (optimized prompt before cross-framework benchmark)
- → `/orq-agent:analytics` for post-experiment cost/latency comparison
- → `/orq-agent:traces` for drill-down on per-framework failures (filter by `--identity <framework>`)

**Section 4 — Done When (falsifiable):**
- [ ] User supplies dataset id, evaluator set, model (or `--isolate-model` with model list), and one invocation adapter per framework
- [ ] Each of the 5 agents passes smoke-invocation precheck (one test call, non-empty response)
- [ ] Fairness checks pass: same dataset across jobs, same evaluator set across jobs, same model across jobs (unless `--isolate-model`)
- [ ] Script emitted in the chosen language (Python or TypeScript) under the user's cwd or configured output path
- [ ] Experiment posted to orq.ai with shared `experiment_id`; Experiment UI link printed

**Section 5 — Destructive Actions:**
- **Create experiment on orq.ai** — AskUserQuestion confirm with a summary of (dataset_id, evaluator set, model(s), 5 framework adapters). No deletions; experiment is immutable.

**Section 6 — Steps (numbered ## Step N:):**

- ## Step 1: Parse args + dispatch
  - Flags: `--lang python|ts` (default: `ts`), `--isolate-model` (bool), `--dataset <id>`, `--evaluators <list>`, `--model <snapshot>`, `--models <list>` (required iff `--isolate-model`).
  - Name of output script: `compare-frameworks-{YYYYMMDD-HHMM}.{py|ts}`.
- ## Step 2: Collect framework adapters
  - Prompt user (AskUserQuestion) for one adapter function per framework. The 5 frameworks MUST appear verbatim in this step's prose: "orq.ai", "LangGraph", "CrewAI", "OpenAI Agents SDK", "Vercel AI SDK".
  - Each adapter = `async (input) => output` shape.
- ## Step 3: Fairness checks (XFRM-02)
  - Assert single `dataset` id across all jobs.
  - Assert identical `evaluators` list across all jobs.
  - Assert identical `model` across all jobs UNLESS `--isolate-model` is set (in which case `models[]` length MUST equal 5).
  - FAIL fast with a clear remediation message on any violation. Emit a "Fairness check failed:" block with the specific mismatch.
- ## Step 4: Smoke-invocation precheck (XFRM-03)
  - For each of the 5 adapters, run ONE test call with a single datapoint from the dataset.
  - Assert non-empty response + no thrown exception.
  - On failure: STOP, surface which framework's adapter failed + raw error. Do NOT proceed to full experiment.
- ## Step 5: Emit evaluatorq script (XFRM-01)
  - Render template from `orq-agent/commands/compare-frameworks/resources/evaluatorq-script-templates.md` (Python + TS scaffolds) substituting the user's dataset / evaluators / model(s) / adapters.
  - TypeScript: `import { experiment } from '@orq-ai/evaluatorq';`
  - Python: `from evaluatorq import experiment`
  - Script MUST contain exactly one `jobs: [...]` entry per framework, keyed by a stable id: `orq-ai`, `langgraph`, `crewai`, `openai-agents-sdk`, `vercel-ai-sdk`.
  - Illustrative model snapshot: `anthropic/claude-sonnet-4-5-20250929` (snapshot-pinned per MSEL-02).
- ## Step 6: Run experiment
  - AskUserQuestion confirm summary → execute the generated script → capture `experiment_id`.
  - MCP-first path: if evaluatorq SDK auto-registers via `ORQ_API_KEY`, no extra call needed; else REST fallback `POST /v2/experiments` with the run artifacts.
- ## Step 7: Link to side-by-side results
  - Print: `https://my.orq.ai/experiments/{experiment_id}` — all 5 jobs share this id; results render side-by-side as columns.
  - Recommend `/orq-agent:analytics --group-by deployment --last 1h` to compare cost/latency.

**Section 7 — Anti-Patterns table (5+ rows):**
| Pattern | Do Instead |
|---------|-----------|
| Different dataset per framework | Use one shared `dataset` id; fairness check blocks otherwise (XFRM-02) |
| Different evaluator sets per framework | Use one shared `evaluators` list; isolating the evaluator dimension is not supported |
| Different models by accident | Use one shared `model`; only `--isolate-model` with `--models` list of length 5 opts in (XFRM-02) |
| Skipping smoke precheck to save time | A silent bad adapter pollutes the whole experiment; precheck is mandatory (XFRM-03) |
| Creating 5 separate experiments | One shared `experiment_id` is what makes side-by-side UI possible (XFRM-03) |
| Using floating model aliases (`-latest`) in the generated script | Snapshot-pin (`claude-sonnet-4-5-20250929`) per MSEL-02 |
