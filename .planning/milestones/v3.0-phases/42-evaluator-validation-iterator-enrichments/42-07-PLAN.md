- Opening: "When judge has TPR < 1.0 or TNR < 1.0, observed pass rate overstates (or understates) true success rate."
- Formula verbatim:
  ```
  theta_hat = (p_observed + TNR − 1) / (TPR + TNR − 1)
  ```
- Derivation (4-6 lines: confusion-matrix logic).
- Worked example: p_observed = 0.80, TPR = 0.95, TNR = 0.92 → theta_hat = (0.80 + 0.92 − 1) / (0.95 + 0.92 − 1) = 0.72 / 0.87 ≈ 0.828 → true rate ~83%.
- Edge case: TPR + TNR ≤ 1 → judge worse than random; do NOT apply correction; warn user.
- Clamping: clamp theta_hat to [0, 1].
- Rendering: quality-report.md new column `Corrected (θ̂)`.

**tpr-tnr-methodology.md** (~150 lines):
- H1: `# TPR / TNR Measurement Methodology (EVLD-06)`
- Confusion matrix with verbatim tokens `TP`, `FN`, `TN`, `FP`:
  ```
                  | Human = Pass | Human = Fail
  Judge = Pass    |     TP       |     FP
  Judge = Fail    |     FN       |     TN
  ```
- Formulas: `TPR = TP / (TP + FN)`, `TNR = TN / (TN + FP)`.
- Minimum sample: ≥30 Pass + ≥30 Fail in held-out test set.
- Workflow: run judge on test split → compare to human labels → populate matrix → compute ratios.
- Validation gate: TPR ≥ 0.90 AND TNR ≥ 0.90 → mark validated.
- Anti-pattern: computing TPR/TNR on train or dev split (leakage).
