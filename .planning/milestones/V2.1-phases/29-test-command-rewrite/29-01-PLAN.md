
Display: "Cleaning previous pipeline outputs..."

**New Step 5: Three-Phase Test Pipeline**

Step 5.1 — Invoke Dataset Preparer:
- Display: `Phase 1/3: Preparing datasets...`
- Read subagent instructions from `orq-agent/agents/dataset-preparer.md`
- Invoke with context: swarm directory path (from Step 3), agent-key filter (if `--agent` specified), mcp_available flag (from Step 2), ORQ_API_KEY (from Step 1.5)

Step 5.2 — Validate dataset-prep.json:
- Check file exists at `{swarm_dir}/dataset-prep.json`
- Check valid JSON via `jq . {swarm_dir}/dataset-prep.json > /dev/null 2>&1`
- Check at least 1 agent with `status: "ready"` via `jq '[.agents | to_entries[] | select(.value.status == "ready")] | length' {swarm_dir}/dataset-prep.json`
- **If any check fails:** Display ABORT message and STOP:
  ```
  ABORT: Dataset preparation failed.

  Expected: {swarm_dir}/dataset-prep.json
  {Specific reason: "File not found" | "Invalid JSON" | "No agents with status ready"}

  The test pipeline stopped before experiment execution.
  Fix the upstream issue and re-run /orq-agent:test.
  ```

Step 5.3 — Invoke Experiment Runner:
- Display: `Phase 2/3: Running experiments...`
- Read subagent instructions from `orq-agent/agents/experiment-runner.md`
- Invoke with context: swarm directory path, agent-key filter (if `--agent` specified), ORQ_API_KEY
- NOTE: Do NOT pass mcp_available -- experiment-runner is REST-only (LOCKED P27 decision)

Step 5.4 — Validate experiment-raw.json:
- Check file exists at `{swarm_dir}/experiment-raw.json`
- Check valid JSON
- Check at least 1 agent with `status: "complete"` or `"partial"` via `jq '[.agents | to_entries[] | select(.value.status == "complete" or .value.status == "partial")] | length' {swarm_dir}/experiment-raw.json`
- **If any check fails:** Display ABORT message and STOP:
  ```
  ABORT: Experiment execution failed.

  Expected: {swarm_dir}/experiment-raw.json
  {Specific reason: "File not found" | "Invalid JSON" | "No agents with status complete"}

  The test pipeline stopped before results analysis.
  Fix the upstream issue and re-run /orq-agent:test.
  ```

Step 5.5 — Invoke Results Analyzer:
- Display: `Phase 3/3: Analyzing results...`
- Read subagent instructions from `orq-agent/agents/results-analyzer.md`
- Invoke with context: swarm directory path (results-analyzer reads both dataset-prep.json and experiment-raw.json from disk)
- NOTE: No agent-key filter needed -- results-analyzer processes all agents present in experiment-raw.json (pre-filtered by upstream steps)

Step 5.6 — Validate test-results.json:
- Check file exists at `{swarm_dir}/test-results.json`
- Check valid JSON
- Check `results.overall_pass` key exists via `jq '.results.overall_pass' {swarm_dir}/test-results.json`
- **If any check fails:** Display ABORT message and STOP:
  ```
  ABORT: Results analysis failed.

  Expected: {swarm_dir}/test-results.json
  {Specific reason: "File not found" | "Invalid JSON" | "Missing results.overall_pass field"}

  The test pipeline stopped before results display.
  Fix the upstream issue and re-run /orq-agent:test.
  ```
