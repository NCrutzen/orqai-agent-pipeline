### 4. Optimistic removal via `pendingRemovalIds` (D-04)

**Hook source:** `web/app/(dashboard)/automations/[swarm]/_shell/selection-context.tsx` lines 90–97:

```typescript
const markPendingRemoval = useCallback((id: string) => {
  setPendingRemovalIds((prev) => {
    if (prev.has(id)) return prev;
    const next = new Set(prev);
    next.add(id);
    return next;
  });
}, []);
```

   `const { markPendingRemoval } = useSelection();`

**Precedent (call-site shape) — `stage-1-widget.tsx:200`:**

No new wiring needed. `recordVerdict` already emits; existing realtime
providers fan it out. The optimistic `markPendingRemoval` hides the row

### Hard-separation guard (RFC lock)

**Source:** `docs/agentic-pipeline/README.md` + `_shell/components/stage-1-widget.tsx:11-17` header comment.

`approvePrediction` operates on the **Stage 1 noise registry only**. It must
not read `swarm_intents` and must not be invoked for Stage 3 rows. The
`activeStage === 1` guard on the footer (D-02b) is the surface-level
enforcement; the wrapper itself stays Stage-1-pure by delegating to
`recordVerdict`, which is already Stage-1-scoped (writes `agent_runs` with
`rule_key` from the noise registry, fires `classifier/verdict.recorded` →
`classifier-verdict-worker` reading `swarm_noise_categories.action`).
