## Decisions Made

- **Seed payment_admittance as a registry row, not a code-side alias.** Plan task 1 + RESEARCH.md Q4 both recommended this; chose `display_order=15` so it sits between `payment` (10) and `auto_reply` (20) without renumbering. Keeps the verdict-worker lookup path branch-free in Wave 2.
- **`outlook_label` for `payment_admittance` matches `payment`** (`'Payment Admittance'`) so both predicted_category values produce the same Outlook side-effect when looked up.
- **Test mock is hand-rolled, not vi.mock'd.** Mirrors `web/lib/classifier/__tests__/cache.test.ts` precedent — programmable result-getters per call let us flip between success and error without re-stubbing the whole client.
