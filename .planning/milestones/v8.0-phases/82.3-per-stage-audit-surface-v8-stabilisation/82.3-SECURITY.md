## Accepted Risks Log

### T-82.3-03 — Cross-tenant screenshot access
- **Accepted:** 2026-05-13
- **Rationale:** MR Automations Toolkit is single-tenant. Any authenticated operator already has full Bulk Review access per existing RLS. No multi-tenancy boundary exists today.
- **Re-evaluate when:** Multi-tenancy is introduced. The signed-URL route would then need per-tenant authorization on the storage path.

### T-82.3-06 — Repudiation of failed mint attempts
- **Accepted:** 2026-05-13
- **Rationale:** Read-only screenshot fetch. `console.error` to Vercel logs is the existing audit channel for this app. A dedicated audit table would add operational cost without proportionate value for a read-only signed-URL mint.
- **Re-evaluate when:** Compliance posture requires append-only audit logs (e.g., SOC2/ISO27001 evidence collection).
