## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /api/automations/audit/signed-url | Authenticated operator session; `path` query is untrusted input |
| route handler → Supabase Storage | Server-side admin client; service-role key never crosses back to client |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-82.3-01 | Tampering | `path` query parameter | mitigate | Regex whitelist `^[a-zA-Z0-9_\-./]+$`, reject `..`, reject leading `/`, length ≤512. Returns 400 `invalid_path`. |
| T-82.3-02 | Information Disclosure | Service-role key | mitigate | Admin client imported from server-only module; never returned in response body. Route file lives under `app/api/` (server-only by Next.js). |
| T-82.3-03 | Information Disclosure | Cross-tenant screenshot access | accept | Bucket scoped to `automation-screenshots`; any authenticated operator already has full Bulk Review access per existing RLS. No multi-tenancy in MR Automations Toolkit today. |
| T-82.3-04 | Elevation of Privilege | Unauthenticated caller mints URL | mitigate | Operator auth check returns 401 before any path validation or storage call. Mirrors existing `/api/automations/*` pattern. |
| T-82.3-05 | Denial of Service | Burst clicks → Supabase rate limit | mitigate | `Cache-Control: private, max-age=200` on response + client-side 200ms cache in `ScreenshotThumb` (Plan 05). Caps to ~5 req/sec per operator per path. |
| T-82.3-06 | Repudiation | Failed mint attempt | accept | `console.error` for storage errors goes to Vercel logs (existing audit channel). No new audit table needed for read-only screenshot fetch. |
