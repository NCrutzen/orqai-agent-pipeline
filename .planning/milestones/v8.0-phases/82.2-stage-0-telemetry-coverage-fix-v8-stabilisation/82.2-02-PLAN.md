    **Q1 — Confirm headline gap (7d):** per-mailbox count of Stage-1 rows in pipeline_events vs matching Stage-0 rows.
    ```sql
    WITH s1 AS (
      SELECT e.mailbox, pe.email_id, pe.swarm_type
      FROM public.pipeline_events pe
      JOIN email_pipeline.emails e ON e.id = pe.email_id
      WHERE pe.stage = 1 AND pe.created_at >= now() - interval '7 days' AND pe.email_id IS NOT NULL
    ),
    s0 AS (
      SELECT email_id, swarm_type FROM public.pipeline_events
      WHERE stage = 0 AND created_at >= now() - interval '8 days' AND email_id IS NOT NULL
    )
    SELECT s1.mailbox, count(*) AS stage1, count(s0.email_id) AS stage0,
           round(100.0 * count(s0.email_id) / nullif(count(*),0), 1) AS pct
    FROM s1 LEFT JOIN s0 USING (email_id, swarm_type)
    GROUP BY 1 ORDER BY 1;
    ```
    Expect: roughly matches the CONTEXT.md evidence table (sales ~91%, debtors 26–45%).

    **Q2 — Gap rows by ingest-route action (the D-A hypothesis):** for the gap rows, what `automation_runs.result->>'action'` did the debtor-email ingest record?
    ```sql
    WITH s1 AS (
      SELECT pe.email_id, pe.swarm_type, e.source_id
      FROM public.pipeline_events pe
      JOIN email_pipeline.emails e ON e.id = pe.email_id
      WHERE pe.stage = 1 AND pe.swarm_type = 'debtor-email'
        AND pe.created_at >= now() - interval '7 days' AND pe.email_id IS NOT NULL
    ),
    s0 AS (
      SELECT email_id FROM public.pipeline_events
      WHERE stage = 0 AND swarm_type = 'debtor-email'
        AND created_at >= now() - interval '8 days' AND email_id IS NOT NULL
    ),
    gap AS (
      SELECT s1.email_id, s1.source_id FROM s1
      LEFT JOIN s0 USING (email_id) WHERE s0.email_id IS NULL
    )
    SELECT
      ar.result->>'action' AS ingest_action,
      ar.result->>'stage' AS ar_stage,
      count(*) AS rows
    FROM gap g
    LEFT JOIN public.automation_runs ar
      ON ar.result->>'message_id' = g.source_id
     AND ar.swarm_type = 'debtor-email'
    GROUP BY 1, 2
    ORDER BY rows DESC;
    ```
    Expect: most gap rows have `ingest_action ∈ {skipped_not_whitelisted, labeled}` (the two D-A target branches). Few/none have `stage_0_safety_pending` (that's the worker-side branch).

    **Q3 — Worker-side override + budget-breach volume (7d):** quantify the rarer D-01 in-scope branches.
    ```sql
    SELECT
      ar.result->>'stage' AS stage,
      ar.status,
      count(*)
    FROM public.automation_runs ar
    WHERE ar.swarm_type = 'debtor-email'
      AND ar.completed_at >= now() - interval '7 days'
      AND (ar.result->>'safety_overridden' = 'true' OR ar.error_message ILIKE '%budget%' OR ar.result->>'stage' = 'stage_0_safety_pending')
    GROUP BY 1, 2 ORDER BY count DESC;
    ```
    Expect: low volume (likely <10 rows); confirms worker-side D-01 fix is real-but-small.

    **Q4 — Suspected-branch volume (7d):** zero rows expected (CONTEXT claim).
    ```sql
    SELECT count(*) FROM public.pipeline_events
    WHERE stage = 0 AND decision = 'injection_suspected'
      AND created_at >= now() - interval '7 days';
    ```

    **Q5 — 30-day backfill candidate volume:** projects the workload for Plan 09.
    ```sql
    WITH s1 AS (
      SELECT DISTINCT email_id, swarm_type FROM public.pipeline_events
      WHERE stage = 1 AND email_id IS NOT NULL
        AND created_at >= now() - interval '30 days'
    ),
    s0 AS (
      SELECT email_id, swarm_type FROM public.pipeline_events
      WHERE stage = 0 AND email_id IS NOT NULL
        AND created_at >= now() - interval '31 days'
    )
    SELECT count(*) FROM s1 LEFT JOIN s0 USING (email_id, swarm_type) WHERE s0.email_id IS NULL;
    ```
    Expect: in the 800–4000 range per RESEARCH §A1 assumption.
