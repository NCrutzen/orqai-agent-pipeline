  <name>Task 2: Verify backfill row signatures + record summary</name>
  <files>(read-only validation; results in SUMMARY)</files>
  <action>
    Use supabase mcp `execute_sql` to verify the backfill wrote sensible rows:

    1. **Total backfilled rows in pipeline_events:**
       ```sql
       SELECT count(*) FROM pipeline_events
       WHERE triggered_by = 'backfill'
         AND decision_details->>'backfill_run_id' = '{run_id from Task 1}';
       ```
       Should match `written` from Task 1.

    2. **Decision breakdown:**
       ```sql
       SELECT decision, count(*)
       FROM pipeline_events
       WHERE triggered_by = 'backfill'
         AND decision_details->>'backfill_run_id' = '{run_id}'
       GROUP BY 1 ORDER BY 2 DESC;
       ```
       Expect: 'safe' dominant; small 'injection_suspected' count (likely 0 per Plan 02 Q4); some 'unknown_legacy' for rows missing automation_runs.

    3. **created_at sanity check** — backfilled rows should have varied timestamps spread across the 30-day window, NOT clustered at the backfill run time:
       ```sql
       SELECT date_trunc('day', created_at) AS day, count(*)
       FROM pipeline_events
       WHERE triggered_by = 'backfill'
         AND decision_details->>'backfill_run_id' = '{run_id}'
       GROUP BY 1 ORDER BY 1;
       ```

    4. **Idempotency proof — re-run the backfill:** trigger again with the same window_days=30. Expect: candidates count drops to ~0 (or to whatever new gap accumulated since the first run); written=0; skipped equals the new candidates. Document the second-run counts.

    Record all four results in the SUMMARY.
  </action>
