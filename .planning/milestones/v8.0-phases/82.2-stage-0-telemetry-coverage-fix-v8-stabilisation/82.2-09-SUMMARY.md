# Phase 82.2 Plan 09: Stage 0 backfill Inngest function Summary

One-liner: One-shot Inngest function `stage0Backfill` reads `stage0_backfill_candidates(30)`, reconstructs missing `pipeline_events.stage=0` rows from `automation_runs.result`, and writes `decision='unknown_legacy'` as honest marker when source data is unavailable.

## What shipped

1. **`web/lib/inngest/functions/stage-0-backfill.ts`** (212 lines) — `stage0Backfill` Inngest function:
   - `id="stage-0/backfill"`, event `pipeline.stage0.backfill`, retries: 1
   - `windowDays` clamp `Math.min(30, Math.max(1, raw ?? 30))` (defense; RPC also clamps)
   - `runId` minted inside `step.run("resolve-run-id", …)` per CLAUDE.md Phase 65 replay-id rule
   - `step.run("find-gaps", …)` calls `admin.rpc("stage0_backfill_candidates", { window_days })`
   - Loop in `BATCH_SIZE=500` chunks; each batch wrapped in `step.run("backfill-batch-${idx}", …)`:
     1. Pre-insert SELECT on `pipeline_events` for the batch's email_ids (defense-in-depth)
     2. Build rows via `buildBackfillRow(candidate, runId)`
     3. Filter out already-covered keys, bulk INSERT direct via `admin.from("pipeline_events").insert(...)` (bypassing `emitPipelineEvent` to honour the historical `created_at`)
   - Returns `{ run_id, candidates_count, written, skipped }`
   - `buildBackfillRow` exported for testability; matches the 4-row table in RESEARCH §Row construction logic

2. **`web/lib/inngest/functions/__tests__/stage-0-backfill.test.ts`** (262 lines) — 8 Vitest cases:
   - T1 `verdict='safe'` → decision='safe' with full provenance
   - T2 `verdict='injection_suspected'` carries forward
   - T3 `stage='stage_0_safety_pending'` → `unknown_legacy` (pending_orphan)
   - T4 no `automation_runs` row → `unknown_legacy` (no_source_record)
   - T5 1200 candidates → 3 INSERTs of 500/500/200
   - T6 idempotency — pre-insert SELECT skips already-covered rows; `skipped` count surfaces correctly
   - T7 replay-safety — `step.run("resolve-run-id", …)` precedes `find-gaps`
   - T8 window clamp — `event.data.window_days=99` passed to RPC clamped to 30

3. **`web/lib/inngest/events.ts`** — added `pipeline.stage0.backfill` typed event with optional `window_days` and `triggeredBy` fields.

4. **`web/app/api/inngest/route.ts`** — imported `stage0Backfill` and added it to the `serve({ functions: [...] })` list adjacent to the other Stage 0 functions.
