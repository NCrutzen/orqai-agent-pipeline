# Phase 82.2 Plan 05: D-06 Synthetic Injection Regression Test Summary

Adds the D-06 worker-level regression test that locks in the post-Plan-04 single-emit invariant for the `injection_suspected` branch of `stage-0-safety-worker.ts`, asserting the full `pipeline_events` row shape (including the new `decision_details.emit_source='main-path'` discriminator) plus the negative-emit invariant (no `classifier/screen.requested`).

## What Was Built

One new `describe("D-06 synthetic injection (Phase 82.2)", ...)` block in `web/lib/inngest/functions/__tests__/stage-0-safety-worker.test.ts` (83 lines added). The test:

1. Mocks `regexScreen` returning `{ matched: "ignore_previous" }`.
2. Mocks `llmInjectionVerdict` returning `verdict='injection_suspected'`, `reason='imperative override detected'`, `matched_span='ignore previous'`, `usage.cost_cents=4`.
3. Mocks budget non-breached.
4. Invokes the worker handler with a minimal `stage-0/email.received` payload (email_id `e-d06-inj`, swarm_type `debtor-email`, `automation_run_id ar-d06`).
5. Asserts:
   - `result.verdict === 'injection_suspected'`
   - Exactly **one** `pipeline_events` row was captured at `stage=0` (single-emit invariant from Plan 04).
   - That row has `swarm_type='debtor-email'`, `stage=0`, `email_id='e-d06-inj'`, `decision='injection_suspected'`, `cost_cents=4`, `triggered_by='pipeline'`.
   - `decision_details` matches `{ emit_source: 'main-path', regex_matched: 'ignore_previous', llm_reason: 'imperative override detected', matched_span: 'ignore previous', safety_overridden: false }`.
   - **NO** `classifier/screen.requested` was sent (injection halts forwarding).
   - An `automation_runs` row was written with `status='predicted'`, `topic='safety_review'`, `result.verdict='injection_suspected'`.

## Why Worker-Level Not E2E

Per RESEARCH §Suspected-branch trigger recommendation, an end-to-end integration test through the `/ingest` route would be redundant — Phase 64 fixtures already exercise the wire-level path from Outlook ingest down to `stage0SafetyWorker`. This test's job is narrower: regression-lock the **post-refactor pipeline_events shape** so a future refactor that silently drops the injection_suspected emit (or the new `emit_source` discriminator) fails CI.

Documented at the head of the describe block.
