# Phase 82.5 Plan 03: Feedback Panel R1+R4+R5 Wiring + W4 Keyboard Coverage Summary

R1 (read-back metadata), R4 (override-note coupling microcopy), and R5 (Save vs Confirm icon + microcopy + soft-confirm dialog) are now wired into `StageFeedbackPanel`. A new `<OthersSaidBlock>` sibling renders the cross-operator notes collapsible. W4 keyboard shortcuts on the focused textarea have automated regression coverage. Panel stays a pure controlled component — parent (Plan 05) owns textarea seeding.

## What Was Built

**Task 1 — `OthersSaidBlock.tsx` (new client component, 146 lines).**
Collapsible cross-operator notes block. Returns `null` when `notes.length === 0` so the panel hides it entirely. Toggle is a `@/components/ui/button` (outline, sm) with text `What others said (N)` / `Hide what others said (N)`. Each note renders `display_name` + ISO-formatted UTC timestamp + verdict pill (color-coded against `--v7-lime` / `--v7-amber` / `--v7-brand-secondary`, matching RowVerdictDot's palette) + `prose_notes` in a `<p>` with `whiteSpace: "pre-wrap"`. **Security:** prose is rendered as text content only — no `dangerouslySetInnerHTML` anywhere (T-82.5.03-01 XSS mitigation, gated by grep).

**Task 2 — `StageFeedbackPanel.tsx` refactor.**
- New optional prop `initialReadBack?: FeedbackReadBack | null` for metadata-only rendering (chip + others block).
- Local `readBack` + `othersOpen` state; sync on `[emailId, stage]` change without ever touching `prose` (controlled-component contract enforced — Plan 05 owns seeding).
- `postFeedback` now follows a successful POST with a GET refresh against `/api/automations/debtor-email/feedback?email_id=…&stage=…` to merge fresh read-back. Refresh failure is non-fatal.
- R5 Save button: transparent background + `--v7-brand-secondary` border + 💾 icon + `Save note · come back later` subtitle. Disabled only when `inFlight !== null` (no longer disabled-when-empty).
- R5 Confirm button: filled `--v7-lime` + ✓ icon + `Confirm · looks right` subtitle. Empty-prose click wraps `postFeedback("confirm")` with `window.confirm("Confirm this stage without writing a note?")` — soft guard per CONTEXT Discretion.
- R4 microcopy `⤓ Override + note save together` rendered under the textarea label in `--v7-amber` + `font-mono` + `--fs-xs`.
- CSS `@media (max-width: 1365px)` collapses `[data-row="primary"]` subtitle on narrow viewports — icons + data-testid identification persist.
- W4 textarea `onKeyDown`: `⌘/Ctrl+s` → preventDefault + `postFeedback("unclear")`; `Esc` → blur + `onAfterConfirm`; `n` / `Enter` → `stopPropagation` so they don't bubble to the document-level shortcut handler while typing. No document-level listeners registered by the panel.
- `<OthersSaidBlock>` mounted below the Save/Confirm row with `notes={readBack?.others ?? []}`.

**Task 3 — `StageFeedbackPanel.keyboard.test.tsx` (new vitest spec, 130 lines).**
Four test cases against the textarea `onKeyDown` contract:
1. `⌘+s` → POST verdict=unclear with correct `{email_id, stage, verdict}` body.
2. `Esc` → `onAfterConfirm` fires once; `onValueChange` is NEVER called with `""` (draft preserved by parent ownership of `value`).
3. `n` → bubble-phase `document` keydown listener is NOT invoked.
4. `Enter` → bubble-phase `document` keydown listener is NOT invoked.

Mocks `global.fetch` to return 200 ok for POST and `{own_latest: null, others: []}` for the GET refresh.

- **Fix:** Refactored the fetch mock to dispatch by method (POST vs GET refresh) via a `findPostCall` helper; stubbed `window.confirm` per test to exercise both accept and cancel paths; replaced the `disables Save when textarea is empty` test with two cases covering empty-save and prose-save (both valid under R5). Added a dedicated R4 microcopy assertion. Result: 8 tests, all passing.

- **Justification:** Plan Task 2 acceptance criteria explicitly allows for this: "vitest snapshot test for the panel still passes (or is updated)". Updating the test is the in-scope path here.
