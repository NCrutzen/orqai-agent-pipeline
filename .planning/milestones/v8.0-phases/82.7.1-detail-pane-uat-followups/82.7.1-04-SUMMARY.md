# Phase 82.7.1 Plan 04: Override-Form Button Polish Summary

One-liner: Stage 1 override-form Submit button switches from amber to brand-secondary lime fill (Confirm rhythm), Discard upgrades to a ghost button with Esc keychip, and the action row gets var(--space-4) top padding under the REG/CAP radio cards — closes E-04.

## Changes

**`web/app/(dashboard)/automations/[swarm]/_shell/components/stage-1-widget.tsx`** (commit `95ca8cf`)

- **D-09 — Submit override (lime fill):** Replaced shadcn `<Button variant="default">` (was `background: var(--v7-brand-primary)`) with a native `<button>` mirroring StageFeedbackPanel's Confirm (L288-316): `background: var(--v7-lime)`, `border: 1px solid var(--v7-lime)`, `borderRadius: var(--v7-radius-pill)`, `color: var(--v7-bg)`, `fontFamily: var(--font-mono)`, `fontWeight: 600`, `fontSize: 12`. Preserved the `⌘⏎` keychip (now styled to match the StageFeedbackPanel pattern: mono, 10px, opacity 0.7, marginLeft 4). Disabled state uses `submitting || !notesValid` (existing semantics).
- **D-10 — Discard changes (ghost):** Replaced shadcn `<Button variant="ghost">` with a native `<button>` mirroring StageFeedbackPanel's Save (L259-286): `background: transparent`, `border: 1px solid var(--v7-brand-secondary)`, `borderRadius: var(--v7-radius-pill)`, `color: var(--v7-brand-secondary)`, mono 12px. Added trailing `<kbd>Esc</kbd>` styled identically to the ⌘⏎ keychip. `onClick={discardOverride}` and `disabled={submitting}` preserved.
- **D-11 — var(--space-4) action-row padding:** Action row container changed from `className="...mt-2"` to `style={{ paddingTop: "var(--space-4)" }}` (mt-2 = ~8px replaced with --space-4 = larger gap, since the row follows radio cards not a textarea).
- **Cleanup:** Removed unused `import { Button } from "@/components/ui/button"` — both Buttons replaced.

### [Rule 1 - Button → native] Replaced shadcn `<Button>` with native `<button>`

- **Found during:** Task 1 implementation
- **Issue:** The D-09/D-10 inline style blocks prescribe `background` / `border` / `borderRadius` / `color` that conflict with shadcn `<Button>` variant defaults (which apply their own background via Tailwind classes). Keeping `<Button variant="default">` with override styles would leave variant classes interfering.
- **Fix:** Replaced both with native `<button type="button">` — matches the StageFeedbackPanel precedent (native button + inline style) exactly.
- **Side effect:** Removed the now-unused `Button` import to keep tsc clean.
- **Commit:** `95ca8cf`
