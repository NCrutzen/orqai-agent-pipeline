<decisions>
## Implementation Decisions

### E-01 — Stage 0 override submit path

- **D-01:** **Per-stage Submit button inside the dirty card** on Stage 0/2/3/4.
  Mirror the per-stage cancel-override link added in 82.7-03 (sibling, same
  row, same styling family). Footer on Stage 1 stays untouched per 82.6 D-02b
  contract. No footer logic changes for Stage 0/2/3/4.
- **D-02:** Per-stage Submit button reuses the same `submitStageOverride`
  callback that the Stage 1 footer Submit invokes today — single code path,
  scoped to the dirty stage. The button label is `Submit override` (singular,
  active stage only) with the existing `⌘⏎` keychip if the form has keyboard
  focus.
- **D-03:** Discard / Cancel affordance on Stage 0/2/3/4 keeps the existing
  per-stage `cancel override` link (no new Discard button needed — the form
  inside the card already has the Escape-keyed Discard from E-04).

### E-02 — Approve loading cue

- **D-04:** **150ms opacity fade-out on the row being pending-removed.** Row
  opacity transitions 1 → 0 over 150ms, then unmounts via the existing
  `pendingRemovalIds` filter in `row-list.tsx`. No button spinner, no
  skeleton. Honors 82.6 D-03 silent UX (no toast).
- **D-05:** Apply same fade to all `pendingRemoval` paths (Approve auto-advance
  AND Submit override post-commit removal) for consistency. No special-casing
  the Approve path.

### E-03 — Brand-color dot affordance

- **D-06:** **Hover tooltip on the brand-color swatch** rendering the brand
  display name (e.g., "Fire Control", "Smeba", "Smeba-Fire"). Uses the existing
  Radix Tooltip pattern already in `predicted-row.tsx` (see the `↻` overridden
  tooltip at `predicted-row.tsx:181-198`). No layout disturbance, no legend
  strip.
- **D-07:** Map from `entity_brand` code → display name lives next to
  `brandColorToken()` in the same module (single-source). Use the brand
  registry's `display_name` if a `swarms.entity_brand` enum metadata table
  exists; otherwise hardcode the 5 Benelux brand display names per
  `project_brand_scope.md` memory.
- **D-08:** Tooltip also applies to the noise-view row layout (the operator
  spotted the dot there). If the noise-view row component is separate from
  `PredictedRow`, mirror the same swatch + tooltip rendering — do NOT introduce
  a second source of truth.

### E-04 — Override form button polish

- **D-09:** **Submit override button fill changes from amber → brand-secondary
  lime** (matches the Confirm / Approve primary precedent from 82.7-01 +
  82.7-02). Amber reads as a warning state, not a commit action. Keep the
  `⌘⏎` keychip on the right edge.
- **D-10:** **Discard changes upgrades to a real ghost button** (transparent
  fill, brand-secondary outline) with the `Esc` keychip as trailing affordance.
  Matches the Save note / Confirm rhythm from StageFeedbackPanel (82.7-01).
- **D-11:** Add `var(--space-4)` top padding above the action row (under the
  REG / CAP radio cards). REG / CAP cards themselves untouched.

### E-05 — Unlabelled rows

- **D-12:** **Investigate both Stage 0 and Stage 1 row-list surfaces** for
  nullable chip rendering. Operator was not sure which tab the missing chip
  was on, so the researcher must scan both `safety-detail-pane`-adjacent
  list components and the noise-view row component.
- **D-13:** Treat root-cause investigation as a research deliverable. Three
  possible roots (per CONTEXT v1):
  - (a) Renderer bug — chip element not drawn when category is `null` or a
    specific value.
  - (b) Data bug — row has no Stage-1 classification at all (Stage 1 didn't
    run or didn't write a verdict).
  - (c) Pre-cutover historical row legitimately has no predictor attribution
    and the placeholder logic is missing.
- **D-14:** Renderer-side fixes (a, c) go in this phase. If root cause is (b)
  data-side and affects historical rows requiring backfill, scope the
  backfill into a separate phase (82.7.2 or noted on the v8.0 punch list);
  ship the renderer fix here as the empty-state placeholder.
- **D-15:** **Visual contract for unknown/missing chip:** show a muted
  placeholder chip with `—` (em-dash) text and `var(--v7-muted)` outline.
  Never render zero chip — the column always has a chip-shaped element to
  preserve row rhythm.

### Claude's Discretion

- Exact fade-out easing curve for D-04 (linear vs ease-out) — Claude picks.
- Exact tooltip delay timing for D-06 (e.g., 300ms hover delay) — Claude
  picks; match existing Radix Tooltip defaults if no project pattern.
- Whether to extract a shared `StageOverrideCard` component to host both the
  Stage-1 footer form and the new per-stage form (DRY decision) — planner
  picks based on how much logic actually duplicates.

</decisions>
