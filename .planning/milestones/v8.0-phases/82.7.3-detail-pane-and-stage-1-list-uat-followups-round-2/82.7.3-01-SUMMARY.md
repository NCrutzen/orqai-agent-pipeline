# Phase 82.7.3 Plan 01: Single Submit/Cancel surface + plain-English eval-type cards Summary

**One-liner:** Stage 1 dirty-branch render now exposes exactly one Submit/Cancel pair (the detail-pane footer) and the eval-type radio uses operator-readable copy ("Recent regression" / "New case") instead of REG/CAP pills.

## What changed

### Task 1 — `stage-step.tsx` (H-01 part 1)
Deleted the inline `<div className="flex items-center justify-end gap-2 mt-1">` action-row block (L253-309 from 82.7.2-01) containing the `stage-${stage.n}-cancel-override` and `stage-${stage.n}-submit-override` buttons. The `Note and override are saved together` muted hint above the deleted block is preserved. `onSubmitDirty` / `onCancelDirty` props remain on `StageStepProps` — unused in this component but kept on the type signature per CONTEXT H-01.

Commit: `c104577`

### Task 2 — `stage-1-widget.tsx` + `stage-1-widget.test.tsx` (H-01 part 2 + BLOCK-02)
Deleted the `<div className="flex items-center justify-end gap-2" style={{paddingTop: "var(--space-4)"}}>` action-row container (L355-422) containing the `Discard changes` ghost button and the lime-pill `Submit override` button. The `Esc` / `⌘⏎` keyboard listeners on `window` stay wired to `discardOverride` / `onClickSubmit` via the `bulk-review:override-discard` / `bulk-review:override-submit` custom events (lines 274-285) — keyboard paths survive intact.

Test re-target: the two `findByRole("button", { name: /submit override/i })` queries at L235-239 and L302-305 are replaced with the canonical submit path — `fireEvent.keyDown(window, { key: "Enter", metaKey: true })` plus `window.dispatchEvent(new Event("bulk-review:override-submit"))`. The widget listens for the custom event directly (independent of `_shell/keyboard-shortcuts.tsx` which is not mounted in the harness). All four test cases pass after the re-target.

Commit: `2005840`

### Task 3 — `eval-type-radio.tsx` (H-02)
Rewrote the two card render blocks: deleted the red `reg` / lime `cap` pill `<span>` markup and the `isReg` branch; `pillClass` prop removed from the `Card` sub-component (replaced with `title`). Cards now render:

- option `"regression"` → title **Recent regression**, subhead **Used to work correctly**
- option `"capability"` → title **New case**, subhead **Never handled correctly before**

Selection feedback still uses the existing `checked` border swap (`var(--v7-brand-primary)` outline + `var(--v7-brand-primary-soft)` background). `RadioGroup`, all `aria-*` attributes, the `c` / `g` keyboard shortcuts (routed via `_shell/keyboard-shortcuts.tsx:225-226`), and the API payload (`eval_type: "capability" | "regression"`) are unchanged.

Commit: `d58ff75`
