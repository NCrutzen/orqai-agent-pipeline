# Phase 82.7.3 Plan 03: Wider detail-pane + per-row mailbox chip Summary

Widened the split-pane parent grid from `minmax(0,1fr) 460px` to
`minmax(640px,1fr) 540px` across all 5 stage shells (G-03) and added a
small mono pill chip in front of the sender text in `RowList` so
operators can identify a row's mailbox without opening the detail pane
(G-04). All edits are file-disjoint from Wave 1 — `_shell/detail-pane.tsx`
was not touched, so the canonical-footer state (G-01) and `var(--v7-lime)`
Submit-button colour remain intact.

## What Changed

### Task 1 — `RowList` (presentation primitive)

`web/app/(dashboard)/automations/[swarm]/_shell/row-list.tsx`:

- `RowListProps` gains optional `mailboxLabels?: Record<number, string>`
  field with JSDoc explaining the H-04 origin and graceful-degrade
  contract. Existing call-sites that omit it compile unchanged.
- Sender-column render restructured: outer `<span data-testid="row-sender">`
  is now `display:flex, align:center, gap:6, minWidth:0`. A chip span
  (with `data-testid="row-mailbox-chip"`) is conditionally prepended
  guarded by `mailboxLabels && r.mailbox_id != null && mailboxLabels[r.mailbox_id]`.
  The original sender text is wrapped in an inner `<span>` that retains
  `overflow:hidden; text-overflow:ellipsis; white-space:nowrap; min-width:0`
  so it remains the truncate target.
- Chip styling uses only existing brand tokens:
  `var(--v7-panel-2)` background, `var(--v7-text-muted)` colour,
  `var(--v7-line)` border, `var(--v7-radius-pill)` radius,
  `var(--font-mono)` font. No hex literals, no new tokens.
- Chip is `flexShrink: 0` so long sender strings ellipses first.
- `feedbackMap`, `rightEdgeSlot`, `emptyState` props unchanged.
- `StageBadge` primitive at L201-226 not touched.
