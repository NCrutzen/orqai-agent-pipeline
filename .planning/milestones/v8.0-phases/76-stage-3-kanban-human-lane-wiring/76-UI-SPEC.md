## Spacing Scale

Locked by sketch findings: 4px-increment scale (CSS vars `--space-1` … `--space-6`). Multiples of 4 only — no exceptions.

| Token | Value | Usage |
|-------|-------|-------|
| --space-1 | 4px | Pill inner padding, icon gaps |
| --space-2 | 8px | Action stack gaps, inline editor row gaps, triage row vertical padding |
| --space-3 | 12px | Pane internal section gap, chip-strip vertical |
| --space-4 | 16px | Triage row horizontal padding, pane horizontal padding, page-header vertical |
| --space-5 | 20px | Page-header / chip-strip horizontal padding |
| --space-6 | 24px | Major section breaks within pane |

Triage row vertical rhythm aligns to the 8px grid (`--space-2`). The 13px body line at line-height 1.45 (~19px) plus `--space-2` top + bottom yields a row that lands within the sketch-006 visual lock once line-height is included.

Detail pane fixed width: `460px` (sketch 006 lock — multiple of 4). Below 1280px viewport, the pane collapses to a slide-over (planning task — keep contract above the desktop breakpoint).

**Triage row CSS pattern:**

```css
.row {
  padding: var(--space-2) var(--space-4); /* 8px 16px — row vertical rhythm aligned to 8px grid */
}
.row.is-selected {
  border-left: 2px solid var(--v7-brand-primary);
  padding-left: calc(var(--space-4) - 2px); /* selected variant compensates the 2px brand-primary left-border via calc — keeps the text optical position aligned with non-selected rows */
}
```

The `calc()` compensation is a CSS implementation detail computed at render time, not a declared spacing token, so it does not introduce a non-grid value into the contract.

V7 tokens. Sizes locked by sketch findings; total = 4 sizes, 2 weights — within ui-checker limits.

| Role | Size | Weight | Line Height |

| Body / row text | 13px (`--fs-sm`) | 400 (regular) | 1.45 |

## Color

V7 dark theme is the operator default. 60/30/10 split locked from production tokens.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--v7-bg)` `#0c1117` | Page background, list rail |
| Secondary (30%) | `var(--v7-bg-2)` `#101722` (and panel `#151d29`) | Tab strip, page header, detail pane, inline-editor surface |
| Accent (10%) | `var(--v7-brand-primary)` `#ff6a34` | Selected-tab underline · selected-row left-border (2px) · primary CTA (`Replay through Stage 4` confirm) · active chip background-soft + border · tab badge text/soft-bg |
| Destructive | `var(--v7-red)` `#ff6b7a` | `handler_error` reason pill, error_detail expanding section text |

Accent reserved for (explicit list — anything else MUST use `--text` / `--text-muted` / `--border`):
1. The active stage tab (bottom border + foreground text)
2. The selected row's 2px left border + the selected chip's background-soft + border
3. The primary CTA in the action stack (`Replay through Stage 4` — same-intent re-fire OR confirm-replay button in inline editor)
4. Tab badge non-zero counts (background-soft pill, brand-primary foreground)
5. Inline-editor outline (1px border on the editor surface during edit mode)
6. Confidence-bar high-end fill (when `value >= 0.66` — uses `--v7-success` green; brand-primary is NOT used for confidence)

Semantic categorical (NOT counted against accent budget — these are reason/state signals, used at low saturation):
- `--v7-blue` + `--v7-blue-soft` → `no_handler` reason pill, mid-range confidence bar
- `--v7-amber` + `--v7-amber-soft` → `low_conf` reason pill, low-end confidence bar fill
- `--v7-red` + `--v7-red-soft` → `handler_error` reason pill
- Neutral `rgba(255,255,255,0.06)` on dark → intent pill background (intent text is `--text-muted`)

Confidence bar value-to-color map (40px wide, 4px tall, only rendered on low_confidence rows — never on no_handler):
- `value < 0.40` → amber fill
- `0.40 ≤ value < 0.66` → blue fill
- `value ≥ 0.66` → green (`--v7-success`) fill

### Destructive confirmation

Per sketch 007 lock: NO confirm modals on Close, Replay, or Reclassify in v1. The misclick mitigation is a future undo-toast (deferred). Destructive UI surface for this phase is **the inline editor itself** — the operator must explicitly pick an intent / noise key and click Confirm; the click on Replay or Reclassify alone does nothing irreversible.

| Action | Confirmation pattern |
|--------|----------------------|
| Replay (same intent) | One click → fires. No modal. Optimistic removal from list. |
| Replay (different intent) | Inline editor opens → operator picks intent → click Confirm replay. |
| Reclassify as noise | Inline editor opens → operator picks noise key → click Confirm reclassify. |
| Close (manual) | One click → fires. No modal. Optimistic removal from list. |

## Accessibility Contract

- **Keyboard navigation:** ↑ ↓ moves selection; ⏎ confirms primary action (or active inline-editor); Esc cancels inline-editor; `N` opens Reclassify-as-noise editor; Space fires Close. Match `web/app/(dashboard)/automations/[swarm]/review/keyboard-shortcuts.tsx` exactly so operators learn one shortcut set.
- **Focus ring:** primary outline uses `var(--v7-brand-primary)` at 2px, 2px offset. Never disable focus rings on actionable elements (buttons, chips, dropdowns).
- **Color is never the sole signal:** every reason pill carries text (`no_handler`, `low_conf`, `handler_error`) plus color. The selected-row state uses both background AND a 2px left border (not color alone).
- **Contrast minimums:** body text on `--v7-bg` and `--v7-bg-2` ≥ WCAG AA (4.5:1). V7 tokens already satisfy this — do not introduce raw hex outside the V7 namespace.
- **Tab semantics:** `<nav>` containing `<a>` elements with `aria-current="page"` on the active tab (already used in the review surface).
