# Plan 52-03: Sidebar Smart Filter Chips with URL State

## Goal
Add 3 smart filter chips ("Only blocked", "Needs review", "High SLA risk") to the sidebar, conditionally rendered when on `/swarm/*`. Clicking a chip toggles `?filter=<key>` in the URL via `router.replace`. The Kanban board (Plan 52-02) already reads this param via `useSearchParams`.

## Tasks

### Task 1: SidebarSmartFilters component
- Create `web/components/v7/sidebar-smart-filters.tsx`:
  - `"use client"`
  - Imports: `useRouter`, `useSearchParams`, `usePathname` from `next/navigation`; `SMART_FILTERS` from `@/lib/v7/kanban/filters`
  - Conditional render: `if (!pathname?.startsWith("/swarm/")) return null`
  - Layout: outer `<div className="flex flex-col gap-3">` with group label "SMART FILTERS" + chip stack
  - Each chip is a button with:
    - Pill border-radius (`--v7-radius-pill`), 44px height, 14px horizontal padding
    - `flex items-center justify-between` for label + arrow/check
    - Idle: `--v7-line` border, transparent bg, `--v7-text` text, `竊蛋 arrow
    - Active (`active === f.key`): teal-soft bg + 3px teal left-border accent + teal text + `笨伝
    - Hover: subtle bg lift
  - Toggle behavior: clicking active chip clears the filter (passes `null` to setFilter)
  - `setFilter` writes `?filter=<key>` (or removes the key) via `router.replace(\`\${pathname}?\${searchParams}\`, { scroll: false })`
