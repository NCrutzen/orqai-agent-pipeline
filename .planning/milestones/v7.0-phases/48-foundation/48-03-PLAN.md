<task type="auto">
  <name>Task 2: Add project_members gate to dashboard layout</name>
  <files>web/app/(dashboard)/layout.tsx</files>
  <action>
1. Update `web/app/(dashboard)/layout.tsx`:
   - After the existing `if (!user) redirect("/login")` check, add a project_members query:
   ```typescript
   const { count } = await supabase
     .from("project_members")
     .select("*", { count: "exact", head: true })
     .eq("user_id", user.id);
   
   if (count === 0) {
     redirect("/access-pending");
   }
   ```
   - This query uses `head: true` for efficiency (no row data, just count)
   - The redirect goes to `/access-pending` which is outside the (dashboard) layout group, so it won't trigger infinite redirect

IMPORTANT: The `project_members` table already exists from Phase 34. This is a read-only query.

Edge case: If a user with an existing email/password account logs in via Microsoft SSO, Supabase OAuth auto-links the identities because emails match. The user retains their existing project_members rows. No special handling needed -- AUTH-02 is handled by Supabase Auth's built-in OAuth identity linking when the provider is OAuth (not SAML).
