Create `orq-agent/templates/agent-spec.md` containing:

1. **Legend** at the top: table mapping each `{{PLACEHOLDER}}` to the corresponding Orq.ai API field and which reference file documents it. Example: `{{AGENT_KEY}}` -> `key` field -> see `references/orqai-agent-fields.md`
2. **Instructions line**: "Replace each `{{PLACEHOLDER}}` with the appropriate value from the architect blueprint and reference files."
3. **Template sections** mapping 1:1 to Orq.ai fields:
   - `# {{AGENT_KEY}}` (heading = agent key)
   - `## Configuration` section: key, role, description (brief purpose for Orq.ai Configuration description field)
   - `## Model` section: `{{MODEL}}` in provider/model-name format, `{{FALLBACK_MODELS}}` ordered list
   - `## Instructions` section: `{{INSTRUCTIONS}}` (full system prompt content)
   - `## Tools` section: subsections for Built-in Tools, Function Tools (with JSON Schema), HTTP Tools, Python Tools, Agent Tools (sub-agents as tools). Each with `{{TOOLS_BUILTIN}}`, `{{TOOLS_FUNCTION}}`, etc.
   - `## Context` section: `{{CONTEXT}}` — what context the agent needs
   - `## Evaluators` section: `{{EVALUATORS}}` — recommended evaluator config
   - `## Guardrails` section: `{{GUARDRAILS}}` — recommended guardrails
   - `## Runtime Constraints` section: `{{MAX_ITERATIONS}}`, `{{MAX_EXECUTION_TIME}}`
   - `## Input/Output Templates` section: `{{INPUT_TEMPLATE}}`, `{{OUTPUT_TEMPLATE}}` with `{{variables}}` syntax
4. Sections that are not applicable for a given agent should include a note: "Not applicable for this agent — omit when configuring in Orq.ai Studio."

Create `orq-agent/templates/orchestration.md` containing:

1. **Legend** at top mapping placeholders to reference fields
2. **Template sections**:
   - `# {{SWARM_NAME}} — Orchestration` (heading)
   - `## Overview`: `{{PATTERN}}` (single/sequential/parallel), `{{AGENT_COUNT}}`, `{{COMPLEXITY_JUSTIFICATION}}`
   - `## Agents`: For each agent: key, role, brief responsibility
   - `## Agent-as-Tool Assignments`: Which sub-agents are tools of which parent agents. `{{TOOL_ASSIGNMENTS}}`
   - `## Data Flow`: What information passes between agents. `{{DATA_FLOW}}` with diagram
   - `## Error Handling`: What happens on sub-agent failure/timeout. `{{ERROR_HANDLING}}`
   - `## Human-in-the-Loop`: Decision points where human approval is needed. `{{HITL_POINTS}}`
   - `## Setup Steps`: Numbered steps for configuring in Orq.ai Studio (create agents in order, assign tools, test)

Use `{{PLACEHOLDER}}` format consistently (double curly braces, ALL_CAPS). This matches Orq.ai's own variable syntax.
