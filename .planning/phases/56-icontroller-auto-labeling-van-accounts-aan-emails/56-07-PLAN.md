**A) `labeling-flip-cron.ts` — implement** (the skeleton from 56-00 should already have most of this; verify and extend per 56-RESEARCH.md lines 650-727):

Make sure the file:
- Has cron string `TZ=Europe/Amsterdam 0 6 * * 1-5` in a `//` comment header (NOT inside `/** */`).
- Constants `FLIP_N_MIN = 50`, `FLIP_CI_LO_MIN = 0.95`, `DEMOTE_CI_LO_MAX = 0.92`.
- Imports `wilsonCiLower` ONLY from `@/lib/classifier/wilson` (verify with `grep "shouldPromote\\|shouldDemote" labeling-flip-cron.ts` → must be 0).
- Exports `evaluateMailbox(admin, mailbox, mutate)` and `pickAction(args)` at module scope (so tests can import without the Inngest harness).
- `evaluateMailbox` queries `agent_runs` filtered by:
  - `swarm_type = 'debtor-email-labeling'`
  - `human_verdict IS NOT NULL`
  - `context->>'icontroller_mailbox_id' = String(mailbox.icontroller_mailbox_id)`
  - ORDER BY verdict_set_at DESC LIMIT 50
- Computes `n = rows.length`, `agree = rows.filter(r => r.human_verdict === 'approve').length`, `ci_lo = wilsonCiLower(n, agree)`
- Inserts into `classifier_rule_evaluations` with `swarm_type='debtor-email-labeling'`, `rule_key=\`mailbox_flip:${mailbox.source_mailbox}\``, `n`, `agree`, `ci_lo`, `action`
- When `mutate=true`:
  - if `mailbox.dry_run && n >= FLIP_N_MIN && ci_lo >= FLIP_CI_LO_MIN` → update `debtor.labeling_settings` set `dry_run=false` where `source_mailbox = mailbox.source_mailbox`
  - else if `!mailbox.dry_run && ci_lo < DEMOTE_CI_LO_MAX` → update `dry_run=true` AND `console.warn('[labeling-flip-cron] demote', { mailbox: mailbox.source_mailbox, n, ci_lo })`
- `pickAction({n, ci_lo, dry_run, mutate})`:
  ```typescript
  const wouldPromote = dry_run && n >= FLIP_N_MIN && ci_lo >= FLIP_CI_LO_MIN;
  const wouldDemote = !dry_run && ci_lo < DEMOTE_CI_LO_MAX;
  if (mutate) return wouldPromote ? "promoted" : wouldDemote ? "demoted" : "no_change";
  return wouldPromote ? "shadow_would_promote" : wouldDemote ? "shadow_would_demote" : "no_change";
  ```
