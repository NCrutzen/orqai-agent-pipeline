# Phase 82.4 Plan 05: Option Z Per-Stage List Loader Summary

Implements `loadStageFeedbackList` — a server-side, paginated, per-stage list
loader that powers the re-scoped Stage 0–3 tabs (Option Z: every email with a
verdict at that stage, not just stuck rows). Plan 06 will wire each stage page
to it; Stage 4 / Kanban human-lane continues to use the existing
`kanban-loader.ts` (unchanged).

## What Shipped

- `feedback-list-loader.ts` (292 lines) exporting `loadStageFeedbackList`,
  `FeedbackListRow`, `FeedbackListParams`, `FeedbackListPage`.
- Test file with 8 vitest cases (7 spec'd + 1 sanity).
- Three-query merge:
  1. `pipeline_events` filtered by `(swarm_type, stage)` + optional
     `.in("decision", NEEDS_ACTION_DECISIONS)` + optional `.lt("created_at", before)`,
     ordered DESC by `created_at`, limit `limit*2`.
  2. `email_pipeline.emails` cross-schema join for subject/sender/received_at/mailbox_id.
  3. `email_feedback` join on `(email_id, stage)` (+ optional `operator_id`),
     first-write-wins for latest verdict per email.
- Sort buckets: `needs_action` first, `auto_handled` second, `own_reviewed`
  third — within bucket `received_at desc`.
- Cursor pagination uses `pipeline_events.created_at` for both the `.lt` filter
  and the returned `nextBefore` — same column both ways prevents page drift.
- Page size default 50, capped at 100.

## Decisions Made

**Closed needs-action decision set.** Captured at the top of the loader as a
`readonly string[]`:

```ts
const NEEDS_ACTION_DECISIONS = [
  "needs_review",
  "unknown",            // Stage 1 closed-list fall-through
  "low_confidence",     // Stage 3 Kanban reason
  "injection_suspected",// Stage 0 (Phase 82.2 backfill)
  "unknown_legacy",     // Stage 0 backfill marker
] as const;
```

Sources verified inline in the file's JSDoc:
- `web/lib/inngest/functions/stage-0-backfill.ts:63, 200` (stage 0 vocab)
- `docs/agentic-pipeline/stage-1-regex.md` (stage 1 `unknown` token)
- `web/app/(dashboard)/automations/[swarm]/stage-3/page.tsx:101` (`low_confidence`)

**Defensive mineOnly.** When `mineOnly=true` and `operatorId` is missing, the
loader returns `{rows:[], nextBefore:null}` rather than throwing or leaking
cross-operator history. V10.0 multi-operator RLS will replace this.

**Cursor column = pipeline_events.created_at.** Test 7 locks this in: the
returned `nextBefore` MUST equal the last surfaced row's
`pipeline_events.created_at`, NOT the joined `email_pipeline.emails.received_at`.
Using different columns for filter vs cursor causes skipped/duplicate rows
across pages. Internal `_pe_created_at` field carries the value through the
merge and is stripped from the public `FeedbackListRow` shape.

**Fetch `limit*2` from pipeline_events.** Post-merge we may drop rows whose
email row was deleted upstream; over-fetching gives the bucket-sort headroom
before truncation to `limit`.

## Hard-Separation Lock (RFC docs/agentic-pipeline/README.md)

The loader queries `pipeline_events` filtered by an explicit `stage` parameter.
Each tab (Stage 0/1/2/3) gets its own loader call with its own stage value —
Stage 1 (noise filter, `swarm_noise_categories`) and Stage 3 (intent classifier,
`swarm_intents`) rows are never mixed by this module. Caller-side hard
separation is preserved.
