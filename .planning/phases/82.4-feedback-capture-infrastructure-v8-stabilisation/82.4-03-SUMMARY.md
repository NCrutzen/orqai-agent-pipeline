# Phase 82.4 Plan 03: StageFeedbackPanel — operator-facing capture surface

Prose-notes textarea + Save button + ✓ Confirm chip mounted inside each Stage 0–3 audit expander. Wires to POST `/api/automations/debtor-email/feedback`. Confirm auto-collapses the expander for operator momentum.

## What Shipped

**Component:** `web/components/automations/bulk-review/audit/StageFeedbackPanel.tsx`

- `"use client"` React component, 205 lines
- Props: `{ stage: 0|1|2|3, emailId: string, onAfterConfirm?: () => void }`
- Textarea (4000 char max, vertical resize), Save button, ✓ Confirm chip
- POST body: `{ email_id, stage, verdict: 'unclear'|'confirm', prose_notes? }`
- In-flight: both buttons disabled, label flips to "Saving…"
- 2xx: textarea cleared. On confirm: `onAfterConfirm()` fires.
- Non-2xx: inline error rendered with `role="alert"`
- data-testid `stage-feedback-panel-{stage}` on root
- 12 token references to `var(--v7-*)` — zero Tailwind colour classes

**Tests:** `web/components/automations/bulk-review/audit/__tests__/StageFeedbackPanel.test.tsx`

**Mount point:** `web/app/(dashboard)/automations/[swarm]/stage-1/components/stage-step.tsx`
