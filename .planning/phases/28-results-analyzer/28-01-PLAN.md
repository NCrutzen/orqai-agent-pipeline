  <files>orq-agent/agents/results-analyzer.md</files>
  <action>
Create `orq-agent/agents/results-analyzer.md` following the established subagent pattern (YAML frontmatter, files_to_read, numbered phases). This is a pure computation subagent -- no Orq.ai API calls.

**YAML Frontmatter:**
```yaml
---
name: orq-results-analyzer
description: Reads experiment-raw.json, computes statistical aggregation, determines pass/fail, writes test-results.json and test-results.md
tools: Read, Bash, Glob, Grep
model: inherit
---
```

**files_to_read:** `orq-agent/templates/test-results.json`

**Phase 1: Read Inputs**
- Read `experiment-raw.json` from swarm output directory (or `experiment-raw-holdout.json` for holdout mode)
- Read `dataset-prep.json` from same directory for dataset metadata (split counts, dataset IDs, swarm_name)
- Validate: each agent must have `status: "complete"` and `successful_runs >= 1`
- If experiment-raw.json not found: STOP with clear error message directing user to run experiment-runner first
- Accept optional parameters: `verbose` (boolean, default false), `holdout` (boolean, default false)

**Phase 2: Compute Triple-Run Aggregation (ANLZ-01)**
For each agent, for each evaluator:
- Collect the `aggregate` score from each run (3 values from `runs[].scores.{evaluator}.aggregate`)
- Sort scores and take median: `sorted[1]` (middle of 3)
- Compute mean: `(s1 + s2 + s3) / 3`
- Compute sample variance: `sum((x - mean)^2) / (n-1)` where n=3, so divide by 2
- Compute standard deviation: `sqrt(variance)`
- Compute 95% CI using Student's t-distribution: t-critical = 4.303 for df=2
  - margin = `t_crit * (stddev / sqrt(n))` = `4.303 * (stddev / sqrt(3))`
  - CI = `[median - margin, median + margin]`
  - Clamp to scale bounds: binary/continuous-01 → [0, 1]; continuous-1-5 → [1, 5]
- **IMPORTANT:** Variance uses mean for computation even though median is the reported central tendency. CI centers on median (per user decision).

**Phase 3: Determine Pass/Fail (ANLZ-02)**
- Role-based thresholds (LOCKED -- no override, no config):
  - structural: 0.8
  - conversational: 0.7
  - hybrid: 0.75
- Read agent role from experiment-raw.json `agents.{key}.role`
- Per evaluator: `pass = (median >= threshold)`
- Per agent: `agent_pass = ALL evaluators pass`
- Overall: `overall_pass = ALL agents pass` (strict rule)
- Count total_failure_count per agent: number of per-example scores (across all runs, any evaluator) below threshold

**Phase 4: Category-Sliced Scoring (ANLZ-03)**
- Check if per_example entries in experiment-raw.json have category metadata
  - Source: category comes from dataset rows -- check if `inputs.category` or `category` field exists in per_example entries
- If NO examples have category: set `category_scores` to empty object `{}`
- If SOME examples have category: slice by category, compute per-evaluator median within each category slice
  - Include `count` field per category so downstream knows sample size
  - Per-category pass/fail using same role-based threshold
  - When coverage is incomplete, note in summary: "Category breakdown covers X/Y examples (Z without category metadata)"
- Category taxonomy: happy-path, variation, boundary, adversarial, edge-case, stress (from dataset-generator.md)

**Phase 5: Identify Worst Cases**
- Per agent: find top 3 worst-performing examples
- "Worst" = lowest minimum score across all evaluators per example (bottleneck ranking)
- **Scale normalization for ranking:** Binary/continuous-01 scores used as-is; continuous-1-5 scores normalized to 0-1 via `(score - 1) / 4` for comparison only (reported scores remain in original scale)
- Each worst_case entry includes:
  - `eval_id`: from per_example data
  - `input`: from per_example `input` field
  - `expected_output`: from per_example data if available, otherwise "N/A" (handle gracefully -- experiment-raw.json may not include expected_output)
  - `actual_output`: mapped from per_example `output` field (note: experiment-runner uses `output`, test-results.json uses `actual_output` -- results-analyzer maps between these)
  - `scores`: per-evaluator scores for this example
  - `category`: from per_example data if available
  - `reason`: derived from scores: "Failed {evaluator} ({score} < {threshold})" for each failing evaluator

**Phase 6: Write test-results.json (ANLZ-04)**
- Write test-results.json matching `orq-agent/templates/test-results.json` EXACTLY
- **CRITICAL:** Hardener.md reads specific fields. Every field must be present:
  - `test_run_id`: generated UUID or timestamp
  - `swarm_name`: from dataset-prep.json
  - `tested_at`: ISO timestamp
  - `dataset` section: populated from dataset-prep.json (total_examples, split counts, per_agent_datasets with IDs)
  - `evaluators[]`: top-level array with name, type, threshold, scale, orqai_evaluator_id (from experiment-raw.json evaluators_used)
  - `results.overall_pass`: boolean
  - `results.per_agent[]`: each with agent_key, role, evaluators_used, scores (per-evaluator: median, variance, confidence_interval, pass, threshold, scale, runs), category_scores, worst_cases, total_failure_count
  - `summary`: "X/Y agents passing. {guidance}" where guidance is "All agents ready for /orq-agent:harden." or "N agents failing -- run /orq-agent:iterate to improve scores."
- Use jq or structured JSON output to ensure valid JSON (no manual string concatenation for nested objects)

**Phase 7: Write test-results.md (ANLZ-05 part 1)**
- Full markdown report with sections:
  - Header with swarm name, test date, overall PASS/FAIL status
  - Per-agent section containing:
    - Evaluator scores table (evaluator | median | variance | 95% CI | pass/fail | threshold)
    - Category breakdown table IF categories are available (category | evaluator | median | pass/fail | count)
    - Top 3 worst cases with input text, actual output excerpt, and failure reason
  - Summary section with:
    - Overall pass/fail counts
    - Actionable next-step guidance: "All passing -> ready for /orq-agent:harden" or "N agents failing -> run /orq-agent:iterate"
- Write to swarm output directory as `test-results.md`

**Phase 8: Terminal Summary (ANLZ-05 part 2)**
- Print terminal summary table (always, regardless of verbose flag)
- **Compact mode (default):** One row per agent
  - Columns: agent_key | role | bottleneck_score | PASS/FAIL
  - Bottleneck score = lowest evaluator median per agent (normalized to 0-1 using `(score - 1) / 4` for 1-5 scales)
  - Overall summary line at bottom: "X/Y agents passing"
- **Verbose mode (when verbose parameter is true):** Adds per-evaluator median scores per agent
  - Additional columns for each evaluator's median
- **Category breakdown is test-results.md ONLY -- never in terminal** (LOCKED decision)

**Anti-patterns to avoid:**
- Do NOT use mean instead of median as central tendency
- Do NOT use z-distribution (1.96) -- use Student's t (4.303 for df=2)
- Do NOT use population variance (divide by n) -- use sample variance (divide by n-1)
- Do NOT include category breakdown in terminal output
- Do NOT compare raw scores across different scales without normalization for worst-case ranking
- Do NOT make any Orq.ai API calls -- this is a pure computation subagent
