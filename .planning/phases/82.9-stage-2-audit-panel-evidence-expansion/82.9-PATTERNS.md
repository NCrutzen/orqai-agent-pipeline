```ts
function asCandidates(v: unknown): Array<{ id: string; name: string; contact_person: string | null; recent_invoices: string[] }> {
  if (!Array.isArray(v)) return [];
  const out = [];
  for (const c of v) {
    if (!isRecord(c)) continue;
    const id = asString(c.id);
    const name = asString(c.name);
    if (id === null || name === null) continue;
    out.push({
      id, name,
      contact_person: asString(c.contact_person),
      recent_invoices: asStringArray(c.recent_invoices).slice(0, 5), // defense-in-depth
    });
  }
  return out;
}
```

## Shared Patterns

### JSONB ŌåÆ typed payload narrowing

**Source:** `build-stage-audit-map.ts:72-105` (`isRecord`, `asString`, `asNumber`, `asStringArray`) ŌĆö tolerant helpers that never throw on malformed JSONB.

**Apply to:** All new mapper code (Plan 03). NEVER use `as any` casts when reading `decision_details.inputs.*` per RESEARCH.md anti-patterns.

```ts
function isRecord(v: unknown): v is Record<string, unknown> {
  return typeof v === "object" && v !== null && !Array.isArray(v);
}
function asString(v: unknown): string | null {
  return typeof v === "string" && v.length > 0 ? v : null;
}
function asStringArray(v: unknown): string[] {
  if (!Array.isArray(v)) return [];
  return v.filter((x): x is string => typeof x === "string");
}
```

### Inngest replay-safety

**Source:** `classifier-label-resolver.ts:98-128` ŌĆö `step.run("resolve-debtor", ...)` wraps the entire resolver call. CLAUDE.md hard rule: "Alle side effects in `step.run()` ŌĆö buiten herhaalt bij replay."

**Apply to:** Plan 02's new Layer-2 single-hit `lookupCandidateDetails` call. It lives inside `resolve-debtor.ts` which is already invoked from within `step.run("resolve-debtor", ...)` ŌĆö no new step needed. Do NOT add a separate `step.run` for it (would re-execute the layer-2 lookup but keep the previous step's data).

### Zod schema versioning (back-compat-via-optional)

**Source:** `nxt-zap-client.ts:135-145` ŌĆö all the nullable-optional fields (`brand_id`, `country_id`, `city`, `classification`, `email`).

**Apply to:** `contact_person` and `recent_invoices` per Plan 01. Both `.optional()` lets the resolver parse old Zap responses if Plan 01 ships before the Zap edit (RESEARCH.md Pitfall 5).

### Discriminated-union exhaustiveness

**Source:** RESEARCH.md Pattern 3 + Stage 3 mapper's tolerant fallthrough behavior.

**Apply to:** Plan 03 mapper switch. Use `default:` arm that sets `stage2Inputs = null` (tolerant ŌĆö unknown future method degrades to legacy render). Do NOT `throw` in the mapper default ŌĆö the mapper contract is "never throw on bad JSONB" (build-stage-audit-map.ts line 12: "Inputs are intentionally tolerant: malformed JSONB shapes default to null / [] ŌĆö never throw.").

### Three-tier bound on `recent_invoices`

**Source:** RESEARCH.md Pitfall 6 ŌĆö defense-in-depth.
