## Typography

| Role | Size | Weight | Line Height | Font Family | Usage |
|------|------|--------|-------------|-------------|-------|
| Body | 14px (`text-sm`) | 400 (normal) | 1.43 (`leading-snug`) | Geist Sans | Table cells, card descriptions, tooltip content, "Updated X ago" text |
| Label | 14px (`text-sm`) | 400 (normal) | 1.43 (`leading-snug`) | Geist Sans | KPI card titles, tab labels, table column headers, section headings within tabs |
| Heading | 24px (`text-2xl`) | 700 (bold) | 1.33 | Geist Sans | Page title "Executive Dashboard" |
| KPI Value | 30px (`text-3xl`) | 700 (bold) | 1.2 | Geist Mono | KPI card primary values (e.g., "142", "~EUR8,520", "87/100") |
| Subhead | 16px (`text-base`) | 400 (normal) | 1.5 | Geist Sans | Tab section subheadings (e.g., "Runs Over Time", "Per-Project Health") |

**Weight strategy:** Two weights only -- 400 (normal) and 700 (bold). Visual hierarchy between Body, Label, and Subhead is achieved through font size (14px vs 16px) and spatial context (position, proximity to data), not through intermediate weights. Bold (700) is reserved exclusively for the page heading and KPI card values, which are the primary focal points.

**Key rule:** KPI card values use `font-mono` (Geist Mono) for tabular number alignment. All other text uses the default sans-serif (Geist Sans). No other fonts are permitted in this phase.

## Layout Contract

### Primary Visual Anchor

Primary visual anchor: the 6 KPI cards above the fold, arranged in a 3x2 grid. Within each card, the KPI Value (30px Geist Mono bold) is the dominant element -- the largest and heaviest text on the page. The user's eye lands here first, then scans card titles (14px normal) and trend indicators for context. Everything below the fold (tabs, charts, tables) is secondary drill-down content.

### Page Structure

```
/executive (within (dashboard) layout group)
+----------------------------------------------------------+
| Page header: "Executive Dashboard" + PeriodSelector       |
+----------------------------------------------------------+
| KPI Grid (3x2)                                            |
| [Active Automations] [Exec Throughput] [Health Score]     |
| [~Est Hours Saved]   [~Est Financial]  [Orq.ai Usage]    |
+----------------------------------------------------------+
| Tabs                                                       |
| [Activity & Perf] [Projects & Life] [ROI & Cost] [Source] |
| +------------------------------------------------------+  |
| | Tab content area                                      |  |
| | (charts + tables per tab)                             |  |
| +------------------------------------------------------+  |
+----------------------------------------------------------+
```

### Responsive Breakpoints

| Breakpoint | KPI Grid | Tab Content | Source Status Cards |
|------------|----------|-------------|---------------------|
| < 640px (mobile) | 1 column (`grid-cols-1`) | Single column, charts stack | Single column |
| 640-1023px (tablet) | 2 columns (`sm:grid-cols-2`) | Single column, tables scroll horizontally | 2 columns |
| >= 1024px (desktop) | 3 columns (`lg:grid-cols-3`) | Side-by-side charts where applicable | 3 columns (`lg:grid-cols-3`) |

### Grid Specifications

- **KPI grid:** `grid gap-4 grid-cols-1 sm:grid-cols-2 lg:grid-cols-3` -- 6 cards fill a 3x2 grid on desktop
- **Source status grid:** `grid gap-4 grid-cols-1 sm:grid-cols-2 lg:grid-cols-3` -- 3 cards side by side on desktop
- **Page padding:** `p-6` (24px) consistent with existing dashboard page
- **Section gap:** `space-y-6` (24px) between page header, KPI grid, and tabs
- **Tab content padding:** `py-6` (24px) inside each TabsContent
- **Chart minimum height:** `min-h-[300px]` for area/line charts, `min-h-[250px]` for donut/bar charts
- **Table maximum height:** None (tables render fully, page scrolls)

## Interaction Contract

### Period Selector
- Type: shadcn Select component (client component)
- State: URL search params (`?period=30d`)
- Options: "Last 7 days" (`7d`), "Last 30 days" (`30d`), "This month" (`month`), "This quarter" (`quarter`)
- Default: `30d`
- Behavior: Changing the period triggers a router push, which re-renders the server component page with filtered data
- Width: `w-[160px]` fixed

### KPI Card Hover
- Estimated cards: hovering anywhere on the card surface shows a Tooltip with the formula explanation
- Non-estimated cards: no hover effect beyond default Card ring

### Health Dot
- Static indicator -- no hover interaction
- 3 states:
  - Green: `bg-green-500` filled circle, 8px diameter
  - Yellow: `bg-amber-500` filled circle, 8px diameter
  - Red: `bg-red-500` filled circle, 8px diameter

### Trend Indicator
- Up arrow (TrendingUp icon, 12px) + percentage in green when current > previous
- Down arrow (TrendingDown icon, 12px) + percentage in red when current < previous
- Dash (Minus icon, 12px) in muted-foreground when no change
- Format: `+12%` or `-5%` or `0%`

### Tables (Project Health, Agent Metrics, ROI)
- Column headers are clickable for sorting (ascending/descending toggle)
- Active sort column shows a small up/down arrow indicator
- Project name cells in Project Health table are links to the project detail page
- No row selection, no bulk actions, no pagination (executive dashboard shows all projects)

### Charts
- All charts use shadcn `ChartTooltip` on hover showing exact values
- All charts use shadcn `ChartLegend` below the chart
- Area charts use `fillOpacity={0.4}` for the filled area
- Line charts use `strokeWidth={2}`
- Donut chart shows the total count in the center

### Stale Data Warning
- Card gets `border-amber-300 dark:border-amber-700` border
- Below the "Updated X ago" text, append: "Data may be stale" in `text-[10px] text-amber-600 dark:text-amber-400`
- Zapier fallback warning: "Using data from [date] -- latest scrape had issues" in same styling

### Navigation
- New sidebar item: "Executive" with `BarChart3` icon from Lucide
- Position: between "Dashboard" (Home) and "Projects" (FolderOpen)
- Href: `/executive`

## States

### Loading State
- Full page skeleton using shadcn `Skeleton` component
- 6 skeleton cards in 3x2 grid matching KPI card dimensions
- Skeleton tabs below with placeholder content area
- File: `app/(dashboard)/executive/loading.tsx`

### Empty State (no snapshots yet)
- Centered layout with `BarChart3` icon (64px, muted-foreground) in a rounded muted background circle
- Heading: "No dashboard data yet"
- Body: "The dashboard aggregator has not run yet. Data will appear automatically within 2 hours as collectors populate snapshots."
- No action button (the aggregator runs automatically)

### Stale State
- Per-card: amber border + "Data may be stale" text
- Per-source in Source Status tab: yellow HealthDot + staleness timestamp

### Zapier Fallback State
- Zapier-derived KPI values show normally but with additional warning text
- Source Status tab Zapier card shows: "Using data from [fallback timestamp] -- latest scrape had issues" in amber text

### Error State (snapshot query fails)
- Show: "Unable to load dashboard data"
- Body: "There was a problem reading the latest snapshot. This is usually temporary -- try refreshing the page."
- No retry button (server component -- user refreshes the page)
