<!-- Input contract: test-results.json (written by results-analyzer) -->

test-results.json required fields for failure-diagnoser:
- `.results.overall_pass` — boolean (if true, nothing to diagnose)
- `.results.per_agent[].agent_key` — agent identifier
- `.results.per_agent[].role` — structural/conversational/hybrid
- `.results.per_agent[].scores` — per-evaluator object with median, variance, confidence_interval, pass, threshold, scale, runs
- `.results.per_agent[].category_scores` — per-category per-evaluator breakdown with median, pass, count
- `.results.per_agent[].worst_cases` — bottom 3 examples with eval_id, input, expected_output, actual_output, scores, category, reason
- `.results.per_agent[].evaluators_used` — evaluator names and thresholds
- `.results.per_agent[].total_failure_count` — number of failing examples

<!-- Output contract: iteration-proposals.json (consumed by prompt-editor Phase 31) -->

iteration-proposals.json schema:
- `.iteration` — integer iteration number
- `.proposed_at` — ISO-8601 timestamp
- `.per_agent[]` — array of per-agent proposals
  - `.agent_key` — agent identifier
  - `.approval` — "approved" | "rejected"
  - `.diagnosis` — plain-language diagnosis text
  - `.changes[]` — array of proposed changes
    - `.section` — XML tag name (e.g., "<task_handling>")
    - `.reason` — plain-language reasoning linking evaluator scores to change
    - `.before` — existing section content
    - `.after` — modified section content
</interfaces>

**Phase 1: Read Test Results and Identify Failing Agents**

Extract iterator.md Phase 1 (Steps 1.1-1.4) verbatim with these adjustments:

Step 1.1 — Parse Test Results:
- Read `{swarm_dir}/test-results.json`
- Extract `results.per_agent` array
- For each agent: collect agent_key, role, scores, category_scores, worst_cases, evaluators_used, total_failure_count
- Quick check: if `results.overall_pass` is true, display "All agents passing. No iteration needed." and STOP

Step 1.2 — Identify Failing Agents:
- For each agent in results.per_agent: check each evaluator's `pass` field in `scores`
- Agent is failing if ANY evaluator has `pass: false`
- Extract bottleneck evaluator: lowest median score relative to threshold

Step 1.3 — Build Failure Priority List:
- Sort failing agents by bottleneck score ascending (worst first)
- Bottleneck score = min(scores[eval].median for all failing evals)

Step 1.4 — Display Summary:
- If no agents failing: display "All agents passing. No iteration needed." and STOP
- If agents failing: display "{N} of {M} agents failing. Starting diagnosis."
