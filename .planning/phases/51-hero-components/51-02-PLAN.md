# Plan 51-02: AI Briefing Panel + Orq.ai Briefing Agent

**Goal:** Ship a dedicated Orq.ai Briefing Agent, a server action that invokes it and writes the result to `swarm_briefings`, a React panel consuming the latest briefing via realtime, and a 30-min Inngest cron for automatic refresh.

## Requirements covered
BRIEF-01, BRIEF-02, BRIEF-03

## Tasks

### T1. Zod output schema
Create `web/lib/v7/briefing/schema.ts`:
- `briefingAlertSchema`, `briefingActionSchema`, `briefingOutputSchema` as spec'd in 51-RESEARCH.md
- Export `BriefingOutput` type

### T2. Prompt builder
Create `web/lib/v7/briefing/prompt.ts`:
- `buildBriefingPrompt(input: BriefingInput): string` that returns the XML-tagged prompt from 51-RESEARCH.md with `{swarm_name}`, `{agents_json}`, `{jobs_json}`, `{events_summary}`, `{last_briefing_or_null}` substituted
- Keep prompt content as a single template literal -- easy to read and diff

### T3. Input aggregator
Add to `web/lib/v7/briefing/actions.ts`:
- `gatherBriefingInput(swarmId: string)`: server function using `createAdminClient` to fetch
  - project name
  - swarm_agents rows (pluck name, role, status, metrics)
  - swarm_jobs grouped by stage (counts)
  - agent_events from last 30 min grouped by event_type (counts)
  - last briefing (if any, for continuity)
- Returns `BriefingInput` ready for `buildBriefingPrompt`

### T4. Orq.ai Agent deployment helper
Create `web/lib/v7/briefing/deploy-agent.ts`:
- `ensureBriefingAgent()`: idempotent helper that
  1. Calls `get_agent` for key `swarm-briefing-agent`
  2. If missing, calls `create_agent` with the prompt + JSON schema + model chain from 51-RESEARCH.md
  3. Returns the agent id
- Stored in env/settings after first deployment to avoid repeat MCP calls (cache in `settings` table under key `briefing_agent_id`)

### T5. Server action: generateBriefing
Add to `web/lib/v7/briefing/actions.ts`:
```ts
"use server";
export async function generateBriefing(swarmId: string): Promise<{ ok: boolean }> {
  // 1. cache-check: select latest swarm_briefings where swarm_id=... and expires_at > now(); if exists, return ok
  // 2. gather input
  // 3. build prompt
  // 4. callOrqaiMcp for agent invocation (try direct invoke tool; fall back to REST POST /v2/agents/{id}/invoke)
  // 5. Zod-validate response; on failure, log to settings.orqai_briefing_last_error, return ok:false (UI will render last cached briefing)
  // 6. INSERT swarm_briefings row with narrative = JSON.stringify(validated), metrics_snapshot = input.events_summary + jobs counts, generated_at=now, expires_at=now+5min
  // 7. revalidatePath(`/swarm/${swarmId}`) to ensure SSR shells are fresh
}
```

### T6. `<KpiGrid>` component
Create `web/components/v7/briefing/kpi-grid.tsx`:
- `"use client"`
- Consumes `useRealtimeTable("jobs")` and the briefing's `metrics_snapshot` to compute:
  - Active jobs = count of jobs in ready+progress+review
  - Human review = count in review
  - Blocked = jobs with `priority = 'urgent'` AND `stage NOT IN ('done')` (proxy for blocked until a dedicated field lands)
  - Done today = count in done where `updated_at >= start-of-day`
- Renders 4-column (lg) / 2-column (md) / 1-column (sm) grid per UI-SPEC
- Each cell: label + big number

### T7. `<BriefingPanel>` component
Create `web/components/v7/briefing/briefing-panel.tsx`:
- `"use client"`
- Props: `swarmId: string`
- Consumes `useRealtimeTable("briefings")`; picks the most recent row by `generated_at`
- Parses `narrative` (stored as JSON string) with `briefingOutputSchema.safeParse`; on parse failure, renders the raw string in a `<pre>` fallback
- Layout per UI-SPEC: eyebrow with pulse dot + "Autonomous briefing", Cabinet 28.8px headline, Satoshi 16/1.5 summary, then `<KpiGrid>`, then footer row with "Updated X min ago" + `<form action={generateBriefingFromForm}>` regenerate button
- Alerts: if any alerts exist, render below summary as small inline pills with severity color (info=muted, warn=amber, critical=red)
- Empty state: per UI-SPEC copy
- Regenerate button uses `useTransition` for pending state

### T8. Wire into layout shell
Modify `web/components/v7/swarm-layout-shell.tsx`:
- Replace briefing placeholder with `<BriefingPanel swarmId={swarmId} />`
- KPI placeholder section is now INSIDE the briefing panel, so the second column becomes a smaller glass card labeled "Delegation graph - Phase 53" (keep placeholder, just shrink)
- Pass `swarmId` prop down from the parent `page.tsx` to `SwarmLayoutShell`

### T9. Inngest cron for 30-min refresh
Create `web/lib/inngest/functions/briefing-refresh.ts`:
- ID: `analytics/briefing-refresh`, cron `*/30 * * * *`, retries 3
- Fetches all swarms with at least one `swarm_agents` row
- For each swarm, `step.run(\`refresh-\${swarmId}\`)` calls a shared `generateBriefing` helper (same logic as the server action but without `revalidatePath`)
- Extract the core Orq.ai call + DB write into `web/lib/v7/briefing/generate.ts` so both server action and cron can call it
- Register in `web/app/api/inngest/route.ts` `serve({ functions: [...] })`

### T10. Ensure Briefing Agent deployed (one-time)
- Add a build-time check or manual script: `npx tsx web/scripts/deploy-briefing-agent.ts` that runs `ensureBriefingAgent()`
- Document in SUMMARY that this should be run once before first cron tick
- If MCP `create_agent` fails during autonomous execution, log and continue -- the UI still renders empty state, and the agent can be deployed manually later

### T11. Verify types
- `cd web && npx tsc --noEmit` clean
