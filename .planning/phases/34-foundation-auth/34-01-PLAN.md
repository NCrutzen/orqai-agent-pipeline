       ```bash
       npx shadcn@latest init

       ```

    4. Create `web/.env.local` with placeholder values (user fills in from Supabase/Azure dashboards):
       ```
       NEXT_PUBLIC_SUPABASE_URL=your-supabase-url
       NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=your-anon-key
       SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
       NEXT_PUBLIC_APP_URL=http://localhost:3000
       ```

    5. Create Supabase client factories following RESEARCH.md patterns exactly:
       - `web/lib/supabase/client.ts` -- browser client using `createBrowserClient`
       - `web/lib/supabase/server.ts` -- server client using `createServerClient` with cookie management
       - `web/lib/supabase/admin.ts` -- admin client using `createClient` with `SUPABASE_SERVICE_ROLE_KEY` (server-only, for invites)

    6. Create `web/proxy.ts` (Next.js 16 replaces middleware.ts) following RESEARCH.md Pattern 2:
       - Create Supabase server client with cookie forwarding
       - Call `supabase.auth.getUser()` (NOT `getSession()` -- security requirement from research)
       - If no user and path is not `/login` or `/auth/*`, redirect to `/login`
       - Matcher: exclude static files, images, favicon
       - NOTE: proxy.ts in Next.js 16 runs on Node.js runtime (not Edge)

    7. Create `supabase/schema.sql` with full database schema following RESEARCH.md Pattern 4:
       - `projects` table: id (UUID PK), name (TEXT NOT NULL), description (TEXT), created_by (UUID FK auth.users), created_at, updated_at
       - `project_members` table: project_id + user_id composite PK, joined_at
       - Index: `idx_project_members_user_id` on project_members(user_id) for RLS performance
       - RLS enabled on both tables
       - Policy "Users see own projects": SELECT on projects WHERE id IN (SELECT project_id FROM project_members WHERE user_id = (select auth.uid()))
       - Policy "Authenticated users can create projects": INSERT on projects WITH CHECK auth.uid() = created_by
       - Policy "Members see project members": SELECT on project_members WHERE project_id IN subquery
       - Policy "Project creators can add members": INSERT on project_members WITH CHECK project_id IN (SELECT id FROM projects WHERE created_by = (select auth.uid()))
       - Policy "Any member can add members": INSERT on project_members WITH CHECK for any member (PROJ-04 says all members have equal access, so any member can invite, not just creator)
       - CRITICAL: Always use `(select auth.uid())` wrapper for query planner caching (Pitfall 2 from research)

    8. Update `web/app/layout.tsx` to set up root HTML with Inter font and Tailwind globals. Clean minimal style -- white/light background.

    9. Update `web/app/globals.css` to Tailwind defaults with clean minimal theme -- subtle borders, no corporate branding.
