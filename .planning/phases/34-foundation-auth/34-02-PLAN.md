       - Search users: `/users` with `$search` on displayName and mail, `$select` id/displayName/mail/jobTitle, `$top` 10

    2. Create `web/app/api/invite/route.ts` following RESEARCH.md invite pattern:
       - POST handler accepting `{ email, projectId, supabaseUserId? }`

       - Two paths based on whether user already exists:

       - CRITICAL: Use admin client (service_role key) for inviteUserByEmail -- never expose to client
       - Handle race condition: if user already a member, return success (idempotent)
