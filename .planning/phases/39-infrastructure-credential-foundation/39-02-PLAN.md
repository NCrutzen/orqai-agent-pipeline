**File 1: web/components/credentials/create-credential-modal.tsx**

Client component ("use client"). Follow create-project-modal.tsx pattern exactly for Dialog structure, state management, form submission, and loading states.

Props:
```typescript
interface CreateCredentialModalProps {
  authProfileTypes: AuthProfileType[];
  trigger?: React.ReactNode;
}
```

State: open, name, selectedAuthType (AuthProfileTypeId, default "username_password"), values (Record<string, string>), projectIds (string[]), error, loading.

Form structure (from UI-SPEC > Create Credential Modal):
- Dialog with `className="sm:max-w-lg"`
- DialogHeader: title "Store Credential", description "Credential values are encrypted and cannot be retrieved after saving."
- Form with `className="flex flex-col gap-4 mt-4"`:
  1. **Name field**: label "Name" with required asterisk (`text-sm font-semibold`), Input placeholder "NXT Production Login", helper "A descriptive name to identify this credential" (`text-xs text-muted-foreground`)
  2. **Auth type field**: label "Authentication Type" (`text-sm font-semibold`), AuthProfileTypeSelector component
  3. **Dynamic value fields**: Render input fields based on `selectedAuthType`. Look up the selected type's `field_schema.fields` from `authProfileTypes` prop. For each field:
     - label with field.label (`text-sm font-semibold`), required asterisk if field.required
     - If field.type === "secret": paste-only Input (see Paste-Only behavior below)
     - If field.type === "text": regular Input
     - Helper text under secret fields: LockKeyhole icon + "Write-once -- value cannot be viewed after saving" (`text-xs text-muted-foreground flex items-center gap-1`)
  4. **Custom type extra**: If selectedAuthType === "custom", show an "Add Field" Button (variant="outline", size="sm") that adds a key-value row pair. Each row has: key Input (placeholder "Field name"), value Input (paste-only secret), and a Trash2 icon button to remove.
  5. **Project linking** (optional): Multi-checkbox list of user's projects. Simplify: a note "Project linking will be available after credentials are created" (`text-xs text-muted-foreground`) -- defer the project multi-select to a future enhancement.
- DialogFooter: Cancel button (variant="outline"), Submit button with Plus icon + "Store Credential", loading state shows Loader2 animate-spin + "Storing..."

**Paste-Only Secret Input Behavior** (from UI-SPEC):
- Input with `type="password"` and `readOnly` attribute by default
- `placeholder="Paste your credential value"`
- `onPaste` handler: capture `e.clipboardData.getData("text")`, set field value in state, remove readOnly
- `onKeyDown` handler: if not Ctrl/Cmd+V paste shortcut, call `e.preventDefault()`. Optionally show brief helper text.
- After paste: value held in React state but field shows masked dots (type="password")
- No "show password" toggle

Form submission:
- Validate with Zod: name min 1 max 200, authType string, values record of strings (non-empty)
- POST to `/api/credentials` with `{ name, authType: selectedAuthType, values }`
- On success: close modal, reset form, `router.refresh()`, toast "Credential stored securely" (use sonner toast if available, otherwise skip toast)
- On error: show error text below form, keep modal open

Default trigger button: `<Button><Plus className="size-4" />Store Credential</Button>`
