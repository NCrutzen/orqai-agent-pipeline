## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM agent output → agent_runs.rule_key | Untrusted `parsed.category_key` flows into a string interpolated into rule_key. Used downstream as a Map key, never as SQL. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|

| T-089-02-02 | Tampering / replay | Inngest replay of step.run("llm-call") regenerates rule_key | accept | rule_key is deterministic from parsed.{category_key, confidence}; same input → same key on every replay. crypto.randomUUID() remains inside step.run per Phase 65 learning. |
| T-089-02-03 | Tampering | LLM emits category_key="; DROP TABLE..." | accept | rule_key is never interpolated into raw SQL; only used as a string in Supabase JS .insert payload (parameterized) and as a Map.has(...) key. No injection vector. parsed.category_key is also constrained by the Zod enum in Stage1OutputSchema. |
