# Phase 82.2 Plan 03: pipeline_health table + stage0 coverage RPCs Summary

One-liner: New `public.pipeline_health` table plus two stable plpgsql/sql RPCs (`stage0_coverage_24h`, `stage0_backfill_candidates`) wired for Plans 09/10 to consume without further schema work.

## What shipped

`supabase/migrations/20260513b_pipeline_health.sql` (176 lines) containing:

1. **`public.pipeline_health` table** — one row per probe-run × mailbox, with:
   - `probe_run_id uuid NOT NULL` to group all mailboxes from one cron tick
   - `mailbox text`, `swarm_type text`, `stage1_count bigint`, `stage0_count bigint`
   - `coverage_pct numeric(5,4)` — **GENERATED ALWAYS AS STORED**, NULL when `stage1_count=0`
   - `breached boolean` — **GENERATED ALWAYS AS STORED**, `true` iff `stage1_count > 0 AND coverage < 0.99`
   - `recorded_at timestamptz DEFAULT now()`
   - Index on `(recorded_at DESC)` for trend reads
   - Partial index on `(breached, recorded_at DESC) WHERE breached=true` for cheap /gsd-progress breach queries

2. **RLS** — `service_role` FOR ALL only (mirror of `pipeline_events` posture from `20260506a`); no anon/authenticated access at all because coverage figures are operator-only.

3. **`public.stage0_coverage_24h(mailbox_arg text, swarm_arg text)`** — SQL STABLE function returning `(stage1_count bigint, stage0_count bigint)`. Implementation copied verbatim from RESEARCH §Coverage query (lines 569–594). Uses 25h buffer on the Stage 0 lookback to absorb Stage 0/Stage 1 ordering races.

4. **`public.stage0_backfill_candidates(window_days integer)`** — plpgsql STABLE function returning `(email_id uuid, swarm_type text, completed_at timestamptz, result jsonb)`. Implementation matches RESEARCH §Backfill (lines 305–323) — gap-finding CTE with `LEFT JOIN LATERAL` to `automation_runs` via `email_pipeline.emails.source_id` join key. Input validation:
   - `window_days IS NULL OR window_days < 1` → `RAISE EXCEPTION` with `ERRCODE = '22023'`
   - `window_days > 30` → silently clamped via `LEAST(window_days, 30)` (D-05 cap)

5. **Grants** — `EXECUTE` on both RPCs to `service_role`; `ALL` on table to `service_role`.

6. **Header comment** referencing Phase 82.2 D-04, D-05, D-07, D-08 and pointing at RESEARCH §Backfill and §Coverage query.
