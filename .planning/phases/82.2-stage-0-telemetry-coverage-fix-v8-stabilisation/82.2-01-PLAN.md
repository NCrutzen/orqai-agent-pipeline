<objective>
Add a partial UNIQUE index on `public.pipeline_events (email_id, swarm_type, stage) WHERE email_id IS NOT NULL`.

Purpose: Close the duplicate-row class permanently. RESEARCH.md verified there is currently NO unique constraint on pipeline_events. Without this index, both the live worker (D-01 single-emit refactor) and the backfill (D-03) can write duplicate Stage-0 rows on Inngest replay. With the index, INSERT-on-conflict is well-defined and the worker/backfill can become true UPSERTs.

Output: One migration file. After apply_migration, an attempt to INSERT a duplicate (email_id, swarm_type, stage=0) row fails with a unique-violation error.
</objective>
