# Phase 44 — UI Design Contract

> Visual and interaction contract for frontend components in Phase 44. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note:** Phase 44 has a small UI surface. No new pages or dashboards are introduced. The visual work is limited to two new badge components rendered on existing project cards and the existing dashboard page. All data collection (Zapier scraper, Orq.ai collector, Inngest crons, snapshot tables) is backend-only with no UI.

## Component Inventory

### 1. ProjectStatusBadge

**File:** `web/components/project-status-badge.tsx`
**Pattern:** Config-driven badge (identical to `StepStatusBadge`)
**Variants:** 4 statuses

| Status | Label | Icon | Variant | className |
|--------|-------|------|---------|-----------|
| idea | Idea | `Lightbulb` | outline | `text-muted-foreground` |
| building | Building | `Hammer` | default | `bg-blue-500/10 text-blue-600 border-blue-200 dark:border-blue-800 dark:text-blue-400` |
| testing | Testing | `FlaskConical` | default | `bg-amber-500/10 text-amber-600 border-amber-200 dark:border-amber-800 dark:text-amber-400` |
| live | Live | `Rocket` | default | `bg-green-500/10 text-green-600 border-green-200 dark:border-green-800 dark:text-green-400` |

**Props:**
```typescript
interface ProjectStatusBadgeProps {
  status: "idea" | "building" | "testing" | "live";
  className?: string;
}
```

**Sizing:** Uses shadcn `Badge` defaults: height 20px (h-5), text 12px (text-xs), icon 12px (size-3), horizontal padding 8px (px-2), pill shape (rounded-4xl).

### 2. AutomationTypeTag

**File:** `web/components/automation-type-tag.tsx`
**Pattern:** Config-driven badge, visually subordinate to ProjectStatusBadge
**Variants:** 5 types (including "unknown" default)

| Type | Label | Icon | Variant | className |
|------|-------|------|---------|-----------|
| zapier-only | Zapier | `Zap` | secondary | (default secondary styling) |
| hybrid | Hybrid | `GitBranch` | secondary | (default secondary styling) |
| standalone-app | App | `AppWindow` | secondary | (default secondary styling) |
| orqai-agent | Agent | `Bot` | secondary | (default secondary styling) |
| unknown | Unclassified | `HelpCircle` | outline | `text-muted-foreground` |

**Props:**
```typescript
interface AutomationTypeTagProps {
  type: "zapier-only" | "hybrid" | "standalone-app" | "orqai-agent" | "unknown";
  className?: string;
}
```

**Visual treatment:** Uses `variant="secondary"` to appear less prominent than the status badge. Same height (h-5) and text size (text-xs) as badges, but the muted secondary background creates visual hierarchy: status badge is the primary signal, type tag is supplementary.

### 3. ProjectCard (Modified)

**File:** `web/components/project-card.tsx` (existing, modified)
**Changes:** Add ProjectStatusBadge and AutomationTypeTag to card header.

**Layout change in CardHeader:**

Current layout:
```
[Title]                    [Badge: "No runs yet"]
[Description]
```

New layout:
```
[Title]
[ProjectStatusBadge]  [AutomationTypeTag]
[Description]
```

**Specific layout rules:**
- Remove the existing hardcoded "No runs yet" outline badge
- Place ProjectStatusBadge and AutomationTypeTag in a flex row below the title: `flex items-center gap-2 mt-1`
- ProjectStatusBadge comes first (more prominent), AutomationTypeTag second (subordinate)
- Both badges use the same height (h-5) for alignment
- On narrow cards, badges wrap to a second row (flex-wrap)

**Updated props interface:**
```typescript
interface ProjectCardProps {
  project: {
    id: string;
    name: string;
    description: string | null;
    updated_at: string;
    status: "idea" | "building" | "testing" | "live";
    automation_type: "zapier-only" | "hybrid" | "standalone-app" | "orqai-agent" | "unknown";
    project_members: { user_id: string }[];
  };
}
```

## Accessibility Notes

- Badge icon + text pairing: Icons are decorative (the text label carries the meaning). Icons use `aria-hidden="true"` implicitly via lucide-react defaults. No additional ARIA needed.
- Color is not the sole indicator: Each status has a unique icon AND label text alongside its color, meeting WCAG 2.1 1.4.1 (Use of Color).
- Badge contrast: All badge text colors against their backgrounds meet WCAG AA 4.5:1 minimum contrast (verified: blue-600 on blue-500/10, amber-600 on amber-500/10, green-600 on green-500/10 all exceed 4.5:1 on white background).
