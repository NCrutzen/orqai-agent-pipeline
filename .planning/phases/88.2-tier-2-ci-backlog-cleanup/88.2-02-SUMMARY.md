## Async-vs-sync cookies() decision (Pitfall 2 verification)

**Decision: sync mock.** Consumer call site at `web/lib/supabase/server.ts:5`
uses `const cookieStore = await cookies();`. Since `await nonPromise === nonPromise`,
the sync mock works correctly through the await. Verified live: stage-0/1/2/3
tests that previously failed at the cookies boundary now reach their next
expression cleanly (then fail at downstream assertion-drift, not cookies).

## Why the failure-count target wasn't fully hit (47→29 vs planned 47→≤9)

The planner's failure attribution in RESEARCH.md §"Test Failure Inventory"
was partially incorrect. Two specific mis-attributions:

1. **`lib/pipeline/__tests__/stages.test.ts` — 4 failures.** Plan classified
   these as `admin.schema is not a function` chain failures (W1). They are
   actually **pure assertion-drift**: the test asserts `PIPELINE_STAGES`
   has length 7 with specific names, but the registry now has 8 stages
   (added "discussion" at stepOrder 0). No Supabase mock involvement at
   all — the file imports `getStageByName` from `../stages`, not from a
   Supabase loader. Wave 3 scope.

2. **`app/(dashboard)/automations/[swarm]/stage-{0,2}/__tests__/page.test.tsx`
   — 2 failures.** Plan expected all 4+5 stage tests to pass after cookies
   fix. After cookies AND admin-chain fixes land, 2 assertion-drift failures
   remain: tests expect `/No rows yet/i` text but the rendered empty state
   uses different copy. Wave 3 scope.

3. **`kanban-loader.test.ts` — 2 failures.** Plan expected 2 chain failures.
   After admin-mock migration (no change needed — this test had its own
   working inline `.not`-supporting builder), 2 assertion-drift failures
   surfaced: test expects `mailbox_id` in SELECT cols but production
   `loadKanbanRows` SELECT shape doesn't include it; test expects specific
   `email_metadata` shape that drifted. Wave 3 scope. **No change to this
   file in this plan** — its inline mock was already chain-complete; the
   `.not is not a function` claim in PLAN.md was incorrect.

After this plan, **every remaining failure is assertion/fixture-drift**.
Wave 3 (Plan 88.2-04) will close them per D-07/D-08/D-09 (fix-or-skip
