- **Found during:** F4 triage ŌĆö after the `message_id` validator was satisfied, the handler reached a new `loadBrandRegister` call that hit the (stub-creds) Supabase and threw `MalformedRegistryError: swarm row not found`.

- **Why critical:** without it, all 6 invoice-copy tests crash before reaching their assertions ŌĆö they can't distinguish "handler logic broke" from "test isn't isolated from the registry."
- **Commit:** `555dbfd`

**2. [Rule 3 - Blocking issue] Stub-env detection for live-DB tests**
- **Found during:** triage of `lib/swarms/__tests__/sales-email-stub.test.ts` + `__tests__/static-checks/swarm-hard-separation.test.ts`.
- **Issue:** Plan 02's `test-setup.ts` seeds dummy Supabase env vars so `createAdminClient()` constructs without `.env.local`. The dummies are not reachable hosts, so the `skipIf(!hasCreds)` gates these suites use evaluate to `false` and `beforeAll` actually fetches ŌåÆ ECONNREFUSED.
- **Fix:** detect the well-known stubs (`localhost:54321`, `test-service-role-key`) and treat them as "no creds." Real `.env.local` values still register as creds.
- **Commit:** `8c935fa`

**3. [Plan-scope] kanban-loader.test.ts asserts mailbox_id is NOT selected**
- **Found during:** Plan 02 SUMMARY noted "mailbox_id in SELECT cols but production loadKanbanRows SELECT shape doesn't include it." Production sources `mailbox_id` from a separate `loadEmailMailboxes` call (`kanban-loader.ts:125`) and injects it client-side.
- **Fix:** inverted the SELECT-col assertion (`.not.toContain("mailbox_id")`) and updated the `email_metadata` shape assertion to expect `mailbox_id: null` when the mailboxes map isn't seeded.
- **Commit:** `d6fb7a7`
