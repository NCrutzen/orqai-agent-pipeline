# Phase 88.2: Tier-2 CI backlog cleanup — Specification

**Created:** 2026-05-21
**Updated:** 2026-05-21 (rev2 — post PR #34 merge; scope narrowed to gate-greening)
**Ambiguity score:** 0.17 (gate: ≤ 0.20)
**Requirements:** 5 locked

## Goal

Make the now-live `pr-checks.yml` gates pass green end-to-end on a clean PR against `main`: burn down 194 ESLint errors to 0 and resolve the 47 failing Vitest tests across 14 files (or explicitly `.skip()` non-Tier-2 ones with tracked todos) — so every subsequent PR is mechanically gated by working checks instead of red checks the team learns to ignore.

## Background

The "land the workflow + branch protection" half of Tier-2 already shipped:

- **PR #34** (merged 2026-05-21, `4906a94`) — `.github/workflows/pr-checks.yml` with codegen / lint / typecheck / test / build steps, `"typecheck": "tsc --noEmit"` in `web/package.json`, and `web/types/email-reply-parser.d.ts` shim.
- **Commit `79617e9`** on `main` — verifies GitHub branch protection blocks direct push to `main`.

The remaining problem is the gates are red:

- **Lint:** `npm run lint` exits 1 with **194 errors / 179 warnings**. Distribution:
  - 108 errors (56%) in `web/lib/automations/` — legacy `any`-typed automation scripts the team treats as "trusted boundary" code.
  - 59 errors in `web/lib/inngest/`.
  - By rule: 143 `@typescript-eslint/no-explicit-any`, 37 `@typescript-eslint/no-require-imports`, ~12 React Compiler errors (cascading renders, impure render, JSX in try/catch), residual `prefer-const` / `prefer-as-const` / `react/no-unescaped-entities`.
  - The 179 warnings are almost entirely `no-unused-vars` on `_`-prefixed identifiers — ESLint config's `argsIgnorePattern` does not honour the convention used in tests.

- **Tests:** `npm test` exits 1 with **47 failed / 1137 passed / 32 skipped across 170 files (14 failing files)**. Three distinct root causes:
  1. **`cookies` called outside request scope** in 5 stage-shell tests (`stage-0/__tests__/page.test.tsx`, `stage-1/__tests__/load-page-data.test.ts`, `stage-1/__tests__/page-shell.test.tsx`, `stage-2/__tests__/page.test.tsx`, `stage-3/__tests__/page.test.tsx`) — these import server components that call `cookies()` from `next/headers` but the vitest setup has no mock for it.
  2. **Supabase admin mock missing chain methods** (`admin.from(...).select(...).eq(...).not is not a function`, `admin.schema is not a function`) in `_lib/__tests__/kanban-loader.test.ts`, `lib/pipeline/__tests__/stages.test.ts`, and others.
  3. **Test-fixture data shape drift** (`Cannot read properties of undefined (reading 'stage_decisions')`) in `tests/labeling/orq-agents-client.test.ts` and a handful of others — code-side schema evolved past the fixture snapshots.

Without this phase the CI gate is theatrical: every PR comes back red, contributors learn to merge anyway via admin override, and the Tier-2 enforcement framing in `docs/collaboration.md` becomes a dead letter. Phase 88.2 closes the loop by turning the gates from "live + red" to "live + green".

## Requirements

1. **Lint passes with zero errors**: `npm run lint` exits 0 in `web/`.
   - Current: 194 errors / 179 warnings; exits 1.
   - Target: 0 errors. Warnings are tolerated (ESLint default exit behaviour). Achieved via:
     - Add `web/lib/automations/**` to ESLint `globalIgnores` in `eslint.config.mjs` (these scripts are tactical, externally credentialed, and not part of the dashboard surface — they get reviewed by humans on commit, not by ESLint).
     - Tune `@typescript-eslint/no-unused-vars` to honour `argsIgnorePattern: "^_"` and `varsIgnorePattern: "^_"` — clears the 179 `_`-prefix warnings + the corresponding `_args` errors.
     - Burn down residual errors in `lib/inngest/`, `app/(dashboard)/`, `debtor-email-analyzer/src/`, `components/v7/`, `components/automations/`, `tests/swarm-registry/`, `tests/labeling/`, `lib/zapier/`, `lib/dashboard/`, `components/graph/`, `app/api/` — by either narrowing types (`any` → concrete) or, for `no-require-imports`, converting to ESM-style imports. React Compiler errors must be fixed at source (no rule disable).
   - Acceptance: `(cd web && npm run lint)` exits 0; `(cd web && npm run lint 2>&1 | grep -E "^✖ .* error")` returns nothing.

2. **Tests pass with zero failures**: `npm test` exits 0 in `web/`.
   - Current: 47 failed tests across 14 files; exits 1.
   - Target: 0 failing tests. Achieved via:
     - **Vitest mock for `next/headers`** in `web/vitest.setup.ts` (or equivalent) so `cookies()`, `headers()`, and `draftMode()` return safe defaults during test runs. Fixes the 5 stage-shell test files.
     - **Supabase admin mock chain extension** in shared test utilities — add the missing `.not()`, `.schema()`, and any other gaps surfaced by failing tests in `kanban-loader.test.ts`, `stages.test.ts`, etc.
     - **Fixture refresh OR `.skip()` with tracked todo** for fixture-drift failures (`orq-agents-client.test.ts`, others). Fixing fixtures is in scope only when the drift is small and obvious; otherwise mark `.skip()` and file `.planning/todos/pending/2026-05-XX-test-fixture-{name}.md` so the gap is tracked, not lost.
   - Acceptance: `(cd web && npm test)` exits 0; the run output reports `0 failed`; every skipped test added by this phase has a corresponding `.planning/todos/pending/` entry referenced in its `.skip()` reason.

3. **Build still passes**: `npm run build` exits 0 in `web/`.
   - Current: presumed passing (Vercel preview-builds succeed) but unverified locally during this phase.
   - Target: any source-level fixes made for Req 1 or Req 2 do not break Next.js production build.
   - Acceptance: `(cd web && npm run build)` exits 0 on the head commit of the phase branch before merge.

4. **`pr-checks` workflow turns green on the phase PR**: end-to-end CI evidence.
   - Current: workflow runs are red on every PR opened against `main`.
   - Target: the PR that lands this phase shows all five `pr-checks /*` status checks green (codegen, lint, typecheck, test, build). After merge, a follow-up empty-commit PR (or the next real PR) also shows all five green — confirming the gate is durably green, not green only because of in-phase fixes.
   - Acceptance: GitHub Actions UI shows ✅ for all five `pr-checks /*` checks on this phase's merge PR; one follow-up PR (this can be a documentation tweak) shows the same five checks green.

5. **CI runtime budget verified**: cold/warm runs measured and inside the agreed budget.
   - Current: budget specified in old SPEC but never measured (workflow short-circuits on lint/test failure today).
   - Target: A green clean-cache run of `pr-checks.yml` completes in ≤ 8 minutes; a green warm-cache run in ≤ 5 minutes. Measured on `ubuntu-latest`.
   - Acceptance: One green run on a feature branch with no cache hits (e.g. fresh branch name) completes ≤ 8 minutes wall-clock per the GitHub Actions UI; a subsequent green run on the same branch completes ≤ 5 minutes. Recorded in a comment on the phase PR.

## Boundaries

**In scope:**
- ESLint config edits in `web/eslint.config.mjs` (ignore `lib/automations/**`, tune unused-vars patterns).
- Targeted source-level lint fixes in `lib/inngest/`, `app/(dashboard)/`, `debtor-email-analyzer/src/`, `components/v7/`, `components/automations/`, `tests/swarm-registry/`, `tests/labeling/`, `lib/zapier/`, `lib/dashboard/`, `components/graph/`, `app/api/`. Mostly narrowing `any` and converting `require()` to `import`.
- React Compiler error fixes at source (no rule disable).
- Vitest setup file changes (`web/vitest.setup.ts` or equivalent) to mock `next/headers#cookies` / `headers` / `draftMode`.
- Shared test-utility extension for Supabase admin mock (`.not()`, `.schema()`).
- Fixture refresh OR `.skip()` + todo for fixture-drift failures.
- One documentation note in `docs/collaboration.md` (or `CLAUDE.md`) confirming Tier-2 CI gate is live and green.
- The phase-PR cycle that demonstrates green checks end-to-end.

**Out of scope:**
- Refactoring `lib/automations/` away from `any` — the eslintignore exists precisely because that surface is not worth typing today; revisit when ownership stabilises.
- Rewriting failing tests as new tests — fix or skip, don't expand.
- Migrating Vitest → Jest or any other testing framework swap.
- Adding `check:supabase` as a CI step — pre-push hook covers it; revisit if drift is observed.
- Tier-3 governance (CODEOWNERS-required review, signed commits, linear history) — defers to second-dev onboarding.
- Playwright `test:e2e` integration into CI — slow and credential-dependent.
- Cleaning up the 179 warnings beyond what falls out of the `argsIgnorePattern` change — warnings are tolerated by `npm run lint`.
- Per-developer Supabase schema overlays — revisit only if shared-DB collisions actually bite.
- Vercel preview-deploy gating.

## Constraints

- **Runtime budget:** ≤ 8 min cold, ≤ 5 min warm on `ubuntu-latest` (Req 5).
- **Node version:** already pinned via `actions/setup-node` in `pr-checks.yml`; phase must not change it.
- **No secrets:** PR-checks workflow must continue to require zero repo secrets so fork PRs still gate correctly.
- **No bypass via rule disable:** React Compiler errors and other rule violations must be fixed at source. Per-file `eslint-disable` is allowed only with a one-line reason comment naming the constraint that justifies it.
- **No bypass via branch-protection admin override:** the phase PR must merge through green checks, not via admin bypass.
- **Skip-with-todo discipline:** every `.skip()` added by this phase must reference a `.planning/todos/pending/` entry; otherwise the gap is invisible.
