## What Was Built

A single new exported async function in `web/app/(dashboard)/automations/[swarm]/stage-1/actions.ts`:

```typescript
export async function approvePrediction(input: {
  row_id: string;
  swarm_type: string;
  decision: "approve" | "reject";
}): Promise<{ ok: true }>
```

Behavior:
1. Validates `input` with `approveInputSchema` (Zod).
2. Selects the `automation_runs` row by `id = row_id`, pulling `swarm_type, entity, rule_key, result, email_id`.
3. Extracts `predicted_category`, `message_id`, `source_mailbox` from `result` jsonb.
4. Defensive fallback: if `message_id`/`source_mailbox` are missing, cross-schema joins `email_pipeline.emails` keyed on `email_id`.
5. Hard-throws on missing `predicted_category`, `rule_key`, or `email_id` (contract violation).
6. Delegates to in-module `recordVerdict({ ... })` without `override_category` (Approve confirms the prediction as-is).
