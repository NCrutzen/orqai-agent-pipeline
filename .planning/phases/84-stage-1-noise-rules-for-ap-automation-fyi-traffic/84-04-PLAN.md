## 3. D-05 Gate Evaluation (Day 7+)
Per (category, swarm) row, choose gate path:
- **Wilson path:** live N >= 30 AND wilsonCiLower(n, agree) >= 0.95 -> promotion eligible.
- **Corpus path:** live N < 30 BUT CORPUS-SAMPLES.md shows >=10 hand-confirmed positives AND 0 hand-confirmed FPs AND shadow-window flagged 0 operator FPs -> promotion eligible.
- **Neither:** hold in `candidate`; investigate FPs OR collect more corpus evidence; re-evaluate weekly.

**Sales-email per-swarm review (R-04 mitigation):** if a category's sales-email row produced operator-flagged FPs that the debtor-email row didn't, drop ONLY the sales-email row via `UPDATE swarm_noise_categories SET enabled=false WHERE swarm_type='sales-email' AND category_key='<key>';` before promoting the debtor-email row.

## 4. Promotion Action (Operator)

## 5. Rollback Procedure
- **Per-category single-swarm rollback (R-04):** `UPDATE swarm_noise_categories SET enabled=false WHERE swarm_type=$1 AND category_key=$2;` -- preserves the row + audit trail.
