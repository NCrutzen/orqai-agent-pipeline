# Phase 82.7.2 Plan 03: F-01 brand-swatch tooltip hit-target enlargement — Summary

One-liner: Wrap the 6×6 brand-color visual swatch in a 14×14 transparent
inline-flex hit-target inside `predicted-row.tsx`, promoting it as the
`TooltipTrigger asChild` so Radix reliably receives pointer events —
matching the hit-area of the working sibling ↻ overridden-glyph tooltip.

## Root cause (per PATTERNS.md F-01 investigation)

`TooltipProvider` is NOT missing — it is mounted globally in
`web/app/layout.tsx:56`. The working sibling ↻ tooltip in the same
file (`predicted-row.tsx:194-211`) proves the Radix Tooltip wiring
works in this exact component. The delta was hit-target size:

| Element | Hit area | Tooltip status |
|---------|----------|----------------|
| Sibling ↻ glyph | ~14×14 (font-size 14, inline-block) | fires correctly |
| Brand swatch (before fix) | 6×6 | silent |
| Brand swatch (after fix) | 14×14 transparent wrapper around 6×6 dot | fires (expected) |

The 6×6 swatch's hover hit-target was too small to reliably register
`pointerenter` through the parent `<button>` row wrapper. Enlarging
the transparent trigger to 14×14 aligns the swatch with the working
sibling and is the minimal change that fixes the bug.

## What changed

Single edit at `predicted-row.tsx` L95-124 (swatch block):

- **Outer wrapper added** — a 14×14 `inline-flex` `<span>` with
  `alignItems/justifyContent: center` and `cursor: "default"`. NO role,
  NO aria-label (screen readers see only the inner element).
- **Inner span unchanged** — the original 6×6 colored dot retains its
  `role="img"`, `aria-label={`Brand: ${brandDisplayName(...)}`}`,
  border-radius, background, and `flexShrink: 0`.
- **`TooltipTrigger asChild`** now wraps the outer 14×14 wrapper instead
  of the inner 6×6 dot.
- **`TooltipContent`** body unchanged: `{brandDisplayName(row.entity_brand)}`.
- **`<TooltipProvider>` retained** around this `<Tooltip>` to match the
  sibling ↻ block exactly. Redundant given the global Provider at
  `layout.tsx:56`, but harmless and consistent with the working analog.

The diff is +26 / -9 lines on a single file.

## What did NOT change

- The sibling ↻ overridden-glyph tooltip block (`predicted-row.tsx`
  L194-211) is byte-identical. `git diff` confirms only the swatch
  block changed.
- No imports added or removed (`Tooltip`, `TooltipContent`,
  `TooltipProvider`, `TooltipTrigger` already imported).
- Row layout (`gridTemplateColumns: "200px 280px 1fr 84px 70px"`) and
  parent `<button>` semantics untouched.
- `brand-color.ts` helpers untouched.
- No data-path changes — Plan 02's `PredictedRow.entity` hydration
  remains the only data-side change in this phase.

resolved regardless of that downstream choice.
