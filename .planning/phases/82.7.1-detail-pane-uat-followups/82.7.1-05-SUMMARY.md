# Phase 82.7.1 Plan 05: StageBadge Em-Dash Placeholder Summary

`StageBadge` now renders a muted em-dash (`—`) placeholder chip when `label` is null, undefined, or whitespace — closing E-05 on the renderer side while preserving Plan 02's fade-out behavior and the D-2 grid contract.

## What Changed

`web/app/(dashboard)/automations/[swarm]/_shell/row-list.tsx` — `StageBadge` only (L192-227 in the new file). Plan 02's edits at L34-99 + L100-185 remain byte-identical (verified by acceptance grep: `_unmount` ×7, `transition: "opacity 150ms ease-out"` at L135, single `gridTemplateColumns` literal).

Behavior:

1. `const trimmed = (label ?? "").toString().trim(); const isEmpty = trimmed.length === 0;`
2. When `isEmpty` → border = `var(--v7-muted)`, background = `transparent`, color = `var(--v7-muted)`, opacity `0.7`, text = `"—"`, `aria-label="No category"`, `data-empty="true"`.
3. When populated → identical to pre-change behavior (same `badgeColors(variant)` shell).
4. Chip shell tokens (`padding: "2px 8px"`, `borderRadius: "var(--v7-radius-pill)"`, mono font, `lineHeight: 1.3`, `flexShrink: 0`) unchanged in both branches → row rhythm preserved per D-15.
