# Phase 82.7.1 Plan 03: Brand-color swatch tooltip (E-03) Summary

Wrapped the existing 6√ó6 brand-color swatch on `PredictedRow` in a Radix Tooltip
that surfaces the brand display name (e.g. "Smeba", "Smeba-Fire"), and added a
colocated `brandDisplayName()` helper to `web/lib/swarms/brand-color.ts` so the
display-name map lives next to the existing `brandColorToken()` color-token map.

## Implementation Notes

- `BRAND_DISPLAY_NAME` map covers the 5 Benelux brands per `project_brand_scope.md`
  memory: `smeba` ‚Üí "Smeba", `smeba-fire` ‚Üí "Smeba-Fire", `berki` ‚Üí "Berki",
  `sicli-noord` ‚Üí "Sicli-Noord", `sicli-sud` ‚Üí "Sicli-Sud". iccafe/iccafe-france
  are intentionally absent (deferred).
- Fallback display name is `"Unknown brand"` (parallels `FALLBACK_TOKEN = "--v7-muted"`).
- The swatch was previously `aria-hidden="true"` since the dot alone was opaque to
  operators. Now that hover reveals the name, the element carries meaningful
  information, so it was promoted to `role="img"` + `aria-label="Brand: {name}"`.
- Tooltip uses Radix defaults (no custom `delayDuration`) to stay consistent
  with the existing ‚Üª overridden-stage tooltip in the same file.
- `_shell/row-list.tsx` was intentionally NOT touched (D-08): noise view rows
  are rendered through the same `PredictedRow` component (per
  `stage-1/page.tsx` L972-997 mapping), so one swatch edit covers both
  surfaces ‚Äî no duplicate import of `brandColorToken`/`brandDisplayName` in
  the shell file.
