<objective>
Split the Wilson-CI promotion/demotion math by predictor so LLM mistakes don't drag the regex's accuracy stream down (and vice versa). Same cron, same N≥50 floor, just two streams per mailbox.

Add the D-03 calibration-drift signal as a second pass inside the same cron run — when the high-conf LLM FP rate exceeds 5% on a rolling-50 window per mailbox, emit `classifier/calibration_drift.detected` and write an alarm audit row. At 2-5% FP, audit-row warn only (no event).

Purpose: This is the feedback-math change that completes the predictor attribution loop. Without it, Plan 05 writes predictor to agent_runs but nothing reads it — the cron continues aggregating per-mailbox and the LLM-pollutes-regex risk persists.

The D-03 threshold values (2% warn / 5% alarm) and emission shape (event on alarm only, audit row on both) are planner-locked per RESEARCH §3 and user directive 2 (research recommendation: 2% warn / 5% alarm, mirror D-10 event-only pattern).

Output: One typed event declaration + per-predictor cron loop + D-03 alert.
</objective>
