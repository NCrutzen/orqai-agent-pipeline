# Phase 999.8 Plan 07: Wave 5 — Stage 1 chip-strip filter chips (predictor + confidence) Summary

Adds two filter chip rows to the `/stage-1` shell: `Predictor` (All | regex | LLM) and `Confidence` (All | high | medium | low). URL contract `?predictor=(regex|llm_2nd_pass)` and `?confidence=(high|medium|low)`. Default view stays unfiltered (D-06). Filters apply to the predicted-row feed; the `?sub=pending` candidate-rule branch is unaffected.

Stage 1 hard-separation lock honored end-to-end: chip strip imports only `_shell/ChipStrip` (pure presentation) and writes `?predictor=` / `?confidence=` URL params. loadPageData filters `pipeline_events.stage=1` rows by the denormalized `decision_details->>predictor` (shipped by Plan 02 Task 3) and `decision_details->>llm_confidence`. No `swarm_intents` reference, no Stage 3 crossings.

File: [`web/app/(dashboard)/automations/[swarm]/stage-1/predictor-confidence-chip-strip.tsx`](../../../web/app/(dashboard)/automations/[swarm]/stage-1/predictor-confidence-chip-strip.tsx)

- URL contract:
  - `"All"` predictor chip → `qs.delete("predictor")`.
  - `"regex"` predictor chip → `qs.set("predictor", "regex")`.
  - `"LLM"` predictor chip → `qs.set("predictor", "llm_2nd_pass")`.
  - `"All"` confidence chip → `qs.delete("confidence")`.
  - `"high"|"medium"|"low"` → `qs.set("confidence", "<value>")`.
  - Axes are orthogonal — both can be active simultaneously.
- Active-state derivation: unknown URL values fall through to `"all"` (no chip stuck active on garbage input).
