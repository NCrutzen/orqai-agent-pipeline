```typescript
import dagre from "@dagrejs/dagre";

```

Constants: `NODE_WIDTH = 200`, `NODE_HEIGHT = 100`.

Export function `getLayoutedElements(nodes: Node[], edges: Edge[], direction: "TB" | "LR" = "TB"): { nodes: Node[]; edges: Edge[] }`:
- Create new `dagre.graphlib.Graph()`, set default edge label `() => ({})`, set graph options `{ rankdir: direction, nodesep: 60, ranksep: 80 }`.
- For each node: `g.setNode(node.id, { width: NODE_WIDTH, height: NODE_HEIGHT })`.
- For each edge: `g.setEdge(edge.source, edge.target)`.
- Run `dagre.layout(g)`.
- Map nodes: get `g.node(node.id)`, set `position: { x: pos.x - NODE_WIDTH / 2, y: pos.y - NODE_HEIGHT / 2 }`.
- Return `{ nodes: layoutedNodes, edges }`.

**Inner component `SwarmGraphInner`** (needs `useReactFlow`):

State:
- `[nodes, setNodes, onNodesChange] = useNodesState([])` -- React Flow managed nodes
- `[edges, setEdges, onEdgesChange] = useEdgesState([])` -- React Flow managed edges
- `selectedAgent: AgentNodeData | null` for the detail panel
- `detailOpen: boolean` for the Sheet open state
- `hasCompleteCelebrated: boolean` ref to prevent double-firing confetti

Initialization (useEffect on `steps` and `runStatus`):
- Call `mapPipelineToGraph(steps, runStatus)` to get initial nodes/edges
- Call `getLayoutedElements(graphData.nodes, graphData.edges, "TB")` for positioned nodes
- Set nodes and edges state
- Call `fitView({ padding: 0.2, duration: 500 })` after a short timeout (100ms) to let React Flow render

Broadcast subscription (useBroadcast):
- Subscribe to channel `run:${runId}`, event `step-update`
- On message: update node statuses based on `payload.stepName` and `payload.status`
  - Use `setNodes` with updater function to map through nodes and update data.status
  - If `payload.runStatus === "complete"`: trigger celebration sequence
  - Update edge animation: set `data.animated = true` for edges connected to running nodes, `false` for others

Progressive node appearance:
- When new nodes appear (architect step completes and provides agent definitions), add them one at a time with a 200ms stagger
- Each new node starts with CSS opacity 0, then animates to 1 via the agent-node entrance animation
- After each addition, recompute dagre layout and call `fitView({ padding: 0.2, duration: 500 })`

Celebration sequence (when run becomes "complete"):
- Guard with `hasCompleteCelebrated.current` ref to fire only once
- Call `confetti({ angle: 60, spread: 55, origin: { x: 0, y: 0.7 }, particleCount: 100, zIndex: 9999 })`
- Call `confetti({ angle: 120, spread: 55, origin: { x: 1, y: 0.7 }, particleCount: 100, zIndex: 9999 })`
- Set a celebration overlay state to show "Pipeline Complete" text and summary stats
- Show overlay with: `bg-background/80 backdrop-blur-sm` centered text, text-2xl semibold "Pipeline Complete", text-sm muted "{n} agents designed" and "Completed in {duration}"
- Auto-dismiss after 3000ms (fade out over 400ms)

Score animation trigger:
- When celebration fires, update all nodes with their scores
- The AgentNode component handles the count-up animation internally

Node click handler:
- `onNodeClick`: set `selectedAgent` from the clicked node's data, set `detailOpen = true`

Empty state:
- When `nodes.length === 0` and `runStatus === "pending"`: show centered text "Waiting for pipeline to start" / "The agent graph will appear as the pipeline designs your swarm." (per UI-SPEC copywriting)
- When `nodes.length === 0` and `runStatus === "running"`: show Skeleton placeholder with "Building swarm..." text

Render:
```tsx
<div className="relative h-full w-full">
```
