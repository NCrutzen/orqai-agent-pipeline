**Step 2: Create `web/lib/supabase/broadcast.ts`**

This file contains BOTH server-side emission helpers AND a client-side React hook.

Server-side helpers (used from Inngest pipeline function):
- `broadcastStepUpdate(runId: string, payload: StepUpdatePayload)` -- sends to channel `run:{runId}` with event name `step-update`. Payload type:
  ```typescript
  export interface StepUpdatePayload {
    stepName: string;
    status: "pending" | "running" | "complete" | "failed" | "skipped";
    displayName: string;
    durationMs?: number;
    stepsCompleted?: number;
    runStatus?: string;
    log?: string;
  }
  ```
  Implementation: create admin client, create channel `run:${runId}`, call `channel.send({ type: "broadcast", event: "step-update", payload })`, then `admin.removeChannel(channel)`.

- `broadcastRunUpdate(runId: string, payload: RunUpdatePayload)` -- sends to channel `runs:live` with event name `run-update`. Payload type:
  ```typescript
  export interface RunUpdatePayload {
    runId: string;
    status: string;
    stepsCompleted: number;
    agentCount?: number;
  }
  ```
  Implementation: create admin client, create channel `runs:live`, call `channel.send({ type: "broadcast", event: "run-update", payload: { runId, ...payload } })`, then `admin.removeChannel(channel)`.

Client-side hook:
- `useBroadcast<T>(channelName: string, eventName: string, onMessage: (payload: T) => void)` -- subscribes to a Broadcast channel. Uses `useRef` for the callback to avoid re-subscribing on callback change. Uses `useEffect` with cleanup that calls `supabase.removeChannel(channel)`. Import `createClient` from `@/lib/supabase/client`.

Export all types: `StepUpdatePayload`, `RunUpdatePayload`.

**Step 3: Create `web/lib/pipeline/graph-mapper.ts`**

This utility maps pipeline step results to React Flow graph data.

Types to export:
```typescript
export interface AgentNodeData {
  name: string;
  role: string;
  model: string;
  toolCount: number;
  tools: string[];
  status: "idle" | "running" | "complete" | "failed";
  score?: number;
  description?: string;
  instructions?: string;
}

export interface GraphData {
  nodes: Array<{ id: string; type: "agent"; position: { x: number; y: number }; data: AgentNodeData }>;
  edges: Array<{ id: string; source: string; target: string; type: "animated"; data: { animated: boolean } }>;
}
```

Functions to export:
- `parseArchitectOutput(output: string): AgentNodeData[]` -- Parses the architect step's text result to extract agent definitions. The architect output is freeform markdown/text. Use regex patterns to extract agent names, roles, and relationships. Look for patterns like agent name headers, role descriptions, model mentions, and tool lists. Return array of AgentNodeData with status "idle" and no score. If parsing fails or output is empty, return an empty array.

- `mapPipelineToGraph(steps: PipelineStep[], runStatus: string): GraphData` -- Takes pipeline_steps array and run status. Finds the architect step's result.output. Calls parseArchitectOutput. Creates nodes for each agent. Creates edges from the first node (orchestrator/main agent) to all other nodes (hub-spoke pattern). Node positions are all {x:0, y:0} (dagre layout computes real positions later). Sets node status based on runStatus: if run is "complete" all nodes are "complete", if "running" the orchestrator node is "running", if "failed" mark the orchestrator as "failed".

- `mapStepToNodeStatus(stepName: string, stepStatus: string, nodes: GraphData["nodes"]): GraphData["nodes"]` -- Maps a pipeline step update to graph node status changes. The architect step affects all nodes (they appear). Subsequent steps map to the most relevant node by name similarity.
