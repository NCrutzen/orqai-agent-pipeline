## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (gap-1), inline padding, badge internal spacing |
| sm | 8px | Compact element spacing (gap-2), label-to-input gaps, graph node internal gaps |
| md | 16px | Default element spacing (gap-4), card internal padding (px-4), graph node padding |
| lg | 24px | Section padding (p-6), page content padding, Sheet drawer content padding |
| xl | 32px | Layout gaps between major sections (mt-8) |
| 2xl | 48px | Empty state vertical padding (py-12) |
| 3xl | 64px | Page-level top margin for empty states (mt-16) |

Exceptions: Graph node minimum width is 200px (NODE_WIDTH). Graph node height is 100px (NODE_HEIGHT). Dagre layout uses 60px node separation (nodesep) and 80px rank separation (ranksep). React Flow viewport control buttons use 44px touch targets. Timeline Sheet drawer is 400px wide.

## Interaction Patterns

### Real-Time Updates (Supabase Broadcast)

Replaces Phase 35's 5-second `router.refresh()` polling entirely.

- **Run detail page**: Subscribes to `run:{runId}` channel. Receives `step-update` events with stepName, status, displayName, durationMs, stepsCompleted, runStatus, log.
- **Run list page**: Subscribes to `runs:live` channel. Receives `run-update` events with runId, status, stepsCompleted, agentCount.
- **Channel cleanup**: `supabase.removeChannel(channel)` in useEffect cleanup. Use ref to track channel instance.
- **Server-side emission**: `broadcastStepUpdate()` and `broadcastRunUpdate()` called from Inngest pipeline function after each step status transition.
- **Initial state**: Server component provides initial data. Client component initializes from server props, then overlays Broadcast updates. No flash of stale state.

### Auto-Scroll Behavior

- Timeline drawer auto-scrolls to the currently active (running) step by default.
- If user manually scrolls away from the active step (scroll position differs from active step position by more than 100px), show the floating "Jump to active step" button.
- Clicking the button smooth-scrolls to the active step and hides the button.
- Auto-scroll resumes after the user clicks the button.

### Graph Interaction

- **Zoom**: Mouse wheel or pinch gesture. ReactFlow built-in.
- **Pan**: Click and drag on canvas background. ReactFlow built-in.
- **Drag nodes**: Click and drag on node. Positions reset on page reload (no persistence).
- **Hover node**: Show Tooltip with role description, model name, and tool count. Delay: 300ms. Position: above node.
- **Click node**: Open AgentDetailPanel Sheet from the right side. Sheet closes on Escape, click outside, or X button.
- **fitView**: Called on initial render and when new nodes appear (progressive build). Uses `padding: 0.2` and `duration: 500` for smooth viewport adjustment.

### Progressive Node Appearance

- Graph starts empty when pipeline begins.
- As the architect step outputs agent definitions (parsed from `pipeline_steps.result`), nodes appear one at a time.
- Each node entrance: opacity 0 to 1, scale 0.95 to 1 over 400ms with ease-out easing.
- After each node appears, `fitView()` is called to adjust the viewport to include the new node.
- Dagre layout recomputes on each new node addition.

### Animated Edge Flow

- Edges connected to a currently running node show a moving dot along the path.
- Dot: `circle r="4"` filled with `hsl(var(--primary))`.
- Animation: SVG `animateMotion` with `dur="1.5s"` and `repeatCount="indefinite"`.
- Edges to non-running nodes use static stroke with `stroke-muted-foreground/40`.
- When a node transitions from running to complete, the animated dot stops and the edge becomes static with `stroke-green-500/40`.

### Score Count-Up Animation

- When pipeline completes, scores animate from 0 to final value over 1200ms.
- Easing: ease-out (fast start, slow finish).
- Scores appear on each agent node in the graph.
- Triggered only when run status transitions to "complete" via Broadcast event.
- If no score data is available, node shows a green checkmark instead.

### Completion Celebration

- Triggered when pipeline status transitions to "complete" via Broadcast.
- Sequence: (1) confetti burst from both sides (200 particles, origin y: 0.7), (2) "Pipeline Complete" text fades in center-screen over 600ms, (3) summary stats fade in below over 400ms, (4) celebration overlay auto-dismisses after 3 seconds, settling into normal complete state.
- Implementation: `canvas-confetti` library (6KB, no dependencies).
- Confetti config: two bursts from `origin.x: 0` (angle: 60, spread: 55) and `origin.x: 1` (angle: 120, spread: 55). `zIndex: 9999`.
- Celebration text: centered overlay div with `bg-background/80 backdrop-blur-sm`, fading out after 3 seconds.

### Broadcast Disconnection

- If Broadcast connection drops, show a subtle "Live updates paused -- Reconnecting..." indicator at bottom-left of the graph area.
- Supabase client handles automatic reconnection. Indicator disappears when connection restores.
- No user action required.

## Animation Timing Reference

| Animation | Duration | Easing | Trigger |
|-----------|----------|--------|---------|
| Node entrance | 400ms | ease-out | New agent parsed from architect output |
| Node pulse (running) | continuous | CSS animate-pulse (2s) | Node status becomes "running" |
| Edge dot motion | 1.5s per loop | linear (SVG animateMotion) | Edge source or target node is "running" |
| fitView adjustment | 500ms | ease-in-out (ReactFlow default) | New node added, initial render |
| Score count-up | 1200ms | ease-out | Pipeline status becomes "complete" |
| Confetti burst | ~2500ms (natural decay) | physics-based (canvas-confetti) | Pipeline status becomes "complete" |
| Celebration text fade-in | 600ms | ease-in | After confetti fires |
| Celebration summary fade-in | 400ms | ease-in | 200ms after celebration text |
| Celebration auto-dismiss | 3000ms hold + 400ms fade-out | ease-out | After celebration text appears |
| Sheet open/close | 300ms | ease-in-out (shadcn Sheet default) | Timeline button click, node click |
| Tooltip appear | 300ms delay | CSS transition | Node hover |
