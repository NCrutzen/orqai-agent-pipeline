# Phase 82.5 — Feedback UX Deepdive

## Why this phase exists

Phase 82.4 shipped the durable infrastructure for operator feedback capture (table, POST API, StageFeedbackPanel, override dual-write, snapshot cron). First live testing surfaced UX gaps: notes invisible after Save, no read-back of prior feedback, no row-strip indicators, override + note coupling unclear, multi-stage semantics undefined. Phase 82.5 closes the UX loop so V9.0 synthesis ingests a clean corpus AND operators have a trustworthy review surface.

## Goal

After 82.5 ships:
- An operator can see what they (and others) previously wrote on any row, at any stage.
- The Bulk Review row strip shows reviewed-vs-unreviewed status at a glance.
- The override flow makes it visually obvious that a textarea note is being attached.
- "Save" vs "Confirm" semantics are unambiguous through microcopy + visual treatment.
- Keyboard shortcuts compose with the new panel (Tab order, n=skip, ⏎=approve still work).
- The panel works at narrow viewports (≤1280px sidebar).

## In scope (locked surface area)

### Plan 01 — Read-back of saved notes
On mount, the panel fetches the latest `email_feedback` row for `(email_id, stage, operator_id?)` and pre-populates the textarea + verdict chip state. Cross-operator history is shown as read-only "what others said" under the textarea.

### Plan 02 — Row-strip review indicators
RowList shows a small chip per row when feedback exists: `✓` for confirm, `!` for override, `~` for unclear. Bulk Review operators can spot at a glance what's been touched without expanding the sidebar.

### Plan 03 — Override + note coupling
When the operator opens the override picker (Stage 1 dropdown / Stage 3 reclassify), the panel auto-focuses the textarea + shows a tooltip "Your note will be attached to this override". On submit, success toast says "Override + note saved".

### Plan 04 — Save vs Confirm clarity
Microcopy + treatment update:
- Save: "Save note (revisit later)" — blue, persistent state
- Confirm: "✓ Confirm — looks correct" — green, closes expander
Hover/focus states differentiate. Empty-textarea confirm shows "Confirm without note?" guard.

### Plan 05 — Multi-stage flag-on-override
When an operator overrides Stage N, prompt: "Mark Stage N-1 / N-2 as needing review too?" via an inline chip strip. Default OFF (no surprise writes). Operator clicks → batch writes verdict=`unclear` rows for the upstream stages.

### Plan 06 — Keyboard shortcuts integration
- `n` skips to next row (existing, ensure still works with focused textarea)
- `⏎` approves (existing, debounce so it doesn't fire while typing)
- `⌘+s` saves the note (new)
- `⌘+⏎` confirms (new)
- `Esc` closes expander (new)
Tab order: row → expand → textarea → Save → Confirm → next row.

### Plan 07 — Narrow-viewport pass
Sidebar feedback panel must remain usable at 1280px viewport. Currently the textarea + buttons may overflow. Pass through responsive treatment without breaking the desktop layout (≥1440px).

## Out of scope (defer to 82.6 / V9.0)

- Synthesis layer reading from `email_feedback` (V9.0).
- Cross-row bulk operations on feedback (V9.0 if needed).
- Feedback analytics dashboard (V9.0).
- Permission model for read-back of cross-operator notes (assume all operators see everything for now).

## Open questions

- **OQ-1:** Should the read-back show ALL prior verdicts (history) or just the latest? — recommend: latest + collapsed "show N earlier" below.
- **OQ-2:** Row-strip chip color tokens — do they conflict with existing stage badges? UI audit needed.
- **OQ-3:** Multi-stage flag-on-override (Plan 05) — does it cascade automatically when V9.0 ships, or stay manual? Recommend: manual now, automated when synthesis is online.
- **OQ-4:** Should "Confirm without note" be a hard guard or soft warning? — recommend: soft (just a confirm dialog), don't block flow.

## Success criteria (verifiable at phase exit)

- [ ] Re-opening a previously-saved row pre-populates the textarea.
- [ ] Bulk Review row strip shows reviewed-status chip on at least 95% of rows with prior feedback.
- [ ] Override picker auto-focuses the textarea on open.
- [ ] Save and Confirm buttons have differentiated microcopy + treatment.
- [ ] All keyboard shortcuts work without breaking on a focused textarea.
- [ ] Sidebar feedback panel renders without overflow at 1280px.
- [ ] Existing 82.4 UAT items 1, 2, 4 (#3 blocked on Inngest env config) re-pass.
