## Implementation Notes

### Task 1 — stage-step.tsx
- `StageFeedbackPanel` now receives `initialReadBack={stage.feedbackReadBack ?? null}`.
- New `useEffect` keyed on `[stage.emailId, stage.n]` seeds `proseNotes` from `feedbackReadBack.own_latest.prose_notes` when the local state is the empty default. The panel itself never touches its `value` prop — parent owns seeding (Plan 03 deliberately deferred this).
- R4 helper line `⤓ Override + note save together` added under the dirty-state widget render block. CONTEXT D-1: two copies total (Plan 03 panel + Plan 05 picker), no third.
- All three existing `fireFeedback` invocations preserved unchanged (`fireOverrideFeedback`, ok-branch override-stage click, skipped-branch override-stage click).

### Task 2 — pipeline-flow.tsx
- `StageData` extended with optional `feedbackReadBack?: FeedbackReadBack` (threaded by detail-pane).
- `PipelineFlowProps` gains three optional fields: `futureRange`, `futureExpanded`, `onToggleFuture`. When `futureRange == null`, the render path collapses back to the existing behavior (no regression).
- When collapsed: render stages outside the future range as today, then render a single `<li data-testid="future-pill">` at `s.n === futureRange.startN` with the literal copy `Stages N-M haven't run yet · expand to override pre-emptively`. Other stages in the range return null.
- When expanded: render every stage inline, plus a "Collapse future stages" affordance with `data-testid="future-pill-collapse"`.
- Inline V7 styles (dashed `--v7-border`, `--v7-panel-2` bg, `--v7-text-muted` text, `var(--v7-radius-pill)`).

### Task 3 — detail-pane.tsx
- `feedbackMap` was already declared on `UnifiedDetailPaneProps` from Plan 06 forward-compat work; this plan replaces the no-op (props destructured but unused) with real consumer wiring through `DetailPaneInner`.
- `stagesData` useMemo populates `feedbackReadBack: feedbackMap?.[n]` for Stages 0–3 (Stage 4 omitted per 82.3 out_of_scope). Dep array extended with `feedbackMap` and `row.id`.
- New `futureExpanded` state, reset to `false` via a `useEffect` on `[row.id]`.
- `futureRange` computed by walking backwards over `stagesData[4..0]` while `state === 'skipped'`; requires `endN - startN >= 1` (≥2 stages) to materialise.
- Morph button replaces the static `Approve`:
  - **Approve mode** (`!anyDirty`): label `✓ Approve verdicts that ran (Stages 0+N)`; click does `Promise.all` POSTs to `/api/automations/debtor-email/feedback` with `verdict: "confirm"` per ok stage.
  - **Override mode** (`anyDirty`): label `✓ Submit override (Stage N)` (or `Submit overrides (Stages N, M)` for multi); click dispatches `KEYBOARD_EVENTS.approve` ONLY — no footer POST. Pattern F (PATTERNS.md): stage-step's existing per-stage `fireFeedback` dual-write is the sole `email_feedback` writer on override. Adding a footer POST would create 2× rows per dirty stage per click.
  - `data-testid="detail-pane-primary"` + `data-mode={anyDirty ? "override" : "approve"}` for verifier.
- Reject + Skip footer buttons untouched; keyboard shortcuts untouched.
