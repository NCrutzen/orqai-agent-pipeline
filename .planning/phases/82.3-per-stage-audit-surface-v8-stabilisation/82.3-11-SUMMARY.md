## Deviations from Plan

The original Plan 11 Task 2 envisioned loading `agent_runs` + `automation_runs` server-side for the selected row in each stage page. For stage-1 the shallow shape uses `data.selectedTimeline` (already loaded server-side). For stage-3 and stage-4 the shells receive empty inputs (`timeline: [], agentRuns: [], automationRun: null`) — matching the existing pattern in stage-0 and stage-2 that pre-existed this plan. Audit panels render empty-state copy when inputs are missing, which is the documented expected behavior for historical rows.

Adding per-row data loading for stage-1/3/4 selected rows is a meaningful follow-up but does not block Plan 11's must-haves (the wiring, the type plumbing, the acceptance grep). Captured for V9.0 / Phase 82.4 prose-notes capture (which mounts inside the popup 82.3 ships).
