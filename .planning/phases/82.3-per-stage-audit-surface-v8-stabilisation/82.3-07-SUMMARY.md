# Phase 82.3 Plan 07: Wire StageDetailExpander into stage-step Summary

Generic Collapsible wrapper (`StageDetailExpander`) plus a 7-line insertion in `stage-step.tsx` that renders the expander between the `Current:` line and the override control area for Stages 0-3 only. Stage 4 is excluded.

## What Built

- **`StageDetailExpander`** (`web/components/automations/bulk-review/audit/StageDetailExpander.tsx`) — `"use client"` Collapsible wrapper. Props `{ stage: 0|1|2|3, children: ReactNode }`. Default-closed; trigger flips between `Show details` (ChevronDown, muted) and `Hide details` (ChevronUp, brand-primary). 11px uppercase 600 label per UI-SPEC. `data-testid="stage-detail-expander-{stage}"` on the Collapsible root.
- **`stage-step.tsx`** — added one import and a guarded JSX block (`stage.auditDetails != null && stage.n !== 4`) between the existing Current-value `<div>` and the `{/* Control area */}` `<div>`. Override controls (`ok` / `dirty` / `skipped` branches) untouched.

## Tests

- 5 cases for `StageDetailExpander`: default state, click-to-open, click-to-close, keyboard Enter, testid presence — all pass.
- 5 cases for `stage-step` wiring: Stage 1 renders, Stages 0/2/3 render, Stage 4 NEVER renders, missing `auditDetails` → no expander, override-stage button preserved when `state==='ok'` — all pass.

## Decisions Made

- Compile-time + runtime Stage 4 exclusion (typed prop union narrows the call site, `n !== 4` guard catches any caller that bypasses the type via cast).
- Uncontrolled internal `open` state — operator may open multiple stage expanders simultaneously to compare (per UI-SPEC §Interaction Contract).
