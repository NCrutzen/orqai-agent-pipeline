# Phase 82.3 Plan 05: Stage 2 Evidence Panel + ScreenshotThumb Summary

**One-liner:** Built the Stage 2 evidence renderer (identifier source / confidence / top-3 candidates / before+after iController screenshots) plus a `ScreenshotThumb` client component with abortable signed-URL fetch, 200ms shared cache, click-to-inline-expand, and locked error fallback — both fully covered by vitest TDD.

## What Was Built

### `ScreenshotThumb` (`web/components/automations/bulk-review/audit/ScreenshotThumb.tsx`)
- `"use client"` boundary on line 1.
- Props: `path: string`, `label: string`.
- On mount fetches `/api/automations/audit/signed-url?path={encodeURIComponent(path)}` via `AbortController` (cleanup on unmount).
- States: `loading` → `<Skeleton aria-label="Loading screenshot…" />` at 128×200; `error` → muted locked copy `Screenshot unavailable. Refresh to retry.` + `<button>Refresh</button>` re-runs the fetch; `success` → `<img>` at `max-height: 128px`, click toggles a second `<img data-testid="screenshot-full-{label}">` at `width: 100%` immediately below (NO modal).
- Module-level `Map<string, { url, cachedAt }>` with `SIGNED_URL_TTL_MS = 200` shared across all instances — second mount with the same path within 200ms reuses the URL with zero network calls.

### `Stage2EvidencePanel` (`web/components/automations/bulk-review/audit/Stage2EvidencePanel.tsx`)
- Pure / server-safe (the client boundary lives inside ScreenshotThumb).
- EVIDENCE block: identifier-source chip (chip-pink for normal sources, destructive `#ef4444` chip for `"unresolved"`), confidence chip (lime/amber/muted), candidate list rendered as `<ul>` with mono `account_id` — name (`score.toFixed(2)`).
- `identifier_source === "unresolved"` → renders locked copy `Customer could not be resolved. No candidates returned.` and **skips** the candidate table even when `top_candidates` is non-empty.
- `top_candidates.slice(0, 3)` — extras silently truncated.
- SCREENSHOTS block: side-by-side `<ScreenshotThumb>` for `before` and `after`; both null → locked copy `No iController capture for this run.` (block header preserved); only-one-null → present thumb + muted `—` for the missing side.
- All-null payload → only the top-level muted-italic `No evidence captured for this stage.`
- `data-testid="stage2-raw-json-slot"` placeholder for Plan 09.
