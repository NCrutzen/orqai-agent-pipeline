# Phase 82.3 — UI Design Contract

> Read-only per-stage audit expanders added to the existing Bulk Review detail-pane sidebar. **Does not redesign** the existing sidebar — layers evidence onto Stages 0–3 entries. Override controls (dropdowns, `override stage` links, Approve/Reject/Skip) stay untouched.

Source decisions: CONTEXT.md (locked scope), Sketch Findings skill (Phase 76 — "inline-expand editors, collapsible sub-sections. No drawers, no modals for high-frequency actions").

## Color

V7 OKLCH palette (locked from `web/app/globals.css [data-theme="dark"]`):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0c1117` (`--bg`) | Detail-pane background — unchanged |
| Secondary (30%) | `#151d29` (`--panel`) | Expander panel background — visually nested one level deeper than the stage row, achieved with the existing `--panel` token and a 1px `--border` top divider |
| Accent (10%) | `#ff6a34` (brand-primary orange) | Expander chevron in `open` state ONLY; "Show raw JSON" link in hover state ONLY |
| Destructive | `#ef4444` | Reserved for Stage 0 injection-detected verdict chip and Stage 2 unresolved chip |

Accent reserved for:
- Expander chevron icon when the collapsible is open
- "Show raw JSON" sub-toggle link on hover/focus

Accent is **NOT** used for: regular text, evidence chips, screenshot borders, loading states, "didn't run" labels. Those use neutral `--text-muted` / `--border`.

Semantic categorical chip colors (existing V7 tokens, used for evidence badges):
- Regex match label: teal `--chip-teal`
- Confidence high: lime `--chip-lime`
- Confidence medium: amber `--chip-amber`
- Confidence low: muted `--text-muted`
- Predictor source (regex vs LLM): blue `--chip-blue`
- Identifier source layer (thread/sender/identifier): pink `--chip-pink`

## Copywriting Contract

This phase is read-only, so there is no primary CTA. The "interaction" is expand/collapse + the existing Approve/Reject/Skip footer (untouched). Copy contract covers expander affordance, empty/skipped/loading states, and the data-quality fix.

| Element | Copy |
|---------|------|
| Expander affordance (collapsed) | `Show details` (with chevron-down icon, 11px uppercase 600) |
| Expander affordance (open) | `Hide details` (with chevron-up icon, accent color) |
| Sub-toggle (raw JSON closed) | `Show raw JSON` (mono 12px, muted) |
| Sub-toggle (raw JSON open) | `Hide raw JSON` |
| Section header — reasoning | `REASONING` (11px uppercase 600) |
| Section header — evidence | `EVIDENCE` |
| Section header — screenshots | `SCREENSHOTS` |
| Empty state — no evidence captured (Stage 0/1) | `No evidence captured for this stage.` (muted, italic) |
| Empty state — Stage 2 unresolved | `Customer could not be resolved. No candidates returned.` (muted) |
| Empty state — Stage 2 no screenshots | `No iController capture for this run.` (muted) |
| Empty state — Stage 3 no ranked list | `No ranked intents returned by coordinator.` (muted) |
| Skipped stage | `Stage didn't run — earlier stage short-circuited the pipeline.` (muted, italic) |
| Loading state (signed URL refresh) | `Loading screenshot…` (skeleton shimmer block at thumbnail dimensions, no text label visible — copy is `aria-label` only) |
| Error state (signed URL 401/expired/404) | `Screenshot unavailable. Refresh to retry.` (muted, with inline `Refresh` button) |
| Swarm-not-implemented (sales-email on Stages 2/3) | `Not yet implemented for this swarm.` (muted, italic) |
| Data-quality fix — missing sender_name | Render `(unknown sender)` (muted, italic) instead of literal `"Planning"` fallback string |
| Data-quality fix — missing subject | Render `(no subject)` (muted, italic) — current literal string is correct, the bug is that it leaks for rows that DO have a subject; fix is in the row mapper |

**No destructive actions in this phase.** Expand/collapse and signed URL refresh are non-destructive.

## Interaction Contract

| Element | Behaviour |
|---------|-----------|
| Expander placement | **Inline, below the existing stage row, above the next stage row.** Pushes subsequent stages down. Chosen over side-popover for: scroll-position predictability, consistency with Phase 76 "inline-expand editors" decision, and screen-reader linearity. Locked per sketch-findings skill: "No drawers, no modals for high-frequency actions." |
| Default state | Collapsed. Operator clicks `Show details` to expand. State is per-stage, not persisted across email rows. |
| Multiple expanders open simultaneously | **Allowed.** Operator may open Stage 1 + Stage 3 evidence at once to compare. No accordion-style mutual-exclusion. |
| Screenshot rendering | **Inline thumbnail, max-height 128px, native browser aspect ratio.** Click thumbnail toggles an **inline full-width render** below the thumbnail row (still inside the expander — NOT a modal/lightbox). Second click collapses back to thumbnail. Rule from sketch-findings: no modals for high-frequency actions. |
| Signed URL refresh | One Supabase Storage API call per `Show details` click for the Stage-2 screenshot pair. URLs are NOT prefetched on row open (would waste calls for rows the operator never expands). |
| Raw JSON sub-toggle | Inline collapsible code block (Geist Mono 12px, `--panel-deep` background, 1px `--border`, max-height 240px with internal scroll). NOT a modal. |
| Loading state | Skeleton shimmer at thumbnail dimensions (128px × inferred ratio). No spinner. Target: ≤500ms per acceptance criteria. |
| Error state | Inline muted text + `Refresh` button (text button, not primary action style). |
| Keyboard | `Tab` reaches `Show details` button; `Enter` / `Space` toggles. `Esc` collapses the currently-focused expander. No global Esc-closes-all. |
| Stage 4 entry | **Unchanged.** Renders existing "Stage didn't run — override to set" line. No expander affordance shown. |

## Component Inventory

New components introduced by 82.3 (all live under `web/components/automations/bulk-review/audit/`):

| Component | Purpose |
|-----------|---------|
| `StageDetailExpander` | Generic Collapsible wrapper. Props: `stage` (0–3), `verdict`, `reasoning`, `children`. Renders the `Show details` / `Hide details` toggle + uniform layout. |
| `Stage0EvidencePanel` | Renders regex patterns triggered (chip list), LLM injection verdict (chip + reasoning line), budget headroom (mono line). |
| `Stage1EvidencePanel` | Renders rule-fired chip (regex Pass-1 OR LLM Pass-2), confidence chip, predictor source chip. Degraded mode when no `agent_runs` row exists (Pass-1 short-circuit). |
| `Stage2EvidencePanel` | Renders identifier source chip, confidence chip, top-3 candidates (mono list with score), iController before/after screenshot pair via `ScreenshotThumb`. |
| `Stage3EvidencePanel` | Renders ranked intent list (intent label + confidence chip per row, top-ranked highlighted), coordinator one-line reasoning, why-selected summary. |
| `ScreenshotThumb` | 128px-max thumbnail with click-to-inline-expand. Handles signed-URL fetch on first open, loading skeleton, 401/404 fallback. |
| `RawJsonToggle` | Sub-collapsible showing raw `decision_details` / `tool_outputs` JSON. Geist Mono 12px, 240px max-height with scroll. |
| `StageSkippedLine` | One-line muted render for stages that didn't run. |

Existing components extended (not replaced):
- The Bulk Review detail-pane sidebar component (location to be traced during planning — likely `web/app/(dashboard)/automations/.../detail-pane.tsx`) gains a child slot per stage row for `<StageDetailExpander>`.
- The row mapper that produces `sender_name` / `subject` (likely `web/lib/automations/swarm-bridge/configs.ts`) fixed to never fall back to literal `"Planning"`.
