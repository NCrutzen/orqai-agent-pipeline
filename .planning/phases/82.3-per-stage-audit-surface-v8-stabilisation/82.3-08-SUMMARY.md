# Phase 82.3 Plan 08: Signed-URL Route Handler Summary

Server-side route handler `GET /api/automations/audit/signed-url?path=…` that mints 1h Supabase Storage signed URLs for the `automation-screenshots` bucket. Operator-auth gated, path-traversal hardened, service-role key stays server-side per CLAUDE.md "Infra secrets" rule.

## What Shipped

- **Route handler** (`web/app/api/automations/audit/signed-url/route.ts`, 80 lines)
  - 401 unauthenticated → 400 missing_path → 400 invalid_path → 500 storage_error → 200 `{ url, expires_at }`
  - Uses existing `createClient()` (server auth) + `createAdminClient()` (service-role storage)
  - Path validation: regex `^[a-zA-Z0-9_\-./]+$`, no `..`, no leading `/`, length ≤512
  - Bucket `automation-screenshots` and TTL `3600` are server-side constants
  - `Cache-Control: private, max-age=200` on 200 responses
- **Tests** (`__tests__/route.test.ts`, 8 cases, all passing)
  - happy path · missing path · `..` traversal · leading `/` · >512 chars · disallowed chars · 401 · 500 storage error
