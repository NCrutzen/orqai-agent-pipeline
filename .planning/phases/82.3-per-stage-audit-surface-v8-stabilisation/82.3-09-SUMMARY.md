# Phase 82.3 Plan 09: RawJsonToggle + stage-panel swap — Summary

One-liner: 20% escape-hatch sub-toggle (Geist Mono 12px, 240px scroll) wired into all 4 Stage evidence panels, replacing the slot placeholders Plans 03–06 left behind.

## What was built

- **`RawJsonToggle` component** (`web/components/automations/bulk-review/audit/RawJsonToggle.tsx`)
  - `"use client"`; Radix Collapsible primitives (same as `StageDetailExpander`).
  - Locked copy: `Show raw JSON` (closed, muted) / `Hide raw JSON` (open, brand-primary).
  - Trigger: Geist Mono 12px, brand-primary on hover/focus via Tailwind arbitrary variants.
  - Content: `<pre><code>` rendering `JSON.stringify(raw, null, 2)`, Geist Mono 12px, `max-height: 240px`, `overflow-y: auto`, `var(--v7-panel-2)` bg, 1px `var(--v7-border)`, 4px radius, `var(--space-3)` padding.
  - Default closed. testids: `raw-json-toggle` (root), `raw-json-content` (pre).
- **TDD tests** (5 cases): default closed, click reveals 2-space-indented JSON, empty `{}`, keyboard Enter toggles, root testid present.
- **Stage 0/1/2/3 panel swaps**: Each panel adds `import { RawJsonToggle } from "./RawJsonToggle"` and changes the slot from `<div data-testid="stage{n}-raw-json-slot" />` to a wrapper div with the same testid containing `<RawJsonToggle raw={payload.raw} />`. Mechanically identical across all 4 panels.
