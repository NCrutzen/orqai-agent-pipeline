# Phase 82.3 Plan 10: Display-quality fix for Bulk Review header

## Outcome

Operators no longer see the literal string `"Planning"` (an Outlook display-name artifact emitted by system mailboxes) rendered as the sender of a Bulk Review row. The detail-pane header now routes both sender and subject through pure helpers in `_shell/_lib/display-fallbacks.ts`:

- `displaySender(from_name, from_email)` — rejects the `"Planning"` sentinel, falls through to `from_email`, finally `"(unknown sender)"`.
- `displaySubject(subject)` — trims and falls back to `"(no subject)"`.
