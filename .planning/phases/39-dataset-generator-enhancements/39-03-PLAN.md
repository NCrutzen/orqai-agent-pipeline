    === FILE 2: orq-agent/agents/dataset-generator/resources/coverage-rules.md ===

    Title: `# Coverage Rules (DSET-03)`
    Intro paragraph: single-consumer reference for `orq-agent/agents/dataset-generator.md`. Rules are enforced BEFORE upload; violations BLOCK the upload with a remediation message.

    ## Rule 1: Minimum Count
    Every dimension value MUST appear in ≥2 datapoints. Prevents single-example values from dominating evaluator variance.

    ## Rule 2: Maximum Share
    No single dimension value may exceed 30% of total datapoints across its dimension. Prevents the dataset from collapsing to a dominant slice.

    ## Remediation Messages
    On violation, emit one of these messages verbatim (the "Coverage check failed:" prefix is the grep anchor):
    - "Coverage check failed: value 'X' appears in only 1 datapoint (need ≥2). Add datapoints or adjust dimensions."
    - "Coverage check failed: value 'X' dominates 42% (limit is 30%). Rebalance by adding datapoints for under-represented values."

    ## Worked Example
    Suppose dimension `user_persona: [novice, expert, non-native-speaker]` across 10 datapoints with distribution novice=7, expert=2, non-native-speaker=1:
    - Rule 1 fires: `non-native-speaker` count=1 → "Coverage check failed: value 'non-native-speaker' appears in only 1 datapoint (need ≥2)..."
    - Rule 2 fires: `novice` share=70% → "Coverage check failed: value 'novice' dominates 70% (limit is 30%)..."
    Fix: add 3 more `expert` and 3 more `non-native-speaker` datapoints; novice now 7/16=43% (still >30%) — either remove 4 novice OR add 10 more balancing datapoints.

    ## When Rules Are Skipped
    - `--mode promote-trace`: single datapoint by construction; rules do not apply.
    - `--mode flat` without dimensions: rules skip with warning "Coverage rules require dimension_values; flat-mode datasets are not coverage-checked."

    === FILE 3: orq-agent/agents/dataset-generator/resources/shapes.md ===

    Title: `# Dataset Shapes (DSET-05, DSET-06, DSET-07)`
    Intro paragraph: single-consumer reference for `orq-agent/agents/dataset-generator.md`. Every datapoint carries `category` + `dimension_values` + `shape`. `shape` selects which additional fields appear.

    ## Shape: single (DSET-05)
    Default flat eval pair.
    ```json
    {"category": "happy-path", "dimension_values": {"task_complexity": "simple", "user_persona": "novice"}, "shape": "single", "input": "How do I reset my password?", "expected_output": "To reset your password...", "pass_criteria": ["contains: step-by-step", "format: numbered list"]}
    ```

    ## Shape: multi-turn (DSET-06)
    Used for conversational agents. Adds `messages` (ordered turn list) and `perturbation_scenario` (named stress-test).
    ```json
    {"category": "adversarial", "dimension_values": {"task_complexity": "complex", "user_persona": "expert"}, "shape": "multi-turn", "perturbation_scenario": "topic-drift-then-return", "messages": [{"role": "user", "content": "Can you help me plan a trip to Tokyo?"}, {"role": "assistant", "content": "Sure — what dates?"}, {"role": "user", "content": "Actually forget that, what's the capital of Mongolia?"}, {"role": "user", "content": "OK back to Tokyo — I meant next March."}], "expected_output": "Agent returns to trip planning without confusion, confirms March dates.", "pass_criteria": ["semantic: resumes original task", "contains: March"]}
    ```
    Perturbation scenarios to draw from: `topic-drift-then-return`, `contradictory-followup`, `partial-context-gap`, `rapid-topic-switch`.

    ## Shape: rag (DSET-07)
    Used for agents with `query_knowledge_base` tool. Adds `expected_source_chunk_ids` so retrieval quality can be graded separately from answer quality.
    ```json
    {"category": "happy-path", "dimension_values": {"task_complexity": "simple"}, "shape": "rag", "input": "What is the return policy for electronics?", "expected_source_chunk_ids": ["kb_chunk_returns_electronics_001", "kb_chunk_returns_general_003"], "context": "Electronics may be returned within 30 days...", "expected_output": "Electronics can be returned within 30 days if unopened.", "pass_criteria": ["contains: 30 days", "cites: kb_chunk_returns_electronics_001"]}
    ```
    RAGAS-style evaluators (faithfulness, context_precision, answer_relevancy) consume both `context` and `expected_source_chunk_ids`.

    ## Shape Selection
    - Agent spec has no tools or plain Q&A → `single`.
    - Agent spec is a conversational assistant (multi-turn dialogue in description) → `multi-turn`.
    - Agent spec includes `query_knowledge_base` tool or references a KB → `rag`.
    - CLI override: `--shape single|multi-turn|rag`.
