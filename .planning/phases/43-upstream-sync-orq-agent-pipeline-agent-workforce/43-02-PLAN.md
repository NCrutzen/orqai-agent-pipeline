Purpose: Without automated change detection, upstream pipeline changes go unnoticed until they cause runtime failures. This closes the feedback loop between the two repos.

Output: Cron endpoint, manifest-sync logic, GitHub issue/PR creation, health dashboard Pipeline Sync section.

**Create `web/lib/pipeline/manifest-sync.ts`:**

This module performs the upstream sync check. Key exports:

1. `SyncChange` type:
```typescript
export interface SyncChange {
  path: string;
  tier: 0 | 1 | 2 | 3;
  description: string;
  oldSha?: string;
  newSha?: string;
  isNew: boolean;
  isDeleted: boolean;
}

export interface SyncResult {
  status: "synced" | "drift" | "breaking";
  changes: SyncChange[];
  checkedAt: string;
  error?: string;
  issueUrl?: string;
  prUrl?: string;
}
```

2. `getUpstreamState()` function:
   - Calls GitHub Trees API: `GET https://api.github.com/repos/NCrutzen/orqai-agent-pipeline/git/trees/main?recursive=1`
   - Uses `Authorization: Bearer ${process.env.GITHUB_TOKEN}` header
   - Returns `Map<string, string>` of path -> sha for all blobs
   - On failure: throws error with code `UPSTREAM_SYNC_FAILED`

3. `classifyChange(path, oldSha, newSha)` function implementing exact tier logic from RESEARCH.md:
   - Agent file (`orq-agent/agents/*.md`) new or deleted -> Tier 3
   - Agent file modified (SHA changed) -> Tier 2 (default; could be downgraded to Tier 0 with deeper analysis, but start conservative)
   - Reference/command/template file changed (`orq-agent/references/`, `orq-agent/commands/`, `orq-agent/templates/`) -> Tier 1
   - Everything else -> Tier 0

4. `performUpstreamSync()` function:
   - Reads current manifest via `import { pipelineManifest } from "./stages"`
   - Calls `getUpstreamState()` to get upstream file tree
   - Compares each manifest agent's `mdFile` path against upstream SHAs
   - Compares trackedPaths against upstream
   - Detects NEW agent files in `orq-agent/agents/` that are NOT in the manifest (Tier 3, REQ-6)
   - Detects DELETED agent files (in manifest but not upstream, Tier 3)
   - Classifies all changes
   - Determines overall status: "synced" (no changes), "drift" (Tier 1-2 only), "breaking" (any Tier 3)
   - Returns `SyncResult`

5. `createGitHubIssue(changes: SyncChange[])` function:
   - Creates issue in `NCrutzen/agent-workforce` repo (NOT the upstream repo)
   - Uses `POST https://api.github.com/repos/NCrutzen/agent-workforce/issues`
   - Title: `[Upstream Sync] {count} change(s) detected - Tier {maxTier}`
   - Body: markdown table listing each change with tier, path, description
   - Labels: `["upstream-sync", tier === 3 ? "breaking" : "review"]`
   - Returns issue URL

6. `createAutoUpdatePR(newAgents: SyncChange[])` function (for Tier 3 new agents):
   - Only called when new agents are detected (isNew === true)
   - Creates a branch `upstream-sync/add-{agent-name}-{date}`
   - Reads current manifest from main branch via GitHub API
   - Adds new agent entries to the agents array with:
     - `stepOrder: 0` (disabled by default, user must set order)
     - `sha` from upstream
     - `inputTags: ["useCase"]` (convention default)
     - `contextSources: null` (convention: all prior outputs)
     - `displayName` derived from filename: `architect.md` -> `"Architect"`, `kb-generator.md` -> `"KB Generator"`
   - Commits to branch, creates PR
   - PR title: `[Upstream Sync] Add new agent(s): {names}`
   - PR body: lists new agents with their file paths
   - Returns PR URL

**Create `web/app/api/cron/upstream-sync/route.ts`:**

```typescript
import type { NextRequest } from "next/server";
import { performUpstreamSync, createGitHubIssue, createAutoUpdatePR } from "@/lib/pipeline/manifest-sync";

export async function GET(request: NextRequest) {
  // Verify CRON_SECRET (Vercel sends this automatically for cron jobs)
  const authHeader = request.headers.get("authorization");
  if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
    return new Response("Unauthorized", { status: 401 });
  }

  try {
    const result = await performUpstreamSync();

    // Auto-create GitHub issue for Tier 2-3 changes
    const significantChanges = result.changes.filter(c => c.tier >= 2);
    if (significantChanges.length > 0) {
      result.issueUrl = await createGitHubIssue(significantChanges);
    }

    // Auto-create PR for new agents (Tier 3, isNew)
    const newAgents = result.changes.filter(c => c.tier === 3 && c.isNew);
    if (newAgents.length > 0) {
      result.prUrl = await createAutoUpdatePR(newAgents);
    }

    return Response.json(result);
  } catch (error) {
    return Response.json(
      { status: "error", error: (error as Error).message, checkedAt: new Date().toISOString() },
      { status: 500 }
    );
  }
}
```
