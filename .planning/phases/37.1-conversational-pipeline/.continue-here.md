<decisions_made>

- **Conversation agent over hardcoded logic**: User pushed back on string matching for confirm/feedback. One AI agent handles all intent understanding naturally in any language.
- **AI extraction over regex**: User pointed out regex is fragile for parsing free-form AI output. Haiku extracts agents as structured JSON.
- **Non-blocking narration**: User questioned why approval gates exist at every step. Removed them — pipeline runs straight through after discussion. Approval only for future deploy/test phases.
- **DB-based conversation history**: In-memory array lost on Inngest replays causing duplicate messages. Load from DB on each turn.
- **Reference files gap identified**: Pipeline .md files reference other .md files the AI can't read. Noted as HIGH priority known issue.
- **Inngest replay model**: Learned that Inngest replays the entire function on each waitForEvent resume. Code outside step.run() re-executes. Critical for streaming architecture.
</decisions_made>
