## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps inside a badge (icon ↔ label), highlight underline thickness |
| sm | 8px | Compact element spacing inside the new override-axis card and inside detail-pane sub-sections |
| md | 16px | Default padding inside new tab content (`px-4 py-3` row strip pattern preserved) |
| lg | 24px | Section padding between Safety Review header bar and first row |
| xl | 32px | Layout gaps between override-axis cards in the detail pane |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

**Exceptions:** none. All new surfaces inherit the existing review grid `gap-4` (16 px) and existing row strip `px-4 py-3` paddings.

### Accent reserved for (this phase)

The `--v7-brand-primary` accent is reserved for the existing primary CTA / selected-row left bar (already in use). In Phase 64 it gains EXACTLY ONE additional placement: the active state of the new "Safety Review" tab pill in the queue tree.

It is NOT used for: badges, inline icons, regex-pattern chips, cost numbers, axis-card borders, or budget-breach treatment.

### New semantic role assignments (additive)

This phase introduces no new color tokens. It adds three new placements of EXISTING semantic tokens:

| Surface | Token | Reasoning |
|---------|-------|-----------|
| `injection_suspected` row left-bar (3 px, parallel to existing selected-row bar pattern) | `--v7-amber` (`#ffb547`) on `--v7-amber-soft` row background | Amber means "needs human attention, not a failure" — matches `v7-tiny-dot.status-waiting` precedent. Distinguishes from a destructive deletion. |
| Regex-matched span highlight inside email body (Detail pane) | `background: var(--v7-amber-soft)`; underline `text-decoration: underline wavy var(--v7-amber)` | Amber-soft fill at 13 % opacity is already the established highlight density (see `--v7-amber-soft: rgba(255,181,71,0.13)`). |
| LLM verdict reason chip | `background: var(--v7-blue-soft)`; `color: var(--v7-blue)`; 12 px label | Blue == informational/model-output, never accent. Mirrors existing `--v7-blue-soft` chip treatments. |
| Cost-outlier axis 4 badge (row strip + detail) | `background: var(--v7-pink-soft)`; `color: var(--v7-pink)`; eyebrow label "AXIS 4 · COST OUTLIER" | Pink is the only V7 semantic role not yet allocated to a stage axis (axis 1 = brand-primary verdict, 2 = teal customer match, 3 = blue intent, 4 = pink cost). Locks the 4-axis color contract for downstream phases. |
| Budget-breach badge on halted runs (detail pane + Kanban card link) | `background: var(--v7-red-soft)` (computed `rgba(255,107,122,0.13)`, declared inline since no token exists today); `color: var(--v7-red)`; eyebrow label "BUDGET BREACH" + reason text | Red because the run halted — operator must act. Distinct from amber's "review me" connotation. |

**Note for executor:** if `--v7-red-soft` does not yet exist as a token, declare it inline on the badge component as `rgba(255,107,122,0.13)` matching the soft-token pattern used by `--v7-amber-soft` / `--v7-pink-soft`. Do NOT introduce a new global token in this phase.

## Interaction Contract

- **Safety Review tab activation:** clicking the tree node sets `?tab=safety&topic=safety_review` in the URL. URL is the source of truth (existing pattern). No client-only state.
- **Mark safe & reprocess:** optimistic — row vanishes from Safety Review immediately (existing `pendingRemovalIds` mechanism in `selection-context.tsx`); server emits `pipeline/email.received` with `safety_overridden=true`; if it fails, row re-appears with toast `Failed to reprocess — try again` (existing toast convention).
- **Dismiss:** optimistic remove + toast with 5 s undo (existing pattern).
- **Escalate:** writes a `swarm_jobs` row in the Kanban human-review lane and removes the row from Safety Review. No new escalation UI invented this phase (per CONTEXT D-11).
- **Cost-outlier badge:** pure read-time computation; no operator action attaches to it. The axis 4 card in the detail pane IS the operator surface, not the badge.
- **Budget-breach card link:** clicking the breach badge on a Kanban card opens the source `automation_run` in a new browser tab (`/automations/debtor-email/review?selected={run_id}`). This is the only cross-surface navigation introduced.

## Accessibility

- Amber and pink soft backgrounds at 13 % opacity meet the existing dark-theme contrast ratios already verified by V7 (≥ 4.5:1 on the `--v7-text` foreground). No new contrast work needed.
- The matched-span highlight uses BOTH color (amber-soft fill) AND non-color signal (wavy underline) — colorblind-safe per WCAG 1.4.1.
- All three action buttons have a visible 2 px focus-ring (`focus:ring-2 focus:ring-[var(--v7-brand-primary)]`) matching existing row-strip pattern.
- Cost outlier and budget-breach badges include the eyebrow label as plain text inside the pill (not aria-label-only) so screen readers receive the same signal as sighted users.
- Confirm-toast for Dismiss is keyboard-undoable via `Cmd-Z` within 5 s — already implemented in the existing toast layer; the Dismiss action wires into that contract, no new keymap.
