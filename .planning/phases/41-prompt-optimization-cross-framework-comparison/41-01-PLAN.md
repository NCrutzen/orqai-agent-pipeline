**Banner (verbatim):** `ORQ ► PROMPT OPTIMIZATION`

**Section 1 — Constraints (NEVER/ALWAYS):**
- NEVER mutate the original prompt in place — ALWAYS publish rewrites as a new orq.ai version preserving the original for rollback (POPT-04).
- NEVER rewrite `{{variable}}` placeholders — ALWAYS preserve them literally; scan for them before any rewrite begins (POPT-01).
- NEVER emit more than 5 suggestions per run — cap at 5 even if more guideline gaps exist (POPT-02).
- NEVER apply a rewrite without explicit user approval via AskUserQuestion after diff presentation (POPT-03).
- ALWAYS map every suggestion to exactly one of the 11 guideline anchors: role, task, stress, guidelines, output-format, tool-calling, reasoning, examples, unnecessary-content, variable-usage, recap (POPT-02).

**"Why these constraints:"** paragraph explaining rollback preservation, variable-substitution integrity, cognitive-load cap of 5 suggestions, approval-gate, and taxonomy grounding.

**Section 2 — When to use / When NOT to use:**
- Use: user wants to improve a single prompt against canonical prompting discipline before running an experiment.
- Don't use: user needs automatic scoring (defer to Phase 42 evaluator framework) or framework auto-detection (defer to Phase 37 observability).

**Section 3 — Companion Skills (directional handoffs):**
- → `/orq-agent:test` after new version is created (A/B validation)
- → `/orq-agent:compare-frameworks` for cross-framework benchmarking of the improved prompt
- ← `/orq-agent:trace-failure-analysis` (when failure taxonomy recommends prompt-fix)

**Section 4 — Done When (falsifiable):**
- [ ] Target prompt fetched; `{{variable}}` placeholders inventoried and preserved literally
- [ ] Up to 5 suggestions produced, each mapped to one of the 11 guideline anchors
- [ ] Diff presented to user; AskUserQuestion approval received before rewrite
- [ ] New version published on orq.ai via MCP `create_prompt_version` (or REST fallback); original preserved
- [ ] `/orq-agent:test` recommended for A/B validation

**Section 5 — Destructive Actions:**
- **Create new prompt version on orq.ai** — requires AskUserQuestion confirm after diff review. Original is preserved (rollback-safe).

**Section 6 — Steps (numbered ## Step N: with exact prefix so VALIDATION grep anchors pass):**

- ## Step 1: Fetch target prompt
  - Accept either inline text via arg OR `--prompt-key <key>` for orq.ai fetch via MCP `get_prompt` (REST fallback `GET /v2/prompts/{key}`).
  - Store as `ORIGINAL_PROMPT`.
- ## Step 2: Variable scan (POPT-01)
  - Regex: `\{\{[^}]+\}\}`
  - Capture every placeholder into `VARIABLES[]`; if any appears, emit a "Variables Preserved" block listing each one; the lint anchor phrase "variable-usage" MUST appear verbatim in this step's prose.
- ## Step 3: Analyze against 11 guidelines (POPT-02)
  - For each of the 11 anchors (role, task, stress, guidelines, output-format, tool-calling, reasoning, examples, unnecessary-content, variable-usage, recap), rate coverage Present / Partial / Missing.
  - Reference `orq-agent/commands/prompt-optimization/resources/11-guidelines.md` for per-category rubric.
- ## Step 4: Produce up to 5 suggestions
  - Emit in this exact format per suggestion:
    ```
    ### Suggestion N: [guideline-category]
    **Problem:** ...
    **Proposed change:** ...
    **Guideline anchor:** role | task | stress | guidelines | output-format | tool-calling | reasoning | examples | unnecessary-content | variable-usage | recap
    ```
  - Hard cap 5. If more gaps exist, note "Additional gaps truncated; re-run after applying top 5."
- ## Step 5: Rewrite draft + diff (POPT-03)
  - Compose rewritten prompt preserving every `{{variable}}` literally.
  - Emit side-by-side markdown diff (```diff fenced block).
  - Reference `orq-agent/commands/prompt-optimization/resources/rewrite-examples.md` for before/after patterns.
- ## Step 6: Approval gate
  - AskUserQuestion: "Apply rewrite and publish as new version on orq.ai? (original preserved)" with options yes / no / edit-first.
  - On "no" or "edit-first": STOP (no version created).
- ## Step 7: Publish new version (POPT-04)
  - MCP-first: `create_prompt_version` with parent = original version id; on error, surface verbatim + REST fallback curl: `POST /v2/prompts/{key}/versions` with `ORQ_API_KEY` header.
  - NEVER synthesize a success response if both fail.
- ## Step 8: Recommend A/B validation
  - Emit: "Run `/orq-agent:test --prompt-key <key> --version <new_id>` to A/B validate against the previous version."

**Section 7 — Anti-Patterns table (4+ rows):**
| Pattern | Do Instead |
|---------|-----------|
| Overwriting the original prompt | Publish as a new version; orq.ai keeps the prior one for rollback (POPT-04) |
| Rewriting `{{variable}}` placeholders | Preserve them literally; they are substituted at invocation time (POPT-01) |
| Emitting 8 suggestions to be thorough | Cap at 5 — cognitive load > coverage; user re-runs for more (POPT-02) |
| Applying the rewrite without AskUserQuestion | The diff must be reviewed; approval is mandatory (POPT-03) |
| Mapping a suggestion to "other" | Every suggestion maps to one of the 11 anchors; if it doesn't, drop the suggestion |
