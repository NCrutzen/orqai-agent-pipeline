None of substance. Two small implementation choices documented as decisions in the frontmatter:

1. **INTEG-03's "real helper" injection.** Plan 03 suggested `vi.doUnmock` + dynamic import. Vitest's hoisted `vi.mock` makes this brittle because the worker module already imported the mock at top-level.
