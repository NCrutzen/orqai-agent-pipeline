## Deviations from Plan

### Wave 1 fixture repair (Rule 1 auto-fix)

**Found during:** Task 1 verification — final assertion in `strip-quoted-history.test.ts` (`12k regression: ... delta_chars < -8000`) failed with `expected 0 to be less than -8000`.

**Issue:** The synthetic 18,124-byte `12k-token-real-sample.txt` fixture authored in Wave 1 did NOT insert a blank line before each of the 20 repeated `Van: KLANT <klant@voorbeeld.nl>` Outlook NL header blocks. As a result, `email-reply-parser` could not split the body into fragments at the `Van:` divider — the library's NL Outlook regex `/^\s*(Van\s?:.+\s?\n?\s*(\[|<).+(\]|>))/m` is matched against per-fragment starts, and fragments are split at blank lines. Since the synthetic fixture concatenated the operator's signature line `debiteuren@smeba.nl` directly with `Van: KLANT...` (no blank line between), the entire 18 KB body parsed as one fragment, no quoted block was detected, `delta_chars = 0`.

This contradicted the Wave 1 SUMMARY's claim that the fixture would satisfy `delta_chars < -8000`, but is consistent with the SUMMARY's own caveat: *"Wave 3 follow-up: Replace with real stranded-email body once Supabase access lands, to validate the `delta_chars < -8000` claim against the production shape (RESEARCH Assumption A2)."*

**Fix:** Inline node script inserted a blank line before each `Van: KLANT <` line that did not already have one. Fixture grew 18,124 → 18,144 bytes (20 newlines added). Re-tested with the parser directly: `delta_chars = -17800`, stripped body length 264 chars (operator preamble preserved). All other fixture-driven tests unaffected (their fixtures were already correctly shaped — the bug was only in the synthetic generator).

This is a Rule 1 auto-fix (the fixture didn't reflect real Outlook NL email shape — real Outlook always emits a blank line before the `Van:` reply header). It is also Rule 3 (blocked Task 1's GREEN contract). The fix preserves the spirit of the Wave 3 follow-up: the fixture is still synthetic, but now structurally matches production Outlook NL emails. Wave 3 may still substitute a real stranded-email body when Supabase access is available.
