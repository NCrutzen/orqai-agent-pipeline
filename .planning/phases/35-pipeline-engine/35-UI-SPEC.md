## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (gap-1), inline padding, badge internal spacing |
| sm | 8px | Compact element spacing (gap-2), label-to-input gaps |
| md | 16px | Default element spacing (gap-4), card internal padding (px-4) |
| lg | 24px | Section padding (p-6), page content padding |
| xl | 32px | Layout gaps between major sections (mt-8) |
| 2xl | 48px | Empty state vertical padding (py-12) |
| 3xl | 64px | Page-level top margin for empty states (mt-16) |

Exceptions: Run card uses 16px internal padding (p-4). Step timeline uses 24px bottom padding per step (pb-6) to create visual breathing room between timeline nodes.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (normal) | 1.43 (20px) | Run card descriptions, step log content, metadata text |
| Label | 14px (text-sm) | 600 (semibold) | 1.43 (20px) | Form labels, step display names, section headings |
| Heading | 24px (text-2xl) | 600 (semibold) | 1.33 (32px) | Page titles: "New Pipeline Run", "All Pipeline Runs", project name |
| Small | 12px (text-xs) | 400 (normal) | 1.33 (16px) | Timestamps, duration, metadata row, helper text, badge labels |

Note: Two weights only -- 400 (normal) for body/small text, 600 (semibold) for labels/headings. The shadcn radix-nova preset uses `text-sm` (14px) as the default body size across all components (cards, tabs, buttons). Page headings use `text-2xl` (24px). There is no display size in this phase. The `text-base` (16px) size is used only for CardTitle default and textarea on mobile (md:text-sm override on desktop).

## Color

All colors use the shadcn/Tailwind CSS variable system with oklch values. No custom hex colors.

| Role | CSS Variable | oklch Value (light) | Usage |
|------|-------------|---------------------|-------|
| Dominant (60%) | --background | oklch(1 0 0) | Page background, card background |
| Secondary (30%) | --muted | oklch(0.97 0 0) | Sidebar background, muted surfaces, tabs list bg, empty state icon bg |
| Accent (10%) | --primary | oklch(0.205 0 0) | Primary CTA buttons ("Start Pipeline"), default badges, sidebar active item |
| Destructive | --destructive | oklch(0.577 0.245 27.325) | Failed status indicators, error messages, retry prompts |

## Layout Patterns

### Page Structure

All Phase 35 pages follow the established pattern from Phase 34:

```
[Sidebar] | [Content Area (p-6)]
           |
           | [Breadcrumb (mb-4)]
           | [Page Header (mb-6): h1 + subtitle]
           | [Page Content]
```

### Breadcrumb Pattern

Breadcrumbs use `text-sm text-muted-foreground` with `ChevronRight` (size-3.5) separators. Current page segment is `font-medium text-foreground`. All ancestor segments are links with `hover:text-foreground transition-colors`.

- New Run: Dashboard > Project > New Pipeline Run
- Run Detail: Dashboard > {Project Name} > {Run Name}

### Run Detail Page Layout

```
[Page Header: run name + status badge]
[Metadata row: Started time | Duration | Steps x/y]
[Retry button (if failed)]

[Collapsible Use Case card]

[Step Timeline]
  [StepLogPanel] -- for each step
    [Timeline dot (colored by status)]
    [Timeline line (border)]
    [Step row: chevron + name + badge + duration]
    [Expanded: log pre block OR error card with retry]
```

### Step Timeline Visual Pattern

- Vertical left-side timeline using colored dots (size-2.5 rounded-full) connected by 1px border lines
- Dot colors: green-500 (complete), blue-500 (running), destructive (failed), muted-foreground/30 (pending)
- Steps auto-expand when status is "failed" or "running"
- Collapsed steps show: chevron + display name + status badge + duration (right-aligned)
- Expanded content indented 20px (ml-5) from step name

### Run Card Layout (List Item)

```
[Card with hover border effect]
  [Top row: Run name (truncate) | Status Badge (right)]
  [Optional: Project name (muted, xs) -- global view only]
  [Optional: Use case excerpt (muted, truncate)]
  [Bottom row: step progress bar | agent count | started time | duration]
  [Optional: Error message (red, truncate) -- failed only]
```

### New Run Form Layout

```
[Card (max-w-2xl)]
  [CardHeader: "Run Configuration"]
  [CardContent: form]
    [Run name input + helper text]
    [Use case textarea (6 rows) + validation]
    [File upload drop zone (dashed border, centered icon+text)]
    [File list (if files attached)]
    [Submit button (full width)]
```

## Interaction Patterns

### Polling (Run Detail)

- 5-second interval polling via `router.refresh()` while run status is "pending" or "running"
- Polling stops automatically when run reaches "complete" or "failed"
- Phase 36 will replace with Supabase Realtime subscriptions

### Form Submission

- `useActionState` for server action form submission
- Submit button disabled until use case has 10+ characters
- Pending state shows spinner + "Starting Pipeline..." text
- Successful submission redirects to run detail page via server-side `redirect()`

### File Upload

- Drag-and-drop zone with visual feedback (border-primary bg-primary/5 on drag-over)
- Click-to-browse fallback
- File list below drop zone with file name + remove (X) button
- No file size limit enforced in UI (server-side Supabase Storage handles limits)

### Step Expansion

- Click step row to toggle expand/collapse
- Failed and running steps auto-expand on initial render
- Chevron icon rotates (ChevronRight collapsed, ChevronDown expanded)

### Retry Flow

- Run-level: "Retry Pipeline" button in header (only visible when run.status === "failed")
- Step-level: "Retry from this step" button inside expanded failed step
- Both trigger the same `retryPipeline` server action
- Button shows spinning icon during retry, re-enables after completion
