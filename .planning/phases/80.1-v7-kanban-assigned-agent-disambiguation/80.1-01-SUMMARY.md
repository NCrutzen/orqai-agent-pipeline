# Phase 80.1 Plan 01: Disambiguate triage helpers Summary

Surfaced and fixed Phase 80 UAT Issue #2 — predicted rows on the v7 KanbanBoard were uniformly attributed to "Stage 3 Dispatcher" regardless of which worker (Stage 1 screen-worker, Stage 2 invoice-copy handler, or Stage 3 classifier) actually emitted them. Both swarm-bridge triage helpers now branch on `tool_outputs` markers to produce the correct per-stage agent label while keeping all variants in the same "progress" lane.

## What Shipped

- **Marker-based attribution** in `web/lib/automations/swarm-bridge/sync.ts`:
  - `intent_first_pass` → "Stage 3 Dispatcher"
  - `handler_output` → "Copy-Document Agent"
  - `stage1_category` → "Screen Worker"
  - (none) → "Pipeline" catch-all
- **Helper signature change** — both helpers now accept `{ status; tool_outputs }`; all four call sites in `sync.ts` (buildTriageJob × 2, buildTriageEvents × 2) updated to pass the full row. `AgentRunRow` already carries `tool_outputs`, so no type/schema change required.
- **Test scaffold** at `web/lib/automations/swarm-bridge/__tests__/sync.test.ts` — 7 cases covering all three markers plus the catch-all, plus the inverse stage-lane assertions.

## TDD Cycle

- **RED** (commit `b72c2ec`): added `sync.test.ts` and exported both helpers from `sync.ts`. Tests ran with the old single-status switch — 6/7 failed, 1 incidental pass (Stage 2 `handler_output` coincidentally matched the old "Copy-Document Agent" default).
- **GREEN** (commit `6acc563`): swapped helper signatures, introduced `predictedStageHint()` with explicit precedence, rewrote `case "predicted"` in both helpers, updated four call sites. All 7 tests green.
- **REFACTOR**: not needed — implementation is the minimal change to satisfy the marker contract.

## Out of Scope (Explicitly Deferred)

- **Stage 1 sub-attribution** (Screen Worker vs Label Resolver vs Spotcheck Sampler). Label Resolver and Spotcheck Sampler write to `automation_runs`, not `agent_runs`, so they never reach these helpers. Future follow-up if more `tool_outputs` markers appear.
- **Stage 0 safety worker** — writes `automation_runs.status='predicted'`, not `agent_runs`, so unaffected by this fix (different table, different code path).
- **Backfill of historical swarm_jobs rows** — bridge sync runs continuously, affected rows refresh on the next tick. No DB migration needed.
