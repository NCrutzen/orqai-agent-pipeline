## 7. Telemetry audit verdict — ✅ PASS

### Static check
- `web/lib/inngest/functions/classifier-label-resolver.ts` lines ~190-220 emit one `pipeline_events` row per branch with `stage: 2` and `swarm_type: swarm_type ?? "debtor-email"`. Verified via:
  ```
  rg -nF 'stage: 2' web/lib/inngest/functions/classifier-label-resolver.ts
  → web/lib/inngest/functions/classifier-label-resolver.ts:212:  stage: 2,
  ```
- `web/lib/inngest/functions/debtor-email-icontroller-tagger.ts` does **not** emit `pipeline_events`. This is **by design** — the tagger is a Stage 2 follow-up *side-effect worker* (DOM tagging in iController), not a stage gate. The Stage 2 stage-gate emit comes from the customer-resolver. Emitting a duplicate `stage=2` row from the tagger would double-count the stage in dashboards.

### Live check (Supabase MCP `execute_sql`, 2026-05-22)

```sql
SELECT stage, swarm_type, COUNT(*) AS n, MAX(created_at) AS last_seen
FROM pipeline_events
WHERE created_at > now() - interval '7 days'
  AND swarm_type = 'debtor-email'
GROUP BY stage, swarm_type ORDER BY stage;
```

| stage | swarm_type | n | last_seen |
|---|---|---|---|
| 0 | debtor-email | 250 | 2026-05-21 18:56:07Z |
| 1 | debtor-email | 247 | 2026-05-21 18:56:11Z |
| **2** | **debtor-email** | **87** | **2026-05-21 18:56:25Z** |
| 3 | debtor-email | 74 | 2026-05-21 18:56:42Z |
| 4 | debtor-email | 36 | 2026-05-18 12:45:11Z |

**Verdict: PASS.** `stage=2` is actively emitted (87 rows / 7 days), with `swarm_type='debtor-email'` consistently labelled. No Wave 2 telemetry-patch task required. D-04 satisfied as a pure verification.
