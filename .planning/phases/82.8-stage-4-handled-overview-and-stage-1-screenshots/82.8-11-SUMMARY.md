# 82.8-11 — Mount detail-pane + screenshot strip for Auto-archived rows

## Outcome

Two surgical edits in `stage-4/client-shell.tsx`:

1. **Widened `selectedEmailId` resolution.** Previously derived only from
   `selectedRow?.result?.email_id` (handler-error kanban). Now also searches
   `needsReviewRows` (kanban-shape) and `autoArchivedRows` (where row.id is
   `pipeline_events.id` and `email_id` is a direct field). Body + timeline
   lookups now resolve for every section.

2. **Hoisted `UnifiedDetailPane` to a sticky right column.** Previously
   rendered inside Section 1 (Handler error) `<CollapsibleContent>`,
   invisible when any other section drove selection. Wrapped all three
   section collapsibles in a 2-col grid (`minmax(640px, 1fr) 540px`); pane
   is rendered in the right column with `position: sticky` so it stays
   visible regardless of scroll position or which section is expanded.

Tasks 3 + 4 from the plan were already implemented by Plan 82.8-05 and
82.8-07 — `bodyMap` covers auto-archived rows (inline-fill at
`page.tsx:166-175`), `timelineMap` covers them (`emailIds` includes
auto-archived at `page.tsx:163-164`), and `screenshotPathsByEmailId` is
already keyed by email_id with the remap to row.id in client-shell
(`screenshotPathsByRowId` at lines 187-212). The reason the pane stayed
empty before today was structural (where it was rendered + how
selectedEmailId was derived), not data-side.

## Files modified

- `web/app/(dashboard)/automations/[swarm]/stage-4/client-shell.tsx`

## Verification

- `tsc --noEmit` exit=0 ✓
- Render path: any-section click → setSelected → useSelection updates →
  `selectedUnified` resolves against `visibleUnified` (all 3 sections
  unioned) → `selectedEmailId` resolves from the matching section's row
  shape → bodyMap/timelineMap/screenshotPathsByRowId all hit → pane
  renders with body, timeline, and screenshot strip.
- Handler-error widget (`extrasBelowPipeline={handlerErrorWidget}`)
  conditionally renders only when `selectedRow` is non-null (i.e. a
  handler-error row is selected). Auto-archived/Needs-review selections
  pass `null` to that slot, which the pane already handled.

## Acceptance env caveat

Backfilled Auto-archived rows in acceptance have no
`screenshot_*_path` because that env never reached
`icontroller_tag_status='tagged'`. The pane mounts and the strip renders
its empty-state copy ("No screenshots available") per `StageScreenshotStrip`
implementation from Plan 06. To see real before/after thumbnails in
acceptance, a non-dry-run iController capture must run; in prod the
strip will populate as live captures roll through.

## Layout notes

- Right column is `position: sticky; top: var(--space-3)` so it follows
  scroll. If sticky behaviour clashes with other page layout, swap to
  fixed-grid-only (no sticky) — the layout still works either way.
- Width 540px matches the established detail-pane width from Phase
  82.7.3.
- Grid alignment is `align-items: start` so the sticky pane reference
  point doesn't grow to match the (much taller) sections column.

## Non-goals (deferred)

  acceptance comfortably.
- Override/reclassify actions on auto-archived rows (Stage 4 stays
