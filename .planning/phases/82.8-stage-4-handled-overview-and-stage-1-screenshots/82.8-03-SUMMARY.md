# Phase 82.8 Plan 03: Screenshot Path Plumbing Summary

Capture-side D-04 change â€” plumbs the storage `path` returned by `captureScreenshot()` (previously discarded) through `labelEmailInIcontroller` and into the new `debtor.email_labels.screenshot_before_path` / `screenshot_after_path` columns added in Plan 82.8-01. New tagged emails now persist stable paths alongside the 1-hour-expiring signed URLs so detail-pane (Plans 04/05) can sign URLs on demand.

   - `errShot` brand-mismatch branch
   - `errShot` SELECTION_DID_NOT_STICK branch
   - `afterShot` (line ~173) â€” labeled-path after-shot
   - `errShot` outer-catch error-shot

   - `already_labeled` â†’ `null` (no screenshot taken)
   - `skipped_conflict` â†’ `null` (no screenshot taken)

   - `failed (SELECTION_DID_NOT_STICK)` â†’ `beforePath` + `afterPath`

   - `failed (catch block)` â†’ `beforePath` + `afterPath` (whichever was populated before the throw)

## Decisions Made

- **Capture path even on error branches** (brand_mismatch, SELECTION_DID_NOT_STICK, outer-catch). These branches already capture `errShot?.url` â€” including `errShot?.path` keeps the URL/path pairs aligned so the read-side can always assume both are present (or both `null`) per branch.
- **`beforePath` set unconditionally to `before?.path ?? null` after the before-shot capture.** Mirrors how `beforeUrl` is set. If the before-shot succeeds but the subsequent typeahead/click throws, the catch-block return carries the populated `beforePath` (correct: a before-screenshot exists in storage).
- **Tagger writes guarded by truthy check.** Mirrors URL pattern â€” when no screenshot was taken (e.g. `already_labeled` short-circuit), the path is `null`, the guard fails, and `updatePayload` simply omits the column (preserving any existing NULL on the row).
