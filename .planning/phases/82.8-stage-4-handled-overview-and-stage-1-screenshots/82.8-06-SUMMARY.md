# Phase 82.8 Plan 06: StageScreenshotStrip Summary

Reusable before/after screenshot strip component built in isolation for the Stage 1 detail-pane audit-expander. File-disjoint from parallel 82.7.1 work; mount-point follows in Plan 07.

## What shipped

- **`StageScreenshotStrip`** — new component at `web/components/automations/bulk-review/audit/StageScreenshotStrip.tsx`. Signature:
  ```ts
  function StageScreenshotStrip(props: {
    beforePath: string | null;
    afterPath: string | null;
    emptyCopy?: string;
  }): JSX.Element
  ```
  Renders two `ScreenshotThumb` thumbnails side-by-side (Before / After) when both paths present; em-dash placeholder for missing sides; muted "No screenshots available" copy when both are null. Custom `emptyCopy` prop overrides the default empty text.

- **Test suite** — 5 Vitest cases at `__tests__/StageScreenshotStrip.test.tsx`:
  1. Both paths → two thumbs with labels "Before" / "After"
  2. Only before → before thumb + dash placeholder for after
  3. Only after → dash placeholder for before + after thumb
  4. Both null → empty-state copy, no thumbs rendered
  5. `emptyCopy` prop overrides default text
