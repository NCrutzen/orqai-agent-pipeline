# Phase 69 Plan 03: Codegen + registry plumbing Summary

Wired the live entity_brand registry → TypeScript codegen → coordinator/types.ts re-export chain. After Plan 02 the live DB carries jsonb-of-objects with 5 Benelux brands; Plan 03 makes the TS type system consume it via a deterministic codegen step, replacing the hardcoded `ENTITY` literal with a generated literal-union.

## What Changed (One-liner)

`npm run codegen` now generates `Entity` and `ENTITY_BRANDS` from `swarms.entity_brand`; `coordinator/types.ts` re-exports them; `web/lib/swarms/registry.ts` exposes `loadEntityBrand` / `loadEntityBrandRegister`; `BODY_VERSION` bumped to `2026-05-04.v2`.

## Tasks Executed

### Task 1: Codegen runs against live registry

Updated `web/scripts/gen-entity-types.ts` from Wave 1 skeleton to full implementation:
- Loads `web/.env.local` via `dotenv` (matches `scripts/audit-orq-agents.ts`).
