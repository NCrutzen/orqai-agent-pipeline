## Cross-Phase Integration (11/12)

### Integration Issue: Holdout Dataset ID Schema Mismatch

**Severity:** High (breaks E2E iterate re-test flow)
**Affects:** ITER-05, ITER-06 at runtime

**Problem:** `iterator.md` Phase 7 reads `dataset.per_agent.{agent_key}.holdout_dataset_id` from `test-results.json`, but:
- The template uses `dataset.per_agent_datasets[]` (different key name: `per_agent_datasets` vs `per_agent`)
- Each entry has a single `dataset_id` field, not per-split IDs (`holdout_dataset_id`, `train_dataset_id`, `test_dataset_id`)
- `tester.md` Phase 5 Step 5.3 produces per-split IDs as log output but Phase 8 Step 8.5 only writes the single `dataset_id` to JSON

**Fix required (2 files):**
1. `orq-agent/templates/test-results.json` — add `holdout_dataset_id`, `train_dataset_id`, `test_dataset_id` per agent entry; align key name with iterator expectation
2. `orq-agent/agents/tester.md` Phase 8 Step 8.5 — write all three dataset IDs to test-results.json output
