## Integration Check Results

### Critical Findings

**1. Wrong memory tool identifiers in tool-catalog.md**
`tool-catalog.md` lists `write_to_memory_store` and `delete_from_memory_store` but the authoritative reference (`orqai-agent-fields.md`) uses `write_memory_store` and `delete_memory_document`. Swarms using memory write/delete will get invalid tool configs.
Fix: 2 lines in `orq-agent/references/tool-catalog.md`

**2. OWNER/REPO placeholders in 5 command files**
`update.md`, `deploy.md`, `test.md`, `iterate.md`, and `help.md` contain literal `OWNER/REPO` in GitHub URLs. `install.sh` has the correct values (`NCrutzen/orqai-agent-pipeline`). The update command is completely non-functional.
Fix: Global find-replace in 5 files

**3. Research brief not passed to orchestration generator**
Orchestrator Wave 3 passes blueprint + spec files to orchestration generator but omits the research brief path. KB Design section in ORCHESTRATION.md will lack chunking strategy and metadata details from the researcher.
Fix: Add research brief path to Wave 3 orchestration generator inputs in `orq-agent.md`

### Medium Findings

**4. TOOLS.md not in spec-generator files_to_read**
The orchestrator passes TOOLS.md path dynamically, but the spec generator must remember to call Read on it — it's not guaranteed by `<files_to_read>`.

**5. orqai-evaluator-types.md orphaned**
Created in Phase 5 as V2.0 infrastructure. Not loaded by any current subagent. Evaluator recommendations in specs use informal names.

### Low Findings

**6. Researcher missing agentic-patterns.md and tool-catalog.md**
Context management strategy recommendations are not grounded by the agentic patterns reference.
