Sketch 009 row-strip-review-indicators · phase 82.5

Q: How do we show verdict status on rows without clashing with stage badges?

Variant A — Dot before stage badge

A small 8px colored dot at the very left of each row, before the stage badge. Color encodes verdict; empty dashed circle = no feedback yet.

Reads as "is this row touched?" at a glance. The dot is decoupled from the stage badge, so noise/intent treatment stays unchanged.

Path of least resistance — adds a 14px grid column, no badge restyling needed.

Variant B — Verdict letter chip

A small letter chip (C / O / U) clipped to the stage badge. Same color language as the dot, but with a glyph for unambiguous read-back even in color-blind viewing.

Trade-off: doubles the visual weight on the badge edge. Stage badge already does work — this adds adjacency density.

Best for: when a quick legend on hover isn't acceptable; accessible without color.

Variant C — Left-edge color bar

A 3px vertical color bar on the row's left edge encodes verdict. No new grid column, no badge restyling, no glyph competition.

The bar reads as a row-level annotation, scannable down a column without parsing chip text.

Trade-off: selected-state already uses a left-edge accent. Co-locating two signals can be confusing — needs careful color contrast.

Risk: collides with the selected-row left border. Selected row needs a different treatment (e.g., wider bar or solid background).
