Q: What treatment makes "Save (revisit)" vs "Confirm (done)" immediately differentiable?

Variant A — Current (pill + pill)

What ships today: two pill buttons of equal weight, differentiated only by color (blue Save vs lime Confirm). Symmetric layout.

Strength: visually balanced, doesn't push the operator toward either action.

Weakness: ambiguity. Operator B asks "which one closes the row?" — color doesn't carry that meaning intuitively.

Variant B — Primary + secondary text

Confirm becomes the primary action (filled lime button, larger). Save shrinks to a subtle text button on the left ("Save note (revisit later)").

Visual weight encodes the typical-case path: the operator's goal is to close the row, not save drafts. Save stays available but stops competing for attention.

Risk: makes Save feel less important than it is — the corpus quality depends on those notes.

Variant C — Iconography + microcopy

Both buttons keep equal weight but gain icons + secondary microcopy. Save: 💾 with "revisit later." Confirm: ✓ with "close + done."

Explicit semantics inline. No color-only encoding. Two-line buttons take more space but eliminate ambiguity.

Risk: button height grows. May force vertical stacking on narrow viewports.
