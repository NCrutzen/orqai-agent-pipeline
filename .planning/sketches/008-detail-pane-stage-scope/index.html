Sketch 008 detail-pane-stage-scope · phase 82.5

Q: Which stages should the detail pane show, and what does the bottom "Approve" mean?

Variant A — Run + current only, Confirm per stage

The pane shows only stages that actually ran for this row. On Stage 1 tab: Stage 0 (passed) + Stage 1 (current). Stages 2/3/4 are hidden — they belong to other tabs.

The bottom bar drops "Approve". Each stage has its own Confirm button under its details (matches Sketch 011-C treatment). Skip / Reject / Next stay as row-navigation.

  • Mental model: the Stage 1 tab is for Stage 1 decisions. Period.
  • No future stages: if you want to make a Stage 3 call, switch to the Stage 3 tab.
  • Approve doesn't exist as a row-level concept — each stage's verdict is independent.

Strongest opinion. Closest to the operator mental model. Removes the most ambiguity.

Variant B — Full timeline, future stages collapsed

Keeps the full pipeline visible for context, but collapses future-yet-to-run stages into a single thin pill: "Stages 2-4 — haven't run yet · expand."

The bottom "Approve" stays, with relabeled meaning: "Approve all current verdicts as correct." The operator confirms the run stages in one click. Future stages aren't part of that decision.

  • Mental model: show the journey so operators see how far the email got.
  • Future stages: collapsed but accessible. Expand if you want to pre-emptively override one.
  • Approve: means "every stage that ran for this row is correct."

Compromise. Preserves the unified-shell uniformity from Phase 76 while reducing visual noise. But keeps the ambiguous "Approve" semantics.

Variant C — Run + current, bottom button reflects active stage

Same as A on stage scoping (hide future stages). Different on actions: the per-stage Confirm chip is gone — the bottom button is the single action surface, and its label reflects the active tab:

  • Stage 1 tab → "✓ Confirm Stage 1"
  • Stage 3 tab → "✓ Confirm Stage 3"

One Confirm per row per tab. The note textarea stays inline under the active stage. Skip/Reject still navigate.

Cleanest action model. Bottom bar always means "approve the stage this tab is about." Easier to remember than per-stage Confirm chips.
