Sketch 008 feedback-read-back-state · phase 82.5

Q: How does StageFeedbackPanel look when re-opening a row that already has feedback?

Variant A — Pre-fill + verdict chip

The simplest read-back: textarea pre-populated with the operator's latest prose, verdict shown as a chip next to the stage label. No history surfaced. Lowest cognitive load; matches how most "draft" UIs behave.

Trade-off: cross-operator context invisible. Operator B doesn't know A wrote a note last week.

Best for: solo operator per email_id, low-frequency revisits.

Variant B — Pre-fill + "Others said" collapsible

Same pre-fill as A, plus a dashed-separator block below the textarea: "What others said (2)" — collapsible. Each prior cross-operator note shown as a card with author + timestamp + verdict.

Operator's own prior notes go in the textarea; others' notes stay in the collapsible — never overwritable.

Best for: multi-operator review, training corpus building.

Variant C — Verdict timeline view

The textarea collapses into a horizontal timeline showing all prior verdicts (yours and others) as pill rows. Click any row to expand the full note + edit if it's yours.
