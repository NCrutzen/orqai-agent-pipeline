An AI Agent Team
that runs Debtor Email — and is ready for Sales.

Eleven specialised agents process every customer email end-to-end across Outlook and iController. They sort the noise, identify the customer, decide what's being asked, and draft the reply — leaving humans to do only the work that actually needs human judgment.

The same team is being prepared to run Sales Email next, and to learn-and-promote new patterns autonomously after that.

Section 1 · Today
In production · debtor email shared mailboxes
The Debtor Email Pipeline
Every email lands in Outlook, gets passed across the team, and ends up either auto-handled or as a ready-to-send draft inside iController. We never auto-send — humans always have the final click.
Team A · Triage Every inbound email goes through these four agents, in order.

Safety Agent

Stage 0 · Input safety
Live
  • Detects prompt-injection attempts before any other agent reads the email.
  • Enforces a hard cost ceiling per email — no run can spiral.
  • Suspicious mail goes to a human-only review lane.

Triage Agent

Stage 1 · Deterministic sort
Live
  • Auto-handles three noise categories end-to-end with zero human touch and zero AI cost: out-of-office, auto-replies, payment confirmations.
  • Outlook: applies category label + archives the message.
  • iController: removes the email from the operator queue.

Customer-ID Agent

Stage 2 · Resolver
Live
  • 4-layer match: thread → sender → invoice number → AI tiebreaker.
  • Looks up customers in NXT; brand-aware (Smeba, Sicli, Berki, Fire Control…).
  • iController: tags the email under the matched account.

Intent-Routing Agent

Stage 3 · Coordinator
Live
  • Reads the email and picks one of 8 intents: copy doc, dispute, address change, credit, contract, peppol, general…
  • Multilingual: NL · EN · DE · FR.
  • Hands off to the right specialist below.
Team B · Handlers Specialists. Each does one job in iController, in the customer's language.

Invoice-Copy Drafter

Stage 4 · Handler
Live
  • Pulls the right invoice PDF from NXT.
  • iController: composes a draft reply with the PDF attached, in the customer's language.
  • Operator hits Send. We never auto-send.

What the team does in Outlook

Inbox stays clean — automatically.
  • Reads inbound email from the shared mailbox
  • Applies a colour-coded category label per intent
  • Archives noise (out-of-office, auto-replies, payment confirmations)
  • Leaves only emails that need a human in the inbox

What the team does in iController

Drafts ready, queue cleared.
  • Logs into iController via secure browser automation
  • Tags each email under the matched customer account

The same team learns Sales Email.
The pipeline was designed cross-domain from day one. To onboard the Sales mailbox into the same workforce we add only two parts: a SugarCRM customer lookup and a Sales-specific Intent-Routing Agent. Every other agent — Safety, Triage, all the Handlers — is reused as-is.

Section 3 · After
Phases 71 → 72 · the platform learns from every override
Autonomous promotion — the workforce gets cheaper every week.
When the operator corrects an agent in Bulk Review, the platform writes that signal into a "Learning Inbox". Any pattern that an LLM keeps handling the same way gets promoted to a deterministic rule — faster, cheaper, more reliable. We graduate work down the cost curve automatically.
3a

Bulk Review · 4-axis override

Phase 71

Each email gets one row showing category · customer · intent · output. Operators correct any one axis with a single click. Every correction is its own learning signal — we know exactly which agent missed.

Wrong category? Wrong customer? Wrong intent? Wrong reply?

Learning Inbox · graduated automation

Phase 72
